🧠 What is Machine Learning?

Tom Mitchell's Definition (1997): "A computer program is said to learn from experience E with respect to some class of tasks T and performance measure P, if its performance at tasks in T, as measured by P, improves with experience E."

Think of it this way: a spam filter (task T) improves at recognising spam (P) the more emails it processes (E). The machine "learns" patterns from data rather than being explicitly programmed with rules.

Traditional Programming vs. Machine Learning

🖥️ Traditional Programming

Input: Data + Rules → Output: Answers

You tell the computer exactly what to do step-by-step. If the rules are wrong or incomplete, the program fails.

🤖 Machine Learning

Input: Data + Answers → Output: Rules

You feed the computer data and expected outcomes; it figures out the rules itself.

Why ML? Real-World Examples

  • 📧 Email filtering – Classify spam vs. not spam
  • 🏦 Loan approval – Predict creditworthiness from financial history
  • 🩺 Medical diagnosis – Detect cancer from X-ray images
  • 🎵 Recommendation – Spotify suggests songs you'll love
  • 🚗 Self-driving cars – Navigate using sensor data

🗂️ Learning Paradigms

👩‍🏫 Supervised Learning

Learning with a teacher — the algorithm trains on labeled data (input-output pairs).

Goal: Learn a mapping f(x)→y.

Examples:

  • Classification: Is email spam? (Yes/No)
  • Regression: Predict house price (₹12L)

Algorithms: KNN, Linear Regression, SVM, Decision Trees, Neural Networks

🔍 Unsupervised Learning

Learning without labels — discover hidden patterns or structure in data.

Goal: Find structure, clusters, or compressed representations.

Examples:

  • Clustering: Group customers by buying habits
  • Dimensionality reduction: PCA

Algorithms: K-Means, DBSCAN, PCA, Autoencoders

🎮 Reinforcement Learning

Learning through trial and error — an agent takes actions in an environment and receives rewards/penalties.

Goal: Maximise cumulative reward over time.

Examples:

  • Game playing (AlphaGo beat world champion Go player)
  • Robot navigation
  • Dynamic pricing

Algorithms: Q-Learning, Policy Gradient, Actor-Critic

🔵 K-Nearest Neighbours (KNN)

KNN is the simplest and most intuitive ML algorithm. The idea: "You are like your neighbours." To classify a new point, find its K nearest neighbours in the training data and take a majority vote.

How KNN Works — Step by Step

1
Store all training data (lazy learner — no training phase)
2
For a new query point, calculate distance to every training point
3
Select K nearest neighbours (smallest distances)
4
For classification: majority class wins. For regression: average the K values.

Choosing K

Small K (e.g., K=1): Very sensitive to noise. A single outlier can change the prediction. High variance, low bias → overfitting.
Large K (e.g., K=N): Very smooth decision boundary. Ignores local patterns. Low variance, high bias → underfitting.

Rule of thumb: Try K = √N where N is the number of training samples. Use odd K to avoid ties in binary classification. Always validate with cross-validation.

📏 Distance Metrics

KNN relies entirely on distance. Choosing the right distance metric matters enormously.

Euclidean Distance (L2 Norm)

The straight-line distance — like measuring with a ruler.

d(x,y) = √[(x₁−y₁)² + (x₂−y₂)² + ... + (xₙ−yₙ)²]

For 2D: d = √[(x₁−y₁)² + (x₂−y₂)²]

Best for: Continuous numeric features, when all features have the same scale.

Manhattan Distance (L1 Norm / City-block Distance)

Distance along axes only — like a taxi navigating city blocks.

d(x,y) = |x₁−y₁| + |x₂−y₂| + ... + |xₙ−yₙ|

Best for: When features have different units/scales, high-dimensional data, robust to outliers.

⚖️ Data Normalization

When features have different scales (e.g., Income in thousands, Age in years), large-scale features dominate the distance calculation. Normalization fixes this.

Min-Max Normalization

x_norm = (x − x_min) / (x_max − x_min) → range [0, 1]

Why It Matters — Example

Without normalization: Income (50,000) vs Age (30). The distance would be dominated entirely by income, making age irrelevant!

With normalization: Both features contribute equally to the distance calculation.

📐 Parametric vs Non-Parametric Models

Parametric Models

Make assumptions about the form of the mapping function. Have a fixed number of parameters regardless of training size.

Linear Regression: y = w₀ + w₁x₁ + w₂x₂

Parameters: w₀, w₁, w₂ — fixed at 3 regardless of whether you have 10 or 10M training points.

Advantages:

  • Fast to learn and predict
  • Require less data
  • Highly interpretable

Disadvantages:

  • Strong assumptions may not fit the data
  • Limited model complexity

Examples: Linear Regression, Logistic Regression, Naive Bayes, Perceptron

Non-Parametric Models

Make no strong assumptions about the mapping function. The number of "parameters" grows with training data.

KNN: Stores all training data. With 1000 samples, it "has" 1000 parameters (all the data points).

Advantages:

  • More flexible — can fit complex patterns
  • No assumptions about data distribution
  • Works well with enough data

Disadvantages:

  • Slow to predict (especially KNN)
  • Require more data
  • Can overfit if not regularised

Examples: KNN, Decision Trees, SVMs, Random Forests, Neural Networks

Key Insight: KNN is a "Lazy Learner"

KNN does NO work during training — it just memorises the dataset. All computation happens at prediction time. This makes training instant but prediction slow for large datasets (O(N) per query). This is why it's called a lazy learner (vs. eager learners like Logistic Regression that build a model upfront).
