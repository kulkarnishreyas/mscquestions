🎯 Model Evaluation Overview

Once we train a model, how do we know if it's actually good? We need evaluation metrics — quantitative measures of how well the model performs on data it hasn't seen before.

Why We Need Evaluation Metrics

The core problem: A model can memorise training data perfectly (100% accuracy on training set) but completely fail on new data. We need to measure performance on unseen data to know if the model has actually learned useful patterns.
Generalisation: The ultimate goal is a model that generalises well — performs consistently on both seen (training) and unseen (test) data. Metrics help us detect when this breaks down.

Train / Test Split

The simplest evaluation strategy: split your dataset into two parts before training.

Key Rule: The test set must NEVER be seen by the model during training. Evaluating on training data gives falsely optimistic results — this is called data leakage.

Limitation of single split: The result depends heavily on which samples end up in which set — a lucky or unlucky split can mislead you. This is why cross-validation exists.

🔁 K-Fold Cross-Validation

Instead of one fixed split, K-Fold CV splits the data into K equal parts ("folds") and trains/evaluates K times, using a different fold as the test set each time.

Why it's better: Every sample gets to be in the test set exactly once, and every sample gets to be in the training set K−1 times. The final score is the average across all K evaluations — far more reliable than a single split.

5-Fold Cross-Validation — Visual Diagram

Formula

CV Score = (1/K) × Σᵢ scoreᵢ = (score₁ + score₂ + ... + scoreK) / K

Choosing K

K = N (LOOCV)Lowest bias, highest computeTiny datasets only

Leave-One-Out Cross-Validation (LOOCV)

LOOCV is a special case where K = N (the number of samples). In each iteration, exactly one sample is held out as the test set and all remaining N−1 samples are used for training.

Advantage: Least biased estimate — uses almost all data for training every time. No randomness in the split.
Disadvantage: Extremely expensive for large datasets (N models trained). High variance in the score estimate because each test set has only 1 sample.

📈 Overfitting & Underfitting

These are two fundamental failure modes of machine learning models. Understanding them is key to building models that actually work on new data.

The Three Zones

Understanding the Curves

Underfitting (High Bias)

Model is too simple to capture the patterns in data.

  • High training error
  • High validation error
  • Both errors are similar (both bad)

Fix: More complex model, more features, less regularisation.

Good Fit

Model captures the true patterns without memorising noise.

  • Low training error
  • Low validation error
  • Small gap between them

Goal: This is what we are always aiming for!

Overfitting (High Variance)

Model memorises training data, including noise and outliers.

  • Very low training error
  • High validation error
  • Large gap between them

Fix: More data, regularisation, simpler model, dropout.

Bias-Variance Tradeoff

Every model's error can be decomposed into three parts:

Total Error = Bias² + Variance + Irreducible Noise
Bias: Error from wrong assumptions in the model (e.g., assuming data is linear when it's not). High bias → underfitting. A biased model consistently misses in the same direction.
Variance: Error from sensitivity to fluctuations in the training data. High variance → overfitting. A high-variance model changes drastically with different training sets.

As you increase model complexity, bias decreases but variance increases. The optimal model minimises their sum.

🔲 Confusion Matrix

For binary classification, a confusion matrix is a 2×2 table that shows how many predictions were correct or incorrect, broken down by class.

📐 Classification Metrics

Using the disease example (TP=50, FP=10, FN=10, TN=30), let's calculate all four key metrics.

1. Accuracy

Accuracy = (TP + TN) / (TP + TN + FP + FN)
Calculation: (50 + 30) / (50 + 30 + 10 + 10) = 80 / 100 = 0.80 = 80%

Meaning: The model got 80 out of 100 predictions correct.

When to use: Only when classes are balanced (roughly equal number of positives and negatives). Misleading with imbalanced datasets — a model that always predicts "Negative" on a 95/5 dataset gets 95% accuracy but is useless!

2. Precision

Precision = TP / (TP + FP)
Calculation: 50 / (50 + 10) = 50 / 60 = 0.833 = 83.3%

Meaning: Of all patients the model flagged as sick, 83.3% were actually sick.

When to use: When False Positives are costly. Example: Spam detection — marking important emails as spam (FP) is very bad, so we want high precision.

3. Recall (Sensitivity / True Positive Rate)

Recall = TP / (TP + FN)
Calculation: 50 / (50 + 10) = 50 / 60 = 0.833 = 83.3%

Meaning: Of all patients who actually had the disease, the model caught 83.3% of them.

When to use: When False Negatives are costly. Example: Cancer screening — missing a cancer case (FN) is catastrophic, so we want high recall.

4. F1-Score

F1 = 2 × (Precision × Recall) / (Precision + Recall)
Calculation: 2 × (0.833 × 0.833) / (0.833 + 0.833) = 2 × 0.694 / 1.667 = 0.833 = 83.3%

Meaning: The harmonic mean of precision and recall. Penalises extreme imbalances between the two (e.g., if precision=1.0 but recall=0.1, F1=0.18, not 0.55).

When to use: When you need a single metric that balances precision and recall, especially with imbalanced classes.

📉 Regression Metrics

For regression (predicting continuous values), we measure how far predictions are from actual values. Let's use: actual = [3, 5, 2, 8, 4], predicted = [2.5, 5, 4, 7, 4.5]

Mean ȳ = (3+5+2+8+4)/5 = 22/5 = 4.4

1. MAE — Mean Absolute Error

MAE = (1/n) × Σ|yᵢ − ŷᵢ|
→
MAE = (1/5) × 4.0 = 0.80

Intuitive: on average, predictions are off by 0.80 units. Less sensitive to outliers than MSE because errors are not squared.

2. MSE — Mean Squared Error

MSE = (1/n) × Σ(yᵢ − ŷᵢ)²
→
MSE = (1/5) × 5.50 = 1.10

Squares errors so larger mistakes are penalised much more. Units are squared (e.g., if predicting dollars, MSE is in dollars²).

3. RMSE — Root Mean Squared Error

RMSE = √MSE = √[(1/n) × Σ(yᵢ − ŷᵢ)²]
→
RMSE = √1.10 = 1.049

Same units as the target variable (easy to interpret). The most commonly used regression metric.

4. RSS — Residual Sum of Squares

RSS = Σ(yᵢ − ŷᵢ)²
→
RSS = 5.50

5. R² — Coefficient of Determination

R² = 1 − RSS/TSS where TSS = Σ(yᵢ − ȳ)²
TSS calculation (ȳ = 4.4):
TSS = (3−4.4)² + (5−4.4)² + (2−4.4)² + (8−4.4)² + (4−4.4)²
TSS = 1.96 + 0.36 + 5.76 + 12.96 + 0.16 = 21.2

R² = 1 − 5.50/21.2 = 1 − 0.2594 = 0.7406

Interpretation: R² = 0.74 means the model explains 74% of the variance in the target variable. R²=1 is perfect; R²=0 is as bad as just predicting the mean; R² can be negative (model is worse than predicting the mean).

6. Adjusted R²

Adj R² = 1 − (1−R²)(n−1)/(n−p−1)

where n = number of samples, p = number of predictors (features).

Why it matters: R² always increases (or stays the same) when you add more features, even if they're useless noise. Adjusted R² penalises for adding irrelevant features — it only increases if the new feature genuinely improves the model more than expected by chance.

Example (n=5, p=1): Adj R² = 1 − (1−0.7406)×(5−1)/(5−1−1) = 1 − 0.2594×4/3 = 1 − 0.3459 = 0.654

🧬 Perceptron Learning

Biological Inspiration

The perceptron (1957, Frank Rosenblatt) is inspired by the biological neuron. A neuron receives signals through dendrites, sums them up, and fires an output signal if the total exceeds a threshold.

Perceptron Model Diagram

Perceptron Decision Rule

ŷ = 1 if Σ(wᵢ × xᵢ) + b ≥ θ else ŷ = 0

Perceptron Learning Rule

wᵢ ← wᵢ + η × (y − ŷ) × xᵢ

where η (eta) is the learning rate, y is the true label, ŷ is the predicted label.

  • If prediction is correct (y = ŷ): no update (y−ŷ = 0)
  • If predicted 0, should be 1: weights increase for active inputs
  • If predicted 1, should be 0: weights decrease for active inputs

AND Gate — Simple Example

AND Gate: Output = 1 only if both inputs are 1.
With weights w₁=1, w₂=1 and threshold θ=1.5:
  (0,0): 0+0=0 < 1.5 → ŷ=0 ✓    (1,0): 1+0=1 < 1.5 → ŷ=0 ✓
  (0,1): 0+1=1 < 1.5 → ŷ=0 ✓    (1,1): 1+1=2 ≥ 1.5 → ŷ=1 ✓

Car Door Alarm Example

Target: Alarm = 1 (sound alarm), Alarm = 0 (no alarm)

📐 Linear Regression & Gradient Descent

Simple Linear Regression Model

ŷ = w₀ + w₁ × x

where w₀ is the intercept (bias) and w₁ is the slope (weight). The goal is to find w₀ and w₁ that best fit the data.

Cost Function

We need a measure of how wrong the current weights are. The Mean Squared Error cost function is:

J(w₀, w₁) = (1/2n) × Σᵢ(ŷᵢ − yᵢ)²

The 1/2 is added for mathematical convenience (it cancels with the 2 from the derivative). Minimising J gives us the best fitting line.

Gradient Descent Update Rules

Gradient descent iteratively updates the weights by moving in the direction of steepest descent of J:

w₀ ← w₀ − η × (∂J/∂w₀) = w₀ − (η/n) × Σ(ŷᵢ − yᵢ)
w₁ ← w₁ − η × (∂J/∂w₁) = w₁ − (η/n) × Σ(ŷᵢ − yᵢ) × xᵢ

Intuition Behind Gradient Descent

Analogy: Imagine you're blindfolded on a hilly landscape and want to reach the valley (minimum cost). Each step you take is in the direction you feel is "downhill". The learning rate η determines how big each step is:
  • Too large η: You overshoot the minimum and may diverge (bounce around or fly off)
  • Too small η: You converge very slowly (takes thousands of tiny steps)
  • Just right η: Converge efficiently to the minimum
Learning RateBehaviourRecommendation
η = 0.001Very slow convergenceUse when oscillating
η = 0.01Good balance (our example)Common starting point
η = 0.1Fast but may overshootMonitor loss carefully
η = 1.0Likely divergesToo high for most problems
