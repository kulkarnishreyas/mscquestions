Multiple Linear Regression

Simple linear regression models one feature: y = w₀ + w₁x. Real-world problems have many features — house price depends on size, bedrooms, location, age of property, etc. Multiple Linear Regression extends this naturally to any number of features.

The Model

Y = w₀ + w₁x₁ + w₂x₂ + ... + wₙxₙ

Where:

  • w₀ — bias / intercept (the predicted value when all features are 0)
  • w₁ … wₙ — feature weights (how much Y changes per unit change in each feature)
  • x₁ … xₙ — input features (e.g., size in sq.ft., number of bedrooms)

Matrix (Vectorised) Form

For N training samples each with n features, we stack everything into matrices. This is the compact, computationally efficient form:

Assumptions of Linear Regression

1. Linearity

The relationship between features and target is linear. Violations cause systematic prediction errors.

2. Independence

Observations are independent of each other. Time-series data often violates this (autocorrelation).

3. Homoscedasticity

Variance of residuals is constant across all predicted values (no "funnel" patterns in residual plots).

4. Normality of Residuals

Residuals (ε = y − ŷ) should be approximately normally distributed. Check with Q-Q plots.

5. No Multicollinearity

Features should not be highly correlated with each other. Use VIF (Variance Inflation Factor) to detect. VIF > 10 signals a problem.

6. No Endogeneity

Features should not be correlated with the error term. Omitting important variables causes this.

Multicollinearity and VIF

Problem: If x₁ (size in sq.ft.) and x₂ (size in m²) are both included, they are perfectly correlated. The model cannot tell which feature deserves credit for predictions — weights become unstable and unreliable.

Variance Inflation Factor (VIF):

VIF_j = 1 / (1 − R²_j)

where R²_j is the R² from regressing feature j on all other features.

VIF ValueInterpretationAction
1No multicollinearityFine
1 – 5Moderate correlationUsually acceptable
5 – 10High correlationInvestigate
> 10Severe multicollinearityDrop or combine features

Normal Equation & Worked Example

Deriving the Normal Equation

We want weights W that minimise the Sum of Squared Errors (SSE):

SSE = ||Y − XW||² = (Y − XW)ᵀ(Y − XW)

Expanding and taking the derivative with respect to W, then setting to zero:

1
Expand: SSE = YᵀY − 2WᵀXᵀY + WᵀXᵀXW
2
Differentiate: ∂SSE/∂W = −2XᵀY + 2XᵀXW
3
Set to zero: −2XᵀY + 2XᵀXW = 0
4
Solve for W: XᵀXW = XᵀY
W = (XᵀX)⁻¹ XᵀY
Key insight: The Normal Equation gives the exact analytical solution in one step — no iteration needed. However, it requires computing a matrix inverse, which is O(n³). For large feature spaces (>10,000 features), gradient descent is preferred.

Logistic Regression

Why Not Linear Regression for Classification?

Problem 1: Unbounded Output

Linear regression outputs any real number (−∞ to +∞). A probability must be between 0 and 1. You can get predictions like 1.8 or −0.3, which are meaningless probabilities.

Problem 2: Sensitive to Outliers

A single far-away point can tilt the regression line, shifting the decision boundary drastically even when the boundary should be clear.

Solution: Wrap the linear combination inside a function that maps any real number to (0, 1) — the sigmoid function.

The Sigmoid Function

σ(z) = 1 / (1 + e^{−z})

Key properties:

  • Output always in (0, 1) — valid probability range
  • σ(0) = 0.5 — the threshold point
  • σ(z) → 1 as z → +∞ (very positive input → near certain positive class)
  • σ(z) → 0 as z → −∞ (very negative input → near certain negative class)
  • Derivative: σ'(z) = σ(z)(1 − σ(z)) — elegant and efficient for backprop

The Logistic Regression Model

P(y=1 | x) = σ(z) = σ(w₀ + w₁x₁ + w₂x₂ + ... + wₙxₙ)

The decision boundary is where P(y=1|x) = 0.5, i.e., where z = 0:

w₀ + w₁x₁ + w₂x₂ = 0 → decision boundary (line in 2D, hyperplane in nD)

Loss Function: Cross-Entropy (Log-Loss)

We cannot use squared error for logistic regression — the loss surface becomes non-convex. Instead we use the log-likelihood, which is convex and has a unique minimum.

For a single sample:

L(y, p) = −[y · log(p) + (1−y) · log(1−p)]

Intuition:

  • When y=1: L = −log(p). If p is close to 1, loss ≈ 0. If p is close to 0, loss → ∞. Penalises missed positives heavily.
  • When y=0: L = −log(1−p). If p is close to 0, loss ≈ 0. If p is close to 1, loss → ∞. Penalises false positives heavily.

Total loss over N samples:

J(W) = −(1/N) Σᵢ [yᵢ·log(pᵢ) + (1−yᵢ)·log(1−pᵢ)]

Gradient Descent Update

Taking the gradient of J with respect to each weight wⱼ:

∂J/∂wⱼ = (1/N) Σᵢ (pᵢ − yᵢ) · xᵢⱼ

Remarkably, this looks identical to the gradient for linear regression — only the prediction function differs. The update rule is:

wⱼ ← wⱼ − α · (1/N) Σᵢ (pᵢ − yᵢ) · xᵢⱼ

where α is the learning rate.

One-vs-All (OvA) Multiclass Classification

Logistic regression is inherently a binary classifier — it outputs P(y=1|x). But many real problems have more than 2 classes (e.g., handwritten digit recognition: classes 0–9). One-vs-All (also called One-vs-Rest, OvR) is the standard extension strategy.

The Strategy

For a K-class problem, train K separate binary classifiers:

  • Classifier 1: "Is this class 1 vs. everything else?"
  • Classifier 2: "Is this class 2 vs. everything else?"
  • …
  • Classifier K: "Is this class K vs. everything else?"

At prediction time: run all K classifiers and pick the one with the highest probability.

ŷ = argmax_k P(y=k | x) for k = 1, 2, …, K

Visual: 3-Class OvA Setup

Practical Example: Digit Recognition

MNIST handwritten digit recognition (classes 0–9) requires training 10 binary classifiers:

  • Classifier₀: "Is this a 0 or not?"
  • Classifier₁: "Is this a 1 or not?"
  • … through Classifier₉: "Is this a 9 or not?"

For a new image, run all 10 classifiers. The digit with the highest probability wins.

Pros and Cons of OvA

Advantages

  • Simple and interpretable — reuses standard binary classifiers
  • Training is parallelisable (K classifiers are independent)
  • Works with any binary classifier (logistic regression, SVM, etc.)

Disadvantages

  • Class imbalance: each classifier has 1 positive class vs. K−1 negative classes
  • Probabilities from different classifiers are not directly comparable (not calibrated)
  • K times the training cost of a single classifier

Support Vector Machine (SVM)

Core Intuition: Maximum Margin

Multiple hyperplanes can separate two linearly separable classes. Which one is best? SVMs answer: the one that maximises the margin — the gap between the classes.

Key idea: A classifier with a larger margin is more robust. If a new point lands anywhere in the margin, a wide-margin classifier is less likely to misclassify it than a narrow-margin one. This is the foundation of statistical learning theory (Vapnik & Chervonenkis bounds).

SVM Diagram with Support Vectors

Hard-Margin SVM Formulation

For linearly separable data with binary labels yᵢ ∈ {−1, +1}:

Objective: Maximise margin = 2/||w||, which is equivalent to:

Minimise: ½||w||²

Subject to:

yᵢ(wᵀxᵢ + b) ≥ 1 for all i = 1, …, N

The constraint ensures every point is correctly classified AND at least 1 unit away from the decision boundary. Points exactly at distance 1 are the support vectors.

Decision Function

f(x) = sign(wᵀx + b)
  • If wᵀx + b > 0 → predict class +1
  • If wᵀx + b < 0 → predict class −1

Support Vectors — Why They Matter

Support vectors are the training points that lie exactly on the margin boundaries, satisfying yᵢ(wᵀxᵢ + b) = 1.

  • They are the only points that define the decision boundary — all other training points are irrelevant!
  • If you remove a non-support-vector point, the boundary stays exactly the same
  • This gives SVMs excellent robustness and memory efficiency (store only support vectors)
  • Typically only 2–5% of training points are support vectors

Why Maximise Margin? Generalisation Theory

From VC (Vapnik–Chervonenkis) dimension theory, the generalisation error (test error) is bounded by:

Test error ≤ Training error + O(√(VC_dim / N))

For a maximum-margin classifier, the effective VC dimension decreases as the margin increases. Larger margin → tighter bound on test error → better generalisation to unseen data.

Optimal boundary: The optimal w satisfying all constraints with maximum margin for this dataset is approximately x₁ + x₂ = 1, with support vectors at A=(1,1) and C=(0,0). The margin = 2/||w|| = 2/√2 ≈ 1.414.
