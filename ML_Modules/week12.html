1. Bagging (Bootstrap Aggregating)

Core idea: Train many independent models on different random subsets of the data, then combine their predictions. Averaging reduces variance without increasing bias.

The Bootstrap

A bootstrap sample is created by sampling N points from the training set with replacement. On average, each bootstrap sample contains about 63.2% unique points — the remaining ~36.8% are duplicates, and ~36.8% of original points are never selected (these become the OOB set).

P(point NOT selected in one draw) = (1 − 1/N)
P(point never selected in N draws) = (1 − 1/N)^N → e^{−1} ≈ 0.368 as N→∞

Bagging Algorithm

1
For b = 1 to B: draw bootstrap sample Z*_b (N samples with replacement)
2
Train a base model f̂_b on Z*_b
3
Aggregate:
Regression: F̂_bag(x) = (1/B) Σ_b f̂_b(x)
Classification: majority vote across all B predictions

Bagging Flow Diagram

Why Bagging Reduces Variance

Suppose we have B independent models each with variance σ². Their average has variance σ²/B. In practice the models are correlated (ρ), giving:

Var(average) = ρσ² + (1−ρ)σ²/B

As B → ∞ this approaches ρσ². So bagging works best when models are uncorrelated (ρ small) — which is why we use bootstrap sampling to introduce diversity.

Out-of-Bag (OOB) Error

Since each bootstrap sample leaves out ~37% of data, we can evaluate each model on its OOB samples for free — no separate validation set needed! OOB error is an unbiased estimate of generalization error.

For each training point x_i, collect predictions from all trees where x_i was not in the bootstrap sample. Average (or vote) these predictions → OOB prediction. Compare to true label → OOB error.

2. Random Forest

Random Forest = Bagging + Random Feature Subsets. At each split node, only a random subset of m features is considered. This decorrelates the trees further, reducing the ρ term in the variance formula.

Algorithm

1
For each tree b = 1 to B:
  a) Draw bootstrap sample Z*_b
  b) Grow tree: at each node, pick m random features, find best split among those m
  c) Grow fully (no pruning)
2
Predict by aggregating all trees (vote or average)

Feature Subset Size m

Classification

Default: m = √p (where p = total features)

E.g. with 100 features, consider only 10 at each split

Regression

Default: m = p/3

E.g. with 30 features, consider 10 at each split

Random Forest — Feature Subsets per Node

Feature Importance

For each feature j, sum how much it reduces impurity (Gini or entropy) across all splits in all trees, weighted by the number of samples passing through each split. Normalize to sum to 1. Features that consistently create clean splits get high importance scores.

Importance(j) = (1/B) Σ_b Σ_{nodes t in tree b where feature j is used} p_t × ΔImpurity_t

Key Hyperparameters

ParameterDefaultEffect
n_estimators100More trees = lower variance, diminishing returns past ~200
max_features√p / p/3Smaller = more decorrelated trees, but higher individual bias
max_depthNone (full)Deeper = lower bias, higher variance per tree
min_samples_split2Higher = smoother trees, prevents overfitting on small nodes
bootstrapTrueFalse = use full data (pasting), less variance reduction

3. Boosting Overview

Core contrast with bagging: Bagging trains models in parallel on random subsets. Boosting trains models sequentially, where each new model corrects the mistakes of all previous models.

Original Boosting — Schapire (1990)

Schapire proved that a "weak learner" (slightly better than chance) can be boosted into a "strong learner" (arbitrarily accurate).

1
Train M1 on original training data
2
Train M2 on a filtered set where M1 fails (half correct, half incorrect)
3
Train M3 on cases where M1 and M2 disagree
4
Final prediction: combine M1, M2, M3 by majority vote

General Boosting Mechanism

Bagging vs. Boosting

Bagging: Parallel, reduces variance. Each model sees a different random subset. Best when individual models overfit (high variance). Random Forest is the prime example.
Boosting: Sequential, reduces bias. Each model corrects its predecessor. Best when individual models are too simple (high bias). AdaBoost, Gradient Boosting, XGBoost.

4. AdaBoost (Adaptive Boosting)

Freund & Schapire, 1996. AdaBoost maintains a weight distribution over training samples. Misclassified samples get higher weights so the next classifier focuses on them. The final classifier is a weighted vote of all weak classifiers.

Algorithm (Binary Classification, labels ∈ {−1, +1})

1
Initialize sample weights: w_i = 1/N for all i = 1…N
2
For t = 1 to T:
  1. Train weak classifier f_t on the weighted training data
  2. Compute weighted error: ε_t = Σ_i w_i × 𝟏[y_i ≠ f_t(x_i)]
  3. Compute classifier weight: W_t = ½ ln((1 − ε_t) / ε_t)
  4. Update sample weights:
    • Correct (y_i = f_t(x_i)): w_i ← w_i × e^{−W_t}
    • Misclassified (y_i ≠ f_t(x_i)): w_i ← w_i × e^{+W_t}
  5. Normalize: w_i ← w_i / Σ_j w_j
3
Final prediction: ŷ = sign(Σ_t W_t × f_t(x))
W_t = ½ ln((1 − ε_t) / ε_t)  ·  ŷ = sign(Σ_t W_t f_t(x))
Intuition for W_t: If ε_t = 0.5 (random), W_t = 0 (ignore this classifier). If ε_t → 0 (perfect), W_t → ∞ (trust it completely). If ε_t → 1 (always wrong), W_t → −∞ (invert its vote).

5. Gradient Boosting

Key idea: Instead of re-weighting samples, fit each new model to the negative gradient of the loss. For MSE loss, this equals the residuals. We are doing gradient descent in function space.

General Algorithm

1
Initialize: f₀(x) = argmin_γ Σᵢ L(yᵢ, γ)  (for MSE: f₀ = mean(y))
2
For t = 1 to T:
  1. Compute pseudo-residuals: r_{it} = −∂L(yᵢ, f(xᵢ))/∂f(xᵢ) evaluated at f = f_{t-1}
  2. Fit a weak learner h_t (e.g., shallow tree) to {(xᵢ, r_{it})}
  3. Update: f_t(x) = f_{t-1}(x) + η × h_t(x)
3
Final model: F(x) = f₀(x) + η × Σ_{t=1}^{T} h_t(x)
For MSE loss: r_{it} = yᵢ − f_{t-1}(xᵢ)  (literal residuals)

6. XGBoost

"Extreme Gradient Boosting" — Chen & Guestrin, 2016. XGBoost supercharges gradient boosting with a regularized objective, second-order Taylor expansion for the loss, and engineering optimizations that make it blazing fast.

Regularized Objective

Standard gradient boosting minimizes the loss. XGBoost adds a regularization term to prevent overfitting:

L(θ) = Σᵢ l(yᵢ, ŷᵢ) + Σₖ Ω(fₖ)
Ω(f) = γT + ½λ Σⱼ wⱼ²

Where T = number of leaves, wⱼ = weight of leaf j, γ = minimum gain to make a split, λ = L2 regularization on leaf weights.

Second-Order Taylor Expansion

XGBoost uses a second-order approximation of the loss to build trees more efficiently:

L ≈ Σᵢ [gᵢ fₜ(xᵢ) + ½ hᵢ fₜ(xᵢ)²] + Ω(fₜ)

Where gᵢ = ∂l(yᵢ,ŷᵢ)/∂ŷᵢ (first derivative), hᵢ = ∂²l(yᵢ,ŷᵢ)/∂ŷᵢ² (second derivative / Hessian).

Optimal leaf weight: w*ⱼ = −(Σᵢ∈Iⱼ gᵢ) / (Σᵢ∈Iⱼ hᵢ + λ)

Gain = ½ [ G_L² / (H_L+λ) + G_R² / (H_R+λ) − (G_L+G_R)² / (H_L+H_R+λ) ] − γ

Key Improvements Over Vanilla Gradient Boosting

Regularization

L1 (α) and L2 (λ) on leaf weights + minimum gain threshold γ to prune splits. Controls model complexity directly.

Missing Data

Learns a default split direction for missing values. For each split, tries both directions on missing values and picks the better one.

Approximate Algorithm

Instead of checking all possible split points, uses histogram-based quantile sketches to find approximate best splits in O(data) time.

Parallel Tree Construction

Split finding at each level is parallelized across features. Column sub-sampling (like RF) reduces correlation between trees.

Cache Awareness

Pre-sorts data and uses cache-friendly memory access patterns. Compressed block format for distributed computation.

Early Stopping

Monitor validation loss; stop adding trees when performance stops improving. Prevents overfitting automatically.

7. Stacking (Stacked Generalization)

Stacking learns HOW to combine models. Rather than using a fixed rule (vote/average), a meta-model is trained to optimally weight the predictions of base models. Base models should be diverse (different algorithms).

Algorithm

1
Partition training data into K folds (typically K = 5 or 10)
2
For each fold k = 1…K:
  • Train each base model on the K−1 folds (not fold k)
  • Generate predictions on the held-out fold k → out-of-fold (OOF) predictions
3
Stack OOF predictions as new features → meta-features matrix (N × M, where M = number of base models)
4
Train meta-model on meta-features (with original labels)
5
For test data: each base model predicts on all test data (trained on full training set), feed to meta-model for final prediction

Stacking Architecture

Why K-fold CV Prevents Leakage

If we just trained base models on ALL training data and then trained the meta-model on those predictions, the meta-model would see predictions made on data the base models were trained on (overfitted predictions). K-fold CV ensures each OOF prediction is made by a model that has never seen that sample — honest predictions!

Blending (Simpler Alternative)

Stacking (K-fold CV)

  • Uses all training data
  • More complex to implement
  • Less data wasted
  • Less prone to leakage (when done right)
  • Better for smaller datasets

Blending (Holdout)

  • Split: 70% train, 20% blend, 10% test
  • Train base models on 70%
  • Predict on 20% → meta-features
  • Simpler, less leak risk
  • Wastes 20% for training

Practical Tips for Stacking

  • Diversity matters: Different algorithm families (trees, linear, NN) work better than similar models
  • Simple meta-model: Logistic regression or linear regression often works best at level 1 (prevents over-fitting to base model predictions)
  • Include raw features: Sometimes passing original features to the meta-model alongside base predictions helps
  • Multi-level stacking: Can stack 3+ levels but usually has diminishing returns and high complexity

Decision Guide: Which Ensemble to Use?

Bias-Variance Perspective

Bagging / Random Forest

Bias: unchanged (same as individual tree)
Variance: ↓↓ (divides by B, then ρ)

Use when: individual models overfit

Boosting

Bias: ↓↓ (each iteration reduces error)
Variance: can ↑ if too many rounds

Use when: individual models underfit

Stacking

Bias: ↓ (meta-model corrects)
Variance: ↓ (diverse base models)

Use when: squeeze every % accuracy
