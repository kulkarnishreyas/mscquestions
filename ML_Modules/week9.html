❓ Why Multi-Layer Networks?

A single-layer perceptron can only learn linearly separable problems. Many real-world problems are inherently non-linear. Multi-layer networks unlock the ability to learn complex, non-linear decision boundaries.

The XOR Problem — Single Layer Fails

XOR returns 1 when inputs differ, 0 when they are the same. There is no single straight line that separates the two classes.

Universal Approximation Theorem

Theorem (Cybenko, 1989): A feedforward network with a single hidden layer containing a sufficient number of neurons can approximate any continuous function on a compact subset of Rn to arbitrary precision — given a suitable activation function.

In plain terms: a 2-layer network (1 hidden + 1 output) with enough neurons can learn any function. This is why neural networks are called "universal function approximators".

Implication 1: More neurons = more complex functions can be represented. But we still need backpropagation to find the right weights!
Implication 2: Depth (more layers) is often more efficient than width (one very wide layer). Each layer learns a different level of abstraction.

⚙ The Backpropagation Algorithm

Backpropagation (Rumelhart, Hinton & Williams, 1986) is the algorithm that made deep learning possible. It efficiently computes the gradient of the loss with respect to every weight in the network using the chain rule of calculus.

Key Insight: To train a neural network we need to know how much each weight contributed to the error. Backprop computes this for all weights simultaneously in O(W) time, where W is the total number of weights — far more efficient than finite-difference approximations.

Two-Phase Algorithm

Phase 1: Forward Pass

Feed the input forward through each layer, computing activations. Store all intermediate values (needed for backward pass).

1
Input x enters the network
2
Compute z = W·a + b at each layer
3
Apply activation: a = f(z)
4
Compute loss L(ŷ, y) at output

Phase 2: Backward Pass

Propagate the error signal backward, computing gradients layer by layer using the chain rule.

1
Compute output error δ(L)
2
Propagate δ back to each hidden layer
3
Compute ∂L/∂W and ∂L/∂b
4
Update weights: W ← W − η∇W

The Chain Rule — Heart of Backprop

The chain rule allows us to decompose complex derivatives into a product of simpler ones:

∂L/∂w = (∂L/∂a) × (∂a/∂z) × (∂z/∂w)

Where:

  • ∂L/∂a — how the loss changes with activation (from the layer above)
  • ∂a/∂z — derivative of the activation function f'(z)
  • ∂z/∂w — how the pre-activation changes with the weight = a(l-1)
Why "backpropagation"? The gradient signal flows backward through the network — from the output layer to the input layer. Each layer receives the gradient from the layer above it and passes a modified gradient to the layer below.

➡ Forward Pass — Detailed

During the forward pass we compute the output of every neuron from the first hidden layer to the output. All values are stored because the backward pass needs them.

Notation

  • L — total number of layers (input layer is layer 0)
  • a(l) — activation (output) of layer l
  • z(l) — pre-activation (weighted sum) of layer l
  • W(l) — weight matrix connecting layer l−1 to layer l
  • b(l) — bias vector for layer l
  • f(·) — activation function (sigmoid, ReLU, tanh, …)

Layer-by-Layer Computation

z(l) = W(l) · a(l−1) + b(l)
a(l) = f( z(l) )

where a(0) = x (the input).

Matrix dimensions for a [n₀, n₁, n₂] network:
  • W(1) is n₁ × n₀ — maps n₀ inputs to n₁ hidden neurons
  • W(2) is n₂ × n₁ — maps n₁ hidden to n₂ outputs
  • z(1), a(1) are n₁ × 1 column vectors

Sigmoid Activation Function

σ(z) = 1 / (1 + e−z)
σ'(z) = σ(z) · (1 − σ(z))

The sigmoid maps any real number to (0, 1), making it useful for probabilities and binary outputs. Its derivative has a convenient form that reuses the activation value.

Step-by-Step Forward Pass (3→2→1 Network)

← Backward Pass — Gradient Computation

The backward pass computes how much each weight contributed to the error, then updates the weights to reduce that error.

Error at the Output Layer (δ(L))

The delta (δ) at the output neuron combines the prediction error with the derivative of the activation function:

δ(L) = (a(L) − y) × f'(z(L))

For sigmoid activation and squared error loss, this simplifies to:

δ(L) = a(L) × (1 − a(L)) × (y − a(L))

Note: The sign depends on loss definition. Using (y − ŷ) makes δ positive when network under-predicts.

Propagating Error to Hidden Layers

δ(l) = (W(l+1)T · δ(l+1)) × f'(z(l))

This says: the error at hidden layer l is the weighted sum of errors from the next layer (weighted by the connecting weights), scaled by the activation derivative.

Weight and Bias Updates

W(l) ← W(l) − η × δ(l) × (a(l−1))T
b(l) ← b(l) − η × δ(l)

Where η (eta) is the learning rate — controls how large each update step is.

Summary of Backprop Equations

QuantityFormulaMeaning
Output deltaδL = aL(1−aL)(y−aL)Error signal at output
Hidden deltaδl = (Wl+1Tδl+1) ⊙ f'(zl)Backpropagated error
Weight gradient∂L/∂Wl = δl · (al−1)THow much to change each weight
Weight updateWl ← Wl − η·∂L/∂WlGradient descent step
Bias updatebl ← bl − η·δlBias correction
Intuition: δ(l) measures "how much is neuron l responsible for the error?" The weight update is then proportional to both this responsibility (δ) and how much the upstream neuron fired (a(l-1)). A weight connecting two highly active neurons gets a larger update.

Using rule: Δw = η × δ × asource

Δw₆₄ = η × δ₆ × a₄ = 0.9 × 0.1311 × 0.332 = 0.03916

w₆₄_new = 0.7 + 0.03916 = 0.73916

🌐 Types of Deep Neural Networks

Different architectures are designed for different types of data and tasks. Each introduces structural inductive biases that make learning more efficient.

1. Feedforward Neural Network (FNN / MLP)

The standard multi-layer perceptron. Information flows only forward — no cycles. Every neuron in one layer connects to every neuron in the next (fully connected).

Use cases: Classification, regression, tabular data.

2. Convolutional Neural Network (CNN)

Key idea: Local connectivity + weight sharing. A small filter (kernel) slides across the input, detecting local patterns. The same filter weights are reused everywhere — this dramatically reduces parameters.

Use cases: Image classification, object detection, medical imaging, any grid-structured data.

  • Convolutional layers: Learn local feature detectors (edges, textures, shapes)
  • Pooling layers: Spatial downsampling, translation invariance
  • Fully connected layers: Classification at the end

3. Recurrent Neural Network (RNN)

Key idea: Hidden state is passed from one time step to the next — the network has "memory" of previous inputs. Same weights are used at each time step (weight sharing over time).

Use cases: Text classification, language modelling, time series, speech recognition.

4. Long Short-Term Memory (LSTM)

Problem with RNNs: Vanishing gradients prevent learning long-range dependencies. LSTM adds a cell state (long-term memory) with three gates that control information flow.

  • Forget gate: Decides what to erase from cell state — σ(W_f · [h_{t-1}, x_t])
  • Input gate: Decides what new info to store — σ(W_i · [h_{t-1}, x_t])
  • Output gate: Decides what to output — σ(W_o · [h_{t-1}, x_t])

Use cases: Machine translation, speech recognition, time series forecasting.

5. GRU (Gated Recurrent Unit)

Simplified LSTM with only two gates: reset and update. Fewer parameters, faster training, comparable performance on many tasks.

Gates:

  • Update gate: combine past and new info
  • Reset gate: how much past to forget

No separate cell state — simpler than LSTM.

6. Autoencoder

Unsupervised architecture: encoder compresses input to a latent code; decoder reconstructs original from the code.

x → [Encoder] → z → [Decoder] → x̂

Uses: Dimensionality reduction, denoising, anomaly detection, generative modelling.

The bottleneck forces the network to learn a compact representation.

7. GAN (Generative Adversarial Network)

Two networks compete: a Generator creates fake data, a Discriminator tries to distinguish real from fake.

G(noise) → fake samples ↕ Discriminator judges

Uses: Image synthesis, data augmentation, style transfer, deepfakes.

Training is adversarial — G and D improve together.

⚠️ DNN Training Challenges

Deep networks are powerful but notoriously difficult to train. Understanding these challenges is essential for building working models.

Vanishing Gradients

Problem: In deep networks, gradients shrink as they propagate backward. With sigmoid/tanh (derivatives ≤ 0.25), multiplying many layers together drives gradients toward zero.

Effect: Early layers learn extremely slowly or not at all. Deep networks stall.

|∂L/∂W₁| = |δ^L| × |σ'(z_k)|^L → 0

Solutions:

  • ReLU activation: derivative is 1 (no squashing)
  • Batch normalization: normalizes layer inputs
  • Residual connections (skip connections)
  • Careful weight initialization (Xavier/He)

Exploding Gradients

Problem: Gradients grow exponentially in very deep networks or when weights are large. Updates become enormous, destabilizing training.

Effect: Loss diverges to NaN or oscillates wildly.

||g_t|| → ∞ as depth increases

Solutions:

  • Gradient clipping: cap ||∇|| to threshold
  • Smaller learning rate
  • Batch normalization
  • Weight regularization

Overfitting

Problem: A large network memorizes training data, including noise. Performance on test data is poor.

Effect: Train accuracy high, test accuracy low. Large generalization gap.

Solutions:

  • Dropout: randomly zero out neurons during training (p=0.5 common)
  • L2 regularization: penalize large weights in loss
  • L1 regularization: encourages sparsity
  • Early stopping: stop when val loss stops improving
  • Data augmentation: flips, crops, rotations to expand dataset
  • Batch normalization: mild regularization effect

Computational Complexity

Problem: Modern networks have millions–billions of parameters. Training on large datasets is extremely slow on CPUs.

Effect: Training takes days to months without acceleration.

Solutions:

  • GPU/TPU training: massive parallel computation
  • Mini-batch SGD: update on 32–256 samples at a time
  • Batch normalization: speeds up convergence
  • Mixed precision: float16 instead of float32
  • Distributed training: across multiple machines

⚡ Optimization Algorithms

How we update weights during training has a big impact on speed of convergence and quality of the final model.

Vanilla SGD

Stochastic Gradient Descent — update weights using the gradient on a single (or small batch of) sample(s).

w ← w − η · ∇L(w)

Pros: Simple, memory efficient, can escape local minima.

Cons: Noisy updates, sensitive to learning rate, slow convergence.

Mini-batch SGD

Compromise between full-batch (stable but slow) and single-sample (noisy but fast). Typical batch size: 32–256.

w ← w − η · (1/B) Σ ∇L_i(w)

Momentum

Adds a velocity term that accumulates past gradients, smoothing out oscillations.

v ← μ·v − η·∇L(w)
w ← w + v

μ (momentum coefficient) typically 0.9. Like a ball rolling down a hill — it speeds up in consistent directions.

Adam (Adaptive Moment Estimation)

Most popular optimizer. Combines Momentum (first moment) with RMSProp (second moment). Adapts learning rate per parameter.

m ← β₁·m + (1−β₁)·g
v ← β₂·v + (1−β₂)·g²
w ← w − η·m̂ / (√v̂ + ε)

Default: β₁=0.9, β₂=0.999, ε=10⁻⁸

Learning Rate Schedules

A fixed learning rate is often suboptimal. Common schedules:

  • Step decay: halve η every N epochs
  • Exponential decay: η_t = η₀ × e−kt
  • Cosine annealing: η oscillates between max and min — good for escaping local minima
  • Warmup + decay: start small, ramp up, then decay — used in Transformers
  • Reduce on plateau: automatically reduce η when val loss stops improving
