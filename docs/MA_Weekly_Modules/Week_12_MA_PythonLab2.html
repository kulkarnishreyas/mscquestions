1. Seaborn Visualisation Library

What is Seaborn?

Seaborn is a Python data visualisation library built on top of Matplotlib. It provides a high-level interface for drawing attractive and informative statistical graphics. Where Matplotlib requires explicit configuration, Seaborn offers sensible defaults and tightly integrated pandas DataFrame support.

Key Advantages

  • Built-in themes and colour palettes
  • Direct DataFrame support (data=df)
  • Statistical aggregation built-in (CI, KDE)
  • Figure-level and axes-level interfaces
  • Faceting / multi-plot grids with one call

Seaborn vs Matplotlib

2. Z-Score Outlier Detection

Theory

A Z-score (also called a standard score) measures how many standard deviations a data point is from the mean of its distribution. It standardises values to a common scale, making it easy to identify extreme observations.

Z = (X − μ) / σ
where: X = observed value, μ = population mean, σ = standard deviation

Empirical Rule (68-95-99.7)

  • 68% of data falls within ±1σ
  • 95% of data falls within ±2σ
  • 99.7% of data falls within ±3σ

Outlier Thresholds

  • Moderate outlier: |Z| > 2 (outside 95% band)
  • Extreme outlier: |Z| > 3 (outside 99.7% band)
  • Assumption: data is approximately normally distributed

Worked Example — Marketing Campaign Spend

A marketing analyst notices unusually high spend in one campaign. Z-score analysis reveals the outlier objectively.

import numpy as np
import pandas as pd

# Campaign spend data (£)
spend = np.array([1200, 1500, 1100, 1800, 1300, 45000, 1400, 1600, 900, 1250])

# Step 1: Calculate mean and standard deviation
mean = np.mean(spend)      # 5705.0
std  = np.std(spend)       # 13127.4  (large due to outlier!)

print(f"Mean:  £{mean:,.0f}")
print(f"Std:   £{std:,.0f}")

# Step 2: Compute Z-scores for each value
z_scores = (spend - mean) / std
print("\nZ-scores:")
for s, z in zip(spend, z_scores):
    flag = " ← OUTLIER" if abs(z) > 2 else ""
    print(f"  £{s:,6d}  →  Z = {z:+.2f}{flag}")

# Output:
# £  1,200  →  Z = -0.34
# £  1,500  →  Z = -0.32
# £  1,100  →  Z = -0.35
# £  1,800  →  Z = -0.29
# £  1,300  →  Z = -0.34
# £ 45,000  →  Z = +2.99 ← OUTLIER
# £  1,400  →  Z = -0.33
# £  1,600  →  Z = -0.31
# £    900  →  Z = -0.37
# £  1,250  →  Z = -0.34

# Step 3: Identify outliers at different thresholds
outliers_2 = spend[np.abs(z_scores) > 2]
outliers_3 = spend[np.abs(z_scores) > 3]
print(f"\nModerate outliers (|Z| > 2): {outliers_2}")   # [45000]
print(f"Extreme outliers  (|Z| > 3): {outliers_3}")    # [] — just below 3!

# Step 4: Remove and re-analyse
clean_spend = spend[np.abs(z_scores) <= 2]
print(f"\nClean mean: £{np.mean(clean_spend):,.0f}")    # £1,344
print(f"Clean std:  £{np.std(clean_spend):,.0f}")      # £270
Important Limitation: The mean and std in Step 1 are themselves distorted by the outlier (£45,000 inflates the mean). This is the "masking effect" — extreme outliers can shift Z-scores so that other mild outliers appear normal. Consider robust Z-scores using median and MAD (Median Absolute Deviation) for very skewed data.

Handling Outliers: Four Strategies

import numpy as np
from scipy.stats import mstats

spend = np.array([1200, 1500, 1100, 1800, 1300, 45000, 1400, 1600, 900, 1250])

# ── Strategy 1: Remove (delete if measurement/entry error) ────────────────────
clean = spend[spend < 3000]   # domain knowledge threshold
print("After removal:", clean)

# ── Strategy 2: Winsorise / Cap (replace with percentile boundary) ────────────
# scipy.stats.mstats.winsorize clamps top/bottom proportions
winsorised = mstats.winsorize(spend, limits=[0.05, 0.05])
# 5% from each tail is replaced by the 5th/95th percentile value
print("Winsorised:", np.array(winsorised))

# Alternatively, manual percentile capping:
lower_cap = np.percentile(spend, 5)
upper_cap = np.percentile(spend, 95)
capped = np.clip(spend, lower_cap, upper_cap)
print("Capped:", capped)

# ── Strategy 3: Log Transform (reduce skew caused by large values) ────────────
import numpy as np
log_spend = np.log1p(spend)   # log1p = log(1+x), handles zeros
print("Log-transformed:", log_spend.round(2))

# ── Strategy 4: Keep (if genuine extreme value adds insight) ──────────────────
# e.g., a Black Friday campaign legitimately spent £45,000
# Label it: spend_data['is_outlier'] = np.abs(z_scores) > 2
# Then analyse outliers separately

3. IQR Outlier Detection

Theory

The Interquartile Range (IQR) method is a non-parametric approach to outlier detection. It does not assume normality, making it more robust for skewed marketing data (e.g., revenue, CLV, session duration).

IQR = Q3 − Q1
Lower Bound = Q1 − 1.5 × IQR
Upper Bound = Q3 + 1.5 × IQR
Extreme outliers use multiplier 3.0 instead of 1.5

This is exactly the logic used by boxplots — the whiskers extend to the bounds, and points beyond are plotted individually as outlier markers.

Worked Example — Customer CLV Data

import numpy as np

# Customer Lifetime Value data (£)
clv = np.array([1200, 1500, 1100, 1800, 1300, 12000, 1400, 1600, 900, 800, 1250, 1700])

# Step 1: Calculate quartiles and IQR
Q1  = np.percentile(clv, 25)   # Q1 = 1137.5
Q3  = np.percentile(clv, 75)   # Q3 = 1625.0
IQR = Q3 - Q1                  # IQR = 487.5

print(f"Q1  = £{Q1:.1f}")
print(f"Q3  = £{Q3:.1f}")
print(f"IQR = £{IQR:.1f}")

# Step 2: Calculate bounds
lower = Q1 - 1.5 * IQR         # lower = 1137.5 - 731.25 = 406.25
upper = Q3 + 1.5 * IQR         # upper = 1625.0 + 731.25 = 2356.25

print(f"\nLower bound: £{lower:.2f}")
print(f"Upper bound: £{upper:.2f}")

# Step 3: Flag outliers
outlier_mask = (clv < lower) | (clv > upper)
outliers     = clv[outlier_mask]
clean_data   = clv[~outlier_mask]

print(f"\nOutliers detected: {outliers}")          # [12000]
print(f"Clean dataset:     {clean_data}")
print(f"Outlier count: {len(outliers)} of {len(clv)} records ({100*len(outliers)/len(clv):.1f}%)")

# Step 4: Visualise with seaborn
import seaborn as sns
import matplotlib.pyplot as plt

fig, axes = plt.subplots(1, 2, figsize=(12, 4))
sns.boxplot(y=clv, ax=axes[0], color='#4db6ac')
axes[0].set_title('CLV with Outlier')
axes[0].set_ylabel('CLV (£)')

sns.boxplot(y=clean_data, ax=axes[1], color='#80cbc4')
axes[1].set_title('CLV after IQR Removal')
axes[1].set_ylabel('CLV (£)')
plt.tight_layout()
plt.show()

Threshold|Z| > 2 (moderate) or |Z| > 3 (extreme)1.5 × IQR (mild) or 3 × IQR (extreme)

Exam Tip: Revenue and CLV distributions in marketing are almost always right-skewed (most customers spend little, a few spend a lot). Always prefer IQR over Z-score for these variables. Z-score is appropriate when the data is symmetric and bell-shaped.

4. Missing Data — Types & Classification

Before choosing an imputation strategy, you must understand why data is missing. Rubin (1976) identified three mechanisms that have different implications for analysis bias:

MCAR — Missing Completely At Random

The probability that a value is missing is unrelated to any variable in the dataset — observed or unobserved. The missingness is random noise.

  • Example: A survey server glitch randomly dropped 5% of responses
  • Example: A temperature sensor malfunctioned randomly during data collection
  • Implication: Safe to delete missing rows — produces minimal bias (complete-case analysis is valid)
  • Test: Little's MCAR test (statsmodels) — p > 0.05 suggests MCAR
  • Best strategy: Listwise deletion or simple imputation (mean/median)
Safest to drop

MAR — Missing At Random

The probability that a value is missing depends on other observed variables, but not on the missing value itself.

  • Example: Older customers are less likely to complete a digital income survey — age is observed, income is missing
  • Example: Male respondents skip questions about skincare spending more often — gender predicts missingness
  • Implication: Dropping will cause systematic bias — must impute using related variables
  • Best strategy: Multiple imputation, KNN imputation using the predictor variables, or model-based imputation
Must impute using covariates

MNAR — Missing Not At Random

The probability that a value is missing depends on the missing value itself — the most problematic mechanism.

  • Example: High-income customers deliberately skip the income question (high values cause their own missingness)
  • Example: Customers who churned don't respond to satisfaction surveys
  • Example: Extreme purchase values are underreported in customer panels
  • Implication: Standard imputation introduces severe bias — no statistical fix exists
  • Best strategy: Domain knowledge, sensitivity analysis, or explicitly model the missingness mechanism
Most problematic — requires domain knowledge

Detecting Missing Data in Python

import pandas as pd
import numpy as np

# Check for missing values
df = pd.read_csv('customer_data.csv')

# Count missings per column
print(df.isnull().sum())
# age         12
# spend        5
# channel      2
# income      45   ← high missingness — investigate why

# Percentage missing
print((df.isnull().sum() / len(df) * 100).round(2))

# Visual missing map (manual for markdown output)
missing_summary = pd.DataFrame({
    'column':   df.columns,
    'missing':  df.isnull().sum().values,
    'pct':      (df.isnull().sum().values / len(df) * 100).round(1),
    'dtype':    df.dtypes.values
}).sort_values('pct', ascending=False)
print(missing_summary)

# Investigate MAR: does age predict income missingness?
df['income_missing'] = df['income'].isnull().astype(int)
print(df.groupby('income_missing')['age'].mean())
# income_missing
# 0    32.1   ← income present:  mean age 32
# 1    58.4   ← income missing:  mean age 58 → MAR!

5. Missing Value Imputation Methods

Once the missingness mechanism is understood, you can select the appropriate imputation strategy. The goal is to replace missing values with plausible estimates that preserve the statistical properties of the data.

import pandas as pd
import numpy as np
from sklearn.impute import SimpleImputer, KNNImputer

# Sample dataset with mixed types
df = pd.DataFrame({
    'age':     [25, np.nan, 35,      28, np.nan, 42],
    'spend':   [5000, 8000, np.nan, 3200, 15000, np.nan],
    'channel': ['Online', 'Store', 'Online', np.nan, 'Online', 'Store']
})

print("Dataset with missing values:")
print(df)
print("\nMissing counts:", df.isnull().sum().to_dict())

Method 1 — Mean Imputation (Numeric, MCAR/MAR)

Replace each missing value with the arithmetic mean of the non-missing values. Simple and fast, but reduces variance and distorts the distribution shape.

from sklearn.impute import SimpleImputer

imp_mean = SimpleImputer(strategy='mean')

# Fit on non-missing, transform (fill) missing
df['spend_imputed'] = imp_mean.fit_transform(df[['spend']])

# Manual calculation:
# mean(5000, 8000, 3200, 15000) = 7800.0
# Both NaNs in 'spend' are replaced with 7800.0

print(df[['spend', 'spend_imputed']])
#    spend  spend_imputed
# 0   5000           5000
# 1   8000           8000
# 2    NaN           7800  ← imputed
# 3   3200           3200
# 4  15000          15000
# 5    NaN           7800  ← imputed

# Limitation: both fill values are identical — reduces variance
# Original variance ≠ imputed variance

Method 2 — Median Imputation (Skewed Numeric Data)

Use the median instead of the mean. More robust when data is skewed or contains outliers, because the median is not influenced by extreme values.

imp_median = SimpleImputer(strategy='median')

df['age_imputed'] = imp_median.fit_transform(df[['age']])
# Non-missing ages: [25, 35, 28, 42]
# Sorted: [25, 28, 35, 42]
# Median = (28 + 35) / 2 = 31.5
# Both NaN ages → 31.5

print(df[['age', 'age_imputed']])
#    age  age_imputed
# 0  25.0         25.0
# 1   NaN         31.5  ← imputed with median
# 2  35.0         35.0
# 3  28.0         28.0
# 4   NaN         31.5  ← imputed with median
# 5  42.0         42.0

# Guideline: if abs(skewness) > 1, prefer median over mean
print(f"Spend skewness: {df['spend'].skew():.2f}")

Method 3 — Mode Imputation (Categorical Variables)

For categorical columns, the mean/median are undefined. Replace missing values with the most frequent category.

imp_mode = SimpleImputer(strategy='most_frequent')

df['channel_imputed'] = imp_mode.fit_transform(df[['channel']]).ravel()
# Frequency count: Online=3, Store=2
# Mode = 'Online'  → NaN → 'Online'

print(df[['channel', 'channel_imputed']])
#   channel channel_imputed
# 0  Online          Online
# 1   Store           Store
# 2  Online          Online
# 3    NaN           Online  ← imputed with mode
# 4  Online          Online
# 5   Store           Store

# Limitation: can over-represent the dominant category
# and mask genuine uncertainty in the data

Method 4 — KNN Imputation (Use Similar Records)

KNN imputation finds the K most similar (nearest) records using all available columns, then fills the missing value using the average (or mode for categorical) of those K neighbours. Most accurate for structured tabular data with correlated features.

from sklearn.impute import KNNImputer

# KNN works on numeric features only
df_numeric = df[['age', 'spend']].copy()
print("Before KNN imputation:\n", df_numeric)

imp_knn = KNNImputer(n_neighbors=3)
df_knn = pd.DataFrame(
    imp_knn.fit_transform(df_numeric),
    columns=['age', 'spend']
)
print("\nAfter KNN imputation (n_neighbors=3):\n", df_knn.round(1))

# How it works for row 1 (age=NaN, spend=8000):
# - Finds 3 nearest neighbours based on 'spend' (since age is missing)
# - Neighbours: rows with spend closest to 8000 → rows 0,3,4
# - Imputed age = mean of their ages = mean(25, 28, NaN→skipped) ≈ ...
# - More neighbours → smoother, less variance; fewer → noisier, more local

# Choice of k:
# k=3 → local, preserves patterns, but noisy on small data
# k=5 → good default for most datasets
# k=10+ → smooth, may over-average

Method 5 — Forward Fill / Backward Fill (Time Series)

Hot-deck imputation copies values from neighbouring records. In time series, ffill copies the last known value forward; bfill copies the next known value backward.

import pandas as pd
import numpy as np

# Daily sales time series with gaps
ts = pd.Series([100, np.nan, np.nan, 400, np.nan, 600],
               index=pd.date_range('2024-01-01', periods=6, freq='D'))

print("Original:\n", ts)

# Forward fill: carry last observation forward
ts_ffill = ts.ffill()
print("\nForward fill (ffill):\n", ts_ffill)
# 2024-01-01    100
# 2024-01-02    100  ← copied from Jan 1
# 2024-01-03    100  ← copied from Jan 1
# 2024-01-04    400
# 2024-01-05    400  ← copied from Jan 4
# 2024-01-06    600

# Backward fill: use next known value
ts_bfill = ts.bfill()
print("\nBackward fill (bfill):\n", ts_bfill)
# 2024-01-01    100
# 2024-01-02    400  ← next known is Jan 4
# 2024-01-03    400  ← next known is Jan 4
# 2024-01-04    400
# 2024-01-05    600
# 2024-01-06    600

Method 6 — Interpolation (Time Series)

ts = pd.Series([100, np.nan, np.nan, 400, np.nan, 600])

# Linear interpolation: evenly spaced fill between known values
ts_linear = ts.interpolate(method='linear')
print("Linear:", ts_linear.tolist())
# [100.0, 200.0, 300.0, 400.0, 500.0, 600.0]
# Fills evenly: 100 → 400 over 3 steps = +100 per step

# Spline interpolation: smooth curve fit
ts_spline = ts.interpolate(method='spline', order=2)
print("Spline (order=2):", ts_spline.round(1).tolist())

# Time-aware interpolation (respects actual datetime gaps)
ts_indexed = pd.Series([100, np.nan, np.nan, 400, np.nan, 600],
                       index=pd.to_datetime(['2024-01-01','2024-01-03',
                                             '2024-01-04','2024-01-07',
                                             '2024-01-08','2024-01-10']))
ts_time = ts_indexed.interpolate(method='time')
print("\nTime-aware interpolation:\n", ts_time.round(1))

9. Summary, Exam Tips & Course Completion

Week 12 Key Takeaways

Seaborn Plots

  • Distribution: histplot, jointplot, pairplot
  • Categorical: boxplot, barplot, stripplot, swarmplot, countplot, pointplot
  • Regression: lmplot, residplot
  • Matrix: heatmap
  • Figure-level: catplot, displot

Outlier Detection

  • Z-score: assumes normality; |Z|>2 moderate, |Z|>3 extreme
  • IQR: non-parametric; Q1−1.5×IQR to Q3+1.5×IQR
  • IQR preferred for skewed marketing data
  • Handling: remove, cap, transform, keep
  • Winsorize: cmap to percentile boundary

Missing Data

  • MCAR: random — safe to drop
  • MAR: depends on other vars — impute
  • MNAR: depends on itself — domain knowledge needed
  • Mean/Median/Mode: simple imputation
  • KNN: most accurate, uses feature relationships

Exam Tips

  1. Z-score vs IQR: Z-score assumes normality; IQR is non-parametric. Use IQR for revenue, CLV, spend — all are right-skewed in marketing datasets.
  2. Common trap: |Z| > 2 detects moderate outliers; |Z| > 3 detects extreme outliers. Know which threshold you're applying.
  3. MCAR vs MAR: MCAR = dropping is safe (no pattern). MAR = other variables predict missingness (dropping causes bias). MNAR = the missing value itself predicts missingness (hardest case).
  4. Seaborn function levels: catplot() and displot() are figure-level (return FacetGrid); boxplot(), histplot() etc. are axes-level. Only axes-level functions accept an ax= argument.
  5. KNN imputation is the most accurate but computationally expensive. Simple imputation (mean/median) reduces variance artificially.
  6. Always visualise data before and after imputation/outlier removal — check histograms, boxplots, and pairplots.
  7. Log transformation: use np.log1p() (log of 1+x) to handle zeros. Reduces skewness and can satisfy linear regression normality assumptions.
  8. Winsorisation vs removal: winsorise when you believe the measurement is real but extreme; remove when you believe it's an error.

Each question is worth 2 marks — a mix of applied numerical and conceptual items. Click an option to check it; the correct answer and explanation appear instantly.

Q1 · 2 marks A feature has population mean μ = 70 and population standard deviation σ = 8. What is the Z-score of the value x = 82?
The Z-score measures how many σ above the mean the value lies.
Z = (x − μ) / σ = (82 − 70) / 8 = 12 / 8 = 1.5
The answer is 1.5. (12 forgets to divide by σ; 0.67 inverts the ratio.)
Q2 · 2 marks For the sorted data [3, 5, 7, 8, 9, 11, 13, 15], NumPy's linear-interpolation percentiles give Q1 = 6.5 and Q3 = 11.5. What is the IQR?
The interquartile range is the spread between the third and first quartiles.
IQR = Q3 − Q1 = 11.5 − 6.5 = 5.0
The answer is 5.0. (18.0 wrongly adds the quartiles; 9.0 is the median.)
Q3 · 2 marks A value has μ = 60, σ = 10, x = 95, giving a Z-score of 3.5. Using the convention Z > 3 = extreme outlier, how is this value classified?
Compute and compare to the threshold.
Z = (95 − 60) / 10 = 35 / 10 = 3.5 → 3.5 > 3
Since 3.5 exceeds the Z > 3 line, it is an extreme outlier. The Z-method needs no IQR, so (D) is wrong.
Q4 · 2 marks A column has values [10, 12, 14, 16, 18, 100] plus a missing entry. If you fill the missing value by median imputation, what value is inserted?
The median is the middle of the sorted non-null values; with 6 values it is the average of the 3rd and 4th.
sorted: 10, 12, 14, 16, 18, 100 median = (14 + 16) / 2 = 15.0
The answer is 15.0. (28.33 is the mean — pulled up by the outlier 100, which is exactly why median is preferred here.)
Q5 · 2 marks When a numeric column contains strong outliers, why is median imputation generally preferred over mean imputation for filling missing values?
A skewed/outlier-heavy distribution drags the mean toward the extremes, so filling with the mean distorts the column; the median sits at the centre regardless of extreme values. The answer is (B). Imputation fills rows rather than dropping them (C), and it does not force normality (D).
Q6 · 2 marks In a box-and-whisker plot, points drawn beyond the whiskers (past Q3 + 1.5·IQR or below Q1 − 1.5·IQR) represent:
The box spans Q1–Q3 (the middle 50%), the whiskers extend to the 1.5·IQR fences, and individual points beyond them are suspected outliers. The answer is (C). The box (not the points) holds the quartiles and the middle 50%.
