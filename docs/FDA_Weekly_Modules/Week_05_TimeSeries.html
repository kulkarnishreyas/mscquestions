1️⃣ Introduction to Time Series Data

Time series data = observations recorded at successive time intervals (daily stock prices, monthly sales, quarterly GDP).

Key Characteristics

📈 Trend

Long-term upward/downward direction (revenue growth)

🔄 Seasonality

Regular pattern at fixed intervals (Dec retail spike)

🌊 Cyclic Patterns

Longer, irregular cycles (economic booms/busts)

⚡ Irregular Variation

Random shocks (pandemic, geopolitical events)

Stationary vs Non-Stationary

TypePropertiesTests
StationaryConstant mean, variance, autocorrelation over timeADF (p<0.05 = stationary), KPSS
Non-StationaryChanging properties (e.g., growing revenue)Apply differencing, log, detrending

Applications

  • Stock price & portfolio risk analysis
  • Macroeconomic forecasting (GDP, inflation)
  • Demand & inventory planning
  • Weather, agriculture, energy
  • Healthcare/epidemiology monitoring

2️⃣ Time Series Components

Additive vs Multiplicative

ModelFormulaWhen to Use
AdditiveY = Trend + Seasonal + ResidualComponents independent; seasonal magnitude stays constant
MultiplicativeY = Trend × Seasonal × ResidualSeasonal effect grows with trend (financial data often fits)

Importance of Decomposition

  • Reveals true underlying signal vs noise
  • Improves forecast accuracy by modelling each piece
  • Helps detect anomalies in residuals
  • Supports strategic planning (trend) & operational planning (seasonality)

4️⃣ Moving Averages

Simple Moving Average (SMA)

SMA_t = (X_{t-n+1} + ... + X_t) / n
3-period SMA of 10, 20, 30, 40, 50:
SMA₃ = (10+20+30)/3 = 20; SMA₄ = (20+30+40)/3 = 30; SMA₅ = (30+40+50)/3 = 40

Weighted Moving Average (WMA)

WMA = Σ (wᵢ × Xᵢ) / Σ wᵢ   (higher weights for recent data)
Weights 3, 2, 1 over 10, 20, 30:
WMA = (3·30 + 2·20 + 1·10)/(3+2+1) = 140/6 ≈ 23.33

Advanced MAs

TypeUse
EMA (Exponential)Recent data weighted more; smoother yet responsive
DEMA (Double EMA)Less lag — fast-paced trading
TEMA (Triple EMA)Even less lag, sharp trend detection

Trading Signals

  • Golden Cross: 50-day SMA crosses above 200-day → bullish
  • Death Cross: 50-day SMA crosses below 200-day → bearish
  • Volume MA confirms price moves
data['SMA50'] = data['Close'].rolling(window=50).mean() data['SMA200'] = data['Close'].rolling(window=200).mean() data['EMA20'] = data['Close'].ewm(span=20, adjust=False).mean()

5️⃣ Smoothing Techniques

Why Smooth?

Reduce noise → reveal underlying trends → better forecasts.

Comparison

TechniqueCaptures Trend?Seasonality?Best For
SMA✓ (lagged)✗Stable data
WMA✓ (faster)✗Recent-relevant data
SES (Simple Exp Smoothing)✗✗Flat, non-seasonal

6️⃣ Autoregressive Models & ARIMA

AR(p) Model

Xₜ = φ₁ Xₜ₋₁ + φ₂ Xₜ₋₂ + ... + φₚ Xₜ₋ₚ + εₜ

Predicts current value from p past values plus noise.

Building an AR Model

  1. Stationarity Check — ADF test
  2. Order Selection — PACF plot reveals significant lags
  3. Parameter Estimation — OLS or MLE
  4. Model Evaluation — AIC, BIC (lower = better)

ARIMA in Python

import pmdarima as pm from statsmodels.tsa.arima.model import ARIMA # Auto-pick (p,d,q) auto = pm.auto_arima(data, seasonal=False, stepwise=True) print(auto.summary()) # Manual ARIMA model = ARIMA(data, order=(2,1,2)).fit() forecast = model.forecast(steps=30)

ARIMA vs LSTM vs GARCH

ModelBest ForLimitation
ARIMALinear, short-termMisses non-linear shifts, volatility
LSTM (Deep Learning)Non-linear, long sequencesData & compute hungry, less interpretable
GARCHVolatility clustering (financial)Focuses on variance, not levels

7️⃣ Scenario Analysis in Finance

Real-World Applications

  • JPMorgan Chase, Goldman Sachs — ARIMA + LSTM for stock forecasting & HFT
  • Bridgewater Associates — GARCH for portfolio volatility management

How Scenarios Differ

ARIMA scenarios

Adjust trend or seasonality assumptions for recession/expansion

LSTM scenarios

Feed geopolitical, macro inputs; learn complex impacts

GARCH scenarios

Simulate volatility spikes during crises or shocks

Strengths & Limits of ARIMA

✅ Strengths

Linear patterns, interpretable, fast on small/medium data

❌ Limits

Non-linear shifts, volatility clustering, structural breaks

💡 Practical Tip: Combine ARIMA (level), LSTM (complex patterns), and GARCH (volatility) in a hybrid model for robust real-world forecasting.

📝 Summary & Exam Tips

🎯 Key Takeaways

  • 4 components: Trend, Seasonality, Cyclic, Residual
  • Stationary = constant mean/variance. ADF tests it (p < 0.05 = stationary)
  • To make stationary: difference, log, detrend
  • Additive vs Multiplicative decomposition
  • Moving Averages: SMA (equal weight), WMA (custom weights), EMA (recent-heavy)
  • Golden Cross = bullish; Death Cross = bearish
  • Smoothing: SES (flat), Holt (trend), Holt-Winters (trend+seasonal), LOESS (complex)
  • ARIMA: AR(p) + I(d) + MA(q). Use ADF, PACF/ACF, AIC for selection
  • Hybrid: ARIMA (levels) + LSTM (non-linear) + GARCH (volatility)
📌 Pitfalls:
  • Forgetting stationarity check before ARIMA
  • Confusing trend with cycle (cycles are unpredictable in length)
  • Over-smoothing — loses real signals
  • Using ARIMA for non-linear/volatile series
