2️⃣ Linear Regression

Y = β₀ + β₁X₁ + β₂X₂ + ... + ε

Steps

  1. Collect & clean data
  2. Define X (independent) and Y (dependent)
  3. Train/test split (80/20)
  4. Fit via OLS
  5. Evaluate: R², MSE, p-values, t-stats

5 Assumptions

  • Linearity
  • Independence of errors
  • Homoscedasticity (constant variance)
  • Normal errors
  • No omitted variables

Coefficient Interpretation

MetricInterpretation
p < 0.05Variable significant
|t| > 2Strong impact
CI doesn't include 0Likely significant
VIF > 10Multicollinearity
R² (adj)% variance explained
Finance: stock return = β₀ + β₁(interest rate) + β₂(inflation) + ε. Used in CAPM beta calculation, sensitivity analysis.

3️⃣ Logistic Regression

Predicts binary outcomes: default/no-default, fraud/no-fraud, up/down.

P(Y=1) = e^(β₀+β₁X₁+...) / (1 + e^(β₀+β₁X₁+...))

Python Example

from sklearn.linear_model import LogisticRegression from sklearn.metrics import classification_report, roc_auc_score model = LogisticRegression() model.fit(X_train, y_train) y_pred = model.predict(X_test) y_prob = model.predict_proba(X_test)[:,1] print(classification_report(y_test, y_pred)) print("AUC:", roc_auc_score(y_test, y_prob))

Performance Metrics

🎯 Accuracy

(TP+TN)/Total — overall

🔍 Precision

TP/(TP+FP) — of predicted "fraud", how many real?

📡 Recall

TP/(TP+FN) — of real fraud, how many caught?

📈 AUC-ROC

Discriminative power (1 = perfect, 0.5 = random)

⚖️ F1

Harmonic mean of precision & recall

Linear vs Logistic

LinearLogistic
Continuous outcome (price)Binary outcome (default Y/N)
OLS minimises SSEMLE on log-odds
R², MSEAccuracy, AUC, Confusion Matrix

4️⃣ Decision Trees

Series of if-then splits leading to a prediction. Very interpretable.

Pros & Cons

✅ Pros❌ Cons
InterpretableOverfitting
Handles num + catUnstable to data shifts
Non-parametricStruggles with linear relations
Reveals feature importanceBias toward majority class

Combat Overfitting via Pruning

  • Pre-pruning: limit depth or min samples (e.g., max_depth=5)
  • Post-pruning: trim grown branches

Finance Applications

  • Loan approval (credit score + DTI rules)
  • Customer segmentation
  • Fraud transaction classification

5️⃣ Random Forests

Ensemble of many decision trees → vote (classification) or average (regression).

How It Works

  1. Bagging — each tree trains on random bootstrap sample
  2. Random feature selection at each split (decorrelates trees)
  3. Aggregate predictions: majority vote / mean
from sklearn.ensemble import RandomForestClassifier rf = RandomForestClassifier(n_estimators=100, max_depth=5, random_state=42) rf.fit(X_train, y_train) print("Accuracy:", rf.score(X_test, y_test))

Strengths & Limits

✅ Strengths❌ Limits
Reduces overfittingLess interpretable than single tree
Handles missing valuesComputationally heavier
Robust to outliersLarge memory footprint
Feature importance

  • Credit scoring (multi-attribute)
  • Fraud detection (imbalanced data)

Cross-Validation

from sklearn.model_selection import cross_val_score scores = cross_val_score(model, X, y, cv=5) print("CV mean:", scores.mean(), "± std:", scores.std())

Bias-Variance Trade-off

High Bias

Underfit. Simple model misses patterns.

High Variance

Overfit. Memorises training data; poor on test.

Goal

Balance — generalise to new data.

7️⃣ Advanced Techniques

Ensemble Methods

🎒 Bagging

Parallel; average predictions. Reduces variance (Random Forest)

🚀 Boosting

Sequential; each new model fixes errors. Reduces bias (AdaBoost, GBM, XGBoost, LightGBM)

🥞 Stacking

Meta-model combines multiple base models

from sklearn.ensemble import GradientBoostingClassifier gbm = GradientBoostingClassifier() gbm.fit(X_train, y_train)

Deep Learning

  • LSTM — sequential data (stock prediction)
  • CNN — spatial features (text/image-based finance)

Explainability — SHAP & LIME

import shap explainer = shap.TreeExplainer(model) shap_values = explainer.shap_values(X_test) shap.summary_plot(shap_values, X_test)
  • SHAP — feature attribution using Shapley values
  • LIME — local surrogate models

LASSO Regression

from sklearn.linear_model import Lasso lasso = Lasso(alpha=0.01) lasso.fit(X, y) # Coefficients shrunk to 0 = features removed

Reinforcement Learning & Bayesian Networks

  • RL — agent learns optimal policy via reward signals (portfolio management, trading)
  • Bayesian Networks — probabilistic graphical models for uncertainty (credit risk, macroeconomics)

📝 Summary & Exam Tips

🎯 Key Takeaways

  • Linear Regression: continuous Y; OLS; check assumptions; R², p-value
  • Logistic Regression: binary Y; outputs probability; AUC, Confusion Matrix
  • Decision Trees: interpretable; overfit fix = pruning
  • Random Forest: bagging + feature randomness; robust, less interpretable
  • Boosting (GBM, XGBoost): sequential, reduces bias, often beats RF
  • Evaluation: Accuracy, Precision, Recall, F1, AUC for classification; MAE, MSE, RMSE, R² for regression
  • Use SHAP/LIME for explainability in ML models
  • LASSO shrinks weak features to zero (feature selection)
  • Imbalanced classes: prefer F1/AUC over Accuracy
📌 Pitfalls:
  • Accuracy is deceiving on imbalanced data (99% non-fraud)
  • Forgetting cross-validation — single split misleading
  • Ignoring regression assumptions (linearity, normality)
  • Using "black-box" models without explainability tools — regulators reject
