1️⃣ Introduction to Financial Data Analytics

Financial Data Analytics (FDA) = the process of gathering, interpreting, and transforming financial data into actionable insights. It drives business decisions in profitability, cash flow, risk, and compliance.

🎯 The 6 Key Benefits

📈 Enhanced Decision-Making

Real-time insights into profitability, revenue, and expenses — replacing guesswork with data.

🛡️ Effective Risk Management

Predict credit defaults, market downturns, cash shortages. Enables contingency planning.

⚙️ Performance Optimisation

Compare planned vs actual; reallocate resources for better outcomes.

🎯 Accurate Forecasting

Realistic budgets, achievable targets, fewer financial surprises.

⚖️ Regulatory Compliance

Aligns with GAAP, IFRS. Eases audits, reduces legal risk.

🚀 Competitive Advantage

Spot trends early, respond faster to market changes.

💡 The Big Picture: FDA replaces gut-feel decisions with evidence-based ones. Every benefit above traces back to better data → better decisions → better outcomes.

2️⃣ Core Components of FDA Framework

The 3 Types of Analytics

🔍 Descriptive

What happened? Examines historical data, past trends.

🔮 Predictive

What will happen? Regression, ML to forecast revenue/risk.

🎯 Prescriptive

What should we do? Scenario simulation for optimal choices.

4️⃣ Data Collection

Data Collection = systematic gathering & measurement of information to support decisions.

Primary vs Secondary Data

AspectPrimarySecondary
SourceFirsthand (surveys, experiments)Existing records (reports, DBs)
PurposeCurrent studyPast purpose, reused
Cost & TimeHighLow
AccuracyHigh, customisedMay lack precision
ControlFull controlDepends on others' methods
SpecificityHighly specificPartial fit
TimelinessReal-timeMay be outdated
ExamplesTesco surveys, Barclays interviewsBloomberg, Yahoo Finance, govt reports

Reliability vs Validity

🔁 Reliability

Consistency across trials. Same method → same results. Government sources, peer-reviewed journals.

🎯 Validity

Accuracy — does it measure what it claims to? High precision, low error, aligned with research question.

Ethics in Data Collection

  • Informed Consent — voluntary, clear disclosure
  • Privacy & Confidentiality — secure sensitive info
  • Minimising Harm — psychological/financial
  • Data Security — encryption, role-based access
  • Transparency — purpose disclosed

5️⃣ Data Preprocessing Techniques

Preprocessing transforms raw, unorganised financial data into clean, structured form — essential before any analysis or modelling.

Data Cleaning Methods

❓ Missing Values

Mean/median imputation, mode for categorical, KNN for complex cases.

🔁 Duplicates

Remove duplicated transactions to prevent inflated counts.

📍 Outliers

Detect via Z-score (>3) or boxplots. Cap, transform, or exclude.

📏 Standardise Formats

Convert dates to YYYY-MM-DD, normalise currency.

✏️ Entry Errors

Unify "NA"/"Not Available"/"-", fix typos via lookup.

🗑️ Irrelevant Data

Drop fields that don't contribute to objective.

Transformation & Normalisation

TechniquePurpose
Min-Max ScalingBrings features to [0,1] range
Z-Score StandardisationMean 0, SD 1 — for algos needing normal-like distribution
Log TransformCompress large values, reduce skewness
Box-CoxStabilise variance, enhance normality

Missing Data Categories (MCAR/MAR/NMAR)

🎲 MCAR

Missing Completely at Random — no pattern. Safe to drop or simply impute.

🔗 MAR

Missing at Random — depends on observed vars. Imputation works if relationship modelled.

⚠️ NMAR

Not Missing at Random — depends on missing value itself. Hardest; need domain modelling.

Outlier Detection

  • Z-score: |z| > 3 typically flagged
  • Boxplots: values beyond Q1-1.5×IQR or Q3+1.5×IQR
  • Winsorization: cap extreme values rather than drop
💡 Why preprocessing matters: Garbage in, garbage out. A great model on bad data still gives bad answers. Preprocessing typically takes 60-80% of an analyst's time.

8️⃣ Automating Tasks with Python Functions

Reusable Financial Functions

Daily Returns

def calculate_daily_returns(df): return df.pct_change().dropna()

Moving Averages (SMA / EMA)

def moving_average(series, window=20, method='SMA'): if method == 'SMA': return series.rolling(window=window).mean() elif method == 'EMA': return series.ewm(span=window, adjust=False).mean()

Portfolio Return

def portfolio_return(returns, weights): return (returns * weights).sum(axis=1)

Correlation/Covariance Matrices

def get_correlation_and_covariance(df): return df.corr(), df.cov()

📝 Summary & Exam Tips

🎯 Key Takeaways

  • FDA turns financial data into actionable insights for decision-making
  • 9 framework components: collection → integration → analysis → KPIs → viz → automation → scenario → compliance → mitigation
  • 3 analytics types: Descriptive (past), Predictive (future), Prescriptive (recommend)
  • Primary data: firsthand, costly. Secondary: reused, cheap
  • Reliability = consistency. Validity = accuracy/relevance
  • Preprocessing = handle missing (MCAR/MAR/NMAR), duplicates, outliers (Z>3), formats, errors
  • Python finance stack: pandas, numpy, matplotlib, seaborn, scipy, statsmodels, sklearn, yfinance
  • Visualisation: line for trends, bar for categories, box/violin for distributions, heatmap for correlation
  • Automate daily returns (pct_change), moving averages (rolling), portfolio return, correlation matrix
📌 Common Exam Pitfalls:
  • Confusing reliability (consistent) with validity (accurate)
  • Missing data types: MCAR (random) vs MAR (depends on observed) vs NMAR (depends on missing)
  • Forgetting that Pandas pct_change() returns NaN for the first row
  • Pie charts are only good for ≤5 categories — use bar otherwise
  • Outliers must be investigated, not always removed (could be real events like festivals/crises)
