1️⃣ Introduction to Algorithmic Trading

Algorithmic Trading = automated buy/sell decisions executed by computer algorithms based on predefined rules, often in milliseconds.

Key Advantages

⚡ Speed

Milliseconds execution

💧 Liquidity

Higher volume, tighter spreads

🎯 Accuracy

Removes human error/emotion

📊 Data Analysis

Historical + real-time

🎨 Strategy Variety

Arbitrage, trend, market-making

💰 Lower Cost

Less manual oversight

🛡️ Built-in Risk Controls

Stop-loss, exposure limits

Moving Average Crossover (Python)

import pandas as pd, numpy as np data['75_MA'] = data['Close'].rolling(75).mean() data['200_MA'] = data['Close'].rolling(200).mean() data['Signal'] = np.where(data['75_MA'] > data['200_MA'], 1, 0) data['Position'] = data['Signal'].diff() # Position = 1 → buy; -1 → sell

2️⃣ Trading Strategies

5 Strategy Families

📈 Trend Following

MA Crossovers, Bollinger Bands. Best in trending markets.

🔄 Mean Reversion

Prices revert to average. RSI, oversold/overbought.

⚖️ Arbitrage

Exploit price differences. NSE-BSE, statistical, merger.

💱 Market Making

Profit from bid-ask spread. Risky in volatility.

📰 Sentiment-Based

NLP on news, tweets, earnings calls.

Strategy Performance by Market Regime

MarketBest Strategies
Bull (uptrend)Trend-following, sentiment
Bear (downtrend)Arbitrage, mean reversion
SidewaysMean reversion, market-making
High volatilityTrend (with tight risk controls)

Constructing SMA Crossover Strategy

data['SMA50'] = data['Close'].rolling(50).mean() data['SMA200'] = data['Close'].rolling(200).mean() data['Signal'] = 0 data.loc[data['SMA50'] > data['SMA200'], 'Signal'] = 1 data.loc[data['SMA50'] < data['SMA200'], 'Signal'] = -1 data['Position'] = data['Signal'].shift(1) data['StrategyReturns'] = data['Position'] * data['Close'].pct_change() print('Total Return:', data['StrategyReturns'].cumsum().iloc[-1])

3️⃣ Backtesting

Apply a strategy to historical data to assess potential performance before deploying live.

Performance Metrics

MetricDefinition
Net Profit / ROITotal return
Sharpe RatioRisk-adjusted return
Sortino RatioDownside-risk-only adjusted
Max DrawdownPeak-to-trough loss
Win Rate% of profitable trades
Win/Loss RatioAvg win / Avg loss
import numpy as np, pandas as pd dates = pd.date_range('2020-01-01', '2021-01-01', freq='B') prices = 100 * (1 + np.random.randn(len(dates))*0.01).cumprod() returns = np.diff(prices)/prices[:-1] df = pd.DataFrame({'Price':prices[1:],'Return':returns},index=dates[1:]) df['Signal'] = np.where(df['Return']>0, 1, -1) df['StratRet'] = df['Signal'].shift(1) * df['Return'] # Metrics sharpe = df['StratRet'].mean()/df['StratRet'].std() * np.sqrt(252) cum = (1+df['StratRet']).cumprod() dd = (cum - cum.cummax())/cum.cummax() print(f"Sharpe: {sharpe:.2f}, MaxDD: {dd.min():.2%}")

Common Pitfalls

🔍 Data-Snooping Bias

Over-tuning to historical data → poor out-of-sample. Use train/test split.

👁️ Look-Ahead Bias

Using future data in past trades. Lag data correctly.

🎯 Overfitting

Strategy fails live. Cross-validate.

🪦 Survivorship Bias

Only successful firms in data. Use full history including delisted.

🌊 Regime Sensitivity

Performs in bull, fails in bear. Test across regimes.

4️⃣ High-Frequency Trading (HFT)

Specialised algo trading: trades executed in milliseconds/microseconds, large volumes, tiny gains per trade.

Impact on Markets

✅ Positives

Tighter spreads, increased liquidity, faster price discovery

⚠️ Negatives

Uneven playing field; can amplify volatility (flash crashes)

HFT Infrastructure

  • Low-latency networks (microwave, fibre, transoceanic)
  • Algorithmic decision engines with ML
  • Server co-location next to exchange data centres
  • Advanced analytics processing massive real-time feeds

Rewards & Risks

RewardsRisks
Cumulative profits via volumeTech failures
Market influenceVolatility amplification
Tech edgeEthical & regulatory concerns

Liquidity & Volatility

  • Provides liquidity in normal conditions
  • May withdraw liquidity in stress, worsening crashes
  • Regulators must balance innovation vs stability

5️⃣ Risk Management in Trading

Core Techniques

  • Diversification across instruments
  • Hedging via options/futures
  • Risk-Reward Analysis per trade

Position Sizing

📊 Fixed %

Same % of equity each trade

💵 Fixed ₹

Same amount each trade

📈 Volatility-Based

Smaller size in high-volatility

Stop-Loss Orders

🔒 Fixed

% or ₹ below entry

🏃 Trailing

Moves up with profitable price

📊 Volatility-Based

Adjusts with market range

Take-Profit Orders

The mirror image of stop-loss — automatically closes a winning trade when a pre-set profit target is reached. Locks in gains without depending on the trader's emotions or attention.

🎯 Fixed Target

Exit at a pre-defined % or ₹ above entry (e.g., +2%)

📐 Risk-Reward Ratio

Set TP as a multiple of the stop-loss distance (e.g., 1:2 — risk ₹1 to make ₹2)

📈 Volatility-Based

TP scaled by ATR or recent range so it adapts to the regime

💡 Pair stop-loss + take-profit: Together they bracket every position, enforcing discipline and making the algorithm fully autonomous — no manual exit decisions.

Integration into Strategy

  1. Define risk tolerance up front
  2. Embed sizing & stop-loss in code
  3. Monitor and adjust continuously
  4. Leverage automation for consistency

6️⃣ Case Studies (Indian Market)

Statistical Arbitrage by Hedge Fund

Success Factors

Co-location, fast networks

Backtested + risk controls embedded

📊 Data & AI

ML refines patterns continuously

📜 Regulatory Compliance

SEBI throttling, risk limits

Lessons

  • Scalability: small-capital wins ≠ large-capital wins. Test at scale.
  • Adaptability: markets shift; static models fail
  • Risk Mitigation: circuit breakers, kill-switches
  • Cross-functional teams: quants + data + traders

📝 Summary & Exam Tips

🎯 Key Takeaways

  • Algo trading = automated rules-based execution
  • 5 strategy families: trend, mean reversion, arbitrage, market-making, sentiment
  • Backtest before live; measure with Sharpe, Sortino, Drawdown
  • HFT: speed + co-location; risks of volatility amplification
  • Risk: position sizing (fixed%, fixed₹, volatility), stop-losses (fixed/trailing/volatility)
  • Pitfalls: data-snooping, look-ahead, overfitting, survivorship bias
📌 Pitfalls:
  • Over-optimising on historical data → fails live
  • Ignoring regime changes (bull-bear transitions)
  • Scaling tested with small capital — slippage worsens at scale
  • No human oversight on automated systems
