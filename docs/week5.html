AWT GUI Programming

Building graphical user interfaces with Java's Abstract Window Toolkit — event handling, layout managers, and interactive components

Introduction to AWT

The Abstract Window Toolkit (AWT) is Java's original GUI framework, part of the Java Foundation Classes (JFC). AWT provides a set of classes for building platform-native graphical interfaces. It uses the operating system's own GUI components — making AWT apps look different on Windows, macOS, and Linux.

Key Characteristic: AWT components are heavyweight — each component is a wrapper around a native OS widget. This means the OS does the actual rendering, giving native look-and-feel but less control over appearance.

Why learn AWT? AWT's event model (the Delegation Event Model) is the foundation for both Swing and JavaFX. Understanding AWT means understanding how all Java GUI frameworks work at their core.

AWT Component Hierarchy

Every AWT widget extends Component. Containers can hold other components. Understanding this hierarchy is key to building UIs correctly.

Container vs Component: A Container can hold other Components inside it. Panel, Frame, and Dialog are all Containers. Button and Label are simple Components — they cannot contain other components.

Key AWT Components

AWT provides a set of basic UI elements. Each wraps a native OS widget:

Button

java.awt.Button



A push button that triggers an ActionEvent when clicked. Created with new Button("label"). Uses ActionListener for event handling.

Label

java.awt.Label



Displays read-only text. Alignment can be LEFT, CENTER, or RIGHT. label.setText("new") updates it dynamically.

TextField

java.awt.TextField



Single-line text input. tf.getText() reads value; tf.setEchoChar('*') makes it a password field.

TextArea

java.awt.TextArea



Multi-line text input with optional scrollbars. Created with new TextArea(rows, cols). Supports append(), insert(), replaceRange().

Checkbox

java.awt.Checkbox



A toggle-able check box. Can be grouped into a CheckboxGroup for radio-button behavior. cb.getState() returns boolean.

Choice

java.awt.Choice



A drop-down selection list. Items added with choice.add("item"). choice.getSelectedItem() returns selection.

List

java.awt.List



A scrollable list allowing single or multiple selection. new List(visibleRows, multipleSelection).

Scrollbar

java.awt.Scrollbar



A horizontal or vertical scrollbar. Used for value selection in a range. Generates AdjustmentEvent.

Canvas

java.awt.Canvas



A blank drawing surface. Override paint(Graphics g) to draw shapes, images, or text directly.

Frame — The Main Window

A Frame is the top-level window with a title bar, border, and optional menu bar. It's the foundation of any AWT application.

Java — BasicFrame.java
import java.awt.*;
import java.awt.event.*;

public class BasicFrame extends Frame {
    public BasicFrame() {
        super("My First AWT Window");  // sets title
        setSize(400, 300);
        setLayout(new FlowLayout());

        Label lbl = new Label("Hello, AWT!");
        Button btn = new Button("Click Me");

        add(lbl);
        add(btn);

        // Close window on X button
        addWindowListener(new WindowAdapter() {
            public void windowClosing(WindowEvent e) {
                dispose();
            }
        });

        setVisible(true);
    }

    public static void main(String[] args) {
        new BasicFrame();
    }
}

Event Handling — Delegation Event Model

Java AWT uses the Delegation Event Model (introduced in Java 1.1). The idea is simple: the component that generates an event (the source) delegates handling to a separate listener object.

Step 1 — Implement a Listener

Create a class that implements the appropriate listener interface (e.g., ActionListener).

Step 2 — Register the Listener

Call component.addActionListener(listener) to register your listener with the source component.

Step 3 — Handle the Event

Implement the required method (e.g., actionPerformed(ActionEvent e)). This is called automatically when the event occurs.

Event Object Info

e.getSource() — the component that fired the event
e.getActionCommand() — label of button or typed text
e.getWhen() — timestamp of the event

Common Event Types

Event ClassListener InterfaceSource ComponentsMethod
ActionEventActionListenerButton, TextField, MenuItemactionPerformed(e)
MouseEventMouseListenerAny ComponentmouseClicked, mousePressed, ...
KeyEventKeyListenerAny ComponentkeyPressed, keyReleased, keyTyped
WindowEventWindowListenerFrame, DialogwindowClosing, windowOpened, ...
ItemEventItemListenerCheckbox, Choice, ListitemStateChanged(e)
AdjustmentEventAdjustmentListenerScrollbaradjustmentValueChanged(e)
Java — ActionListenerDemo.java
import java.awt.*;
import java.awt.event.*;

public class ActionListenerDemo extends Frame
        implements ActionListener {
    Label result;

    public ActionListenerDemo() {
        super("ActionListener Demo");
        setLayout(new FlowLayout());
        setSize(300, 150);

        Button b1 = new Button("Say Hello");
        Button b2 = new Button("Say Bye");
        result = new Label("Press a button...");

        b1.addActionListener(this);  // register listener
        b2.addActionListener(this);

        add(b1); add(b2); add(result);
        setVisible(true);
    }

    public void actionPerformed(ActionEvent e) {
        String cmd = e.getActionCommand();
        if (cmd.equals("Say Hello"))
            result.setText("Hello, World!");
        else
            result.setText("Goodbye!");
    }
}

FlowLayout

FlowLayout is the default layout for Panel (and Applet). It arranges components left-to-right in a row, wrapping to the next line when the row is full — like words in a paragraph.

Constructor: new FlowLayout() — center-aligned with 5px gaps
new FlowLayout(FlowLayout.LEFT, 10, 10) — left-aligned, 10px H-gap, 10px V-gap
Java — FlowLayoutDemo.java
import java.awt.*;

public class FlowLayoutDemo extends Frame {
    public FlowLayoutDemo() {
        super("FlowLayout Demo");
        setLayout(new FlowLayout(FlowLayout.LEFT, 10, 10));
        setSize(350, 200);

        for (int i = 1; i <= 7; i++) {
            add(new Button("Button " + i));
        }

        setVisible(true);
    }
}

GridLayout

GridLayout arranges components in a rectangular grid. Every cell is the same size, and components fill their cell completely. Components are added left-to-right, top-to-bottom.

Constructor: new GridLayout(rows, cols)
new GridLayout(3, 2, 5, 5) — 3 rows, 2 cols, 5px horizontal gap, 5px vertical gap
Setting rows=0 means "as many rows as needed" and vice versa for cols.
Java — GridLayoutDemo.java
import java.awt.*;

public class GridLayoutDemo extends Frame {
    public GridLayoutDemo() {
        super("GridLayout 3x2");
        setLayout(new GridLayout(3, 2, 5, 5));
        setSize(300, 200);

        String[] labels = {"Name:","Age:","City:",
                            "","",""};
        for (String s : labels) {
            if (s.isEmpty())
                add(new TextField(15));
            else
                add(new Label(s));
        }
        setVisible(true);
    }
}

BorderLayout

BorderLayout is the default layout for Frame and Dialog. It divides the container into five named regions. Each region holds at most one component, and the CENTER region gets all remaining space.

Java — BorderLayoutDemo.java
import java.awt.*;

public class BorderLayoutDemo extends Frame {
    public BorderLayoutDemo() {
        super("BorderLayout Demo");
        // BorderLayout is default for Frame, but explicit here:
        setLayout(new BorderLayout(5, 5));
        setSize(400, 300);

        add(new Button("NORTH"),  BorderLayout.NORTH);
        add(new Button("SOUTH"),  BorderLayout.SOUTH);
        add(new Button("EAST"),   BorderLayout.EAST);
        add(new Button("WEST"),   BorderLayout.WEST);
        add(new Button("CENTER"), BorderLayout.CENTER);

        setVisible(true);
    }
}

Calculator using GridLayout

A classic AWT example — a calculator using GridLayout for the button pad and ActionListener for button events.

Java — CalcFrame.java
import java.awt.*;
import java.awt.event.*;

public class CalcFrame extends Frame
        implements ActionListener {
    TextField display;
    double num1 = 0;
    String op = "";
    boolean newNum = true;

    public CalcFrame() {
        super("Calculator");
        setLayout(new BorderLayout());
        setSize(220, 280);

        display = new TextField("0");
        display.setEditable(false);
        add(display, BorderLayout.NORTH);

        Panel btnPanel = new Panel(
            new GridLayout(4, 4, 3, 3));

        String[] btns = {
            "7","8","9","/",
            "4","5","6","*",
            "1","2","3","-",
            "0",".","=","+"
        };

        for (String label : btns) {
            Button b = new Button(label);
            b.addActionListener(this);
            btnPanel.add(b);
        }

        add(btnPanel, BorderLayout.CENTER);
        setVisible(true);
    }

    public void actionPerformed(ActionEvent e) {
        String cmd = e.getActionCommand();
        if (cmd.matches("[0-9.]")) {
            if (newNum) { display.setText(cmd); newNum = false; }
            else display.setText(display.getText() + cmd);
        } else if (cmd.equals("=")) {
            double num2 = Double.parseDouble(display.getText());
            switch (op) {
                case "+": display.setText("" + (num1 + num2)); break;
                case "-": display.setText("" + (num1 - num2)); break;
                case "*": display.setText("" + (num1 * num2)); break;
                case "/": display.setText("" + (num1 / num2)); break;
            }
            newNum = true;
        } else {
            num1 = Double.parseDouble(display.getText());
            op = cmd;
            newNum = true;
        }
    }
}

KeyListener

The KeyListener interface lets you respond to keyboard events on any Component that has focus.

KeyListener Methods

MethodWhen CalledKey Info Available
keyPressed(KeyEvent e)When a key is pressed downgetKeyCode(), getKeyChar()
keyReleased(KeyEvent e)When a key is releasedgetKeyCode(), getKeyChar()
keyTyped(KeyEvent e)After press+release of a printable keygetKeyChar() only (code is UNDEFINED)
keyChar vs keyCode:
e.getKeyChar() — the Unicode character ('A', '1', '@')
e.getKeyCode() — integer code for all keys including Ctrl/Alt/F1 (use KeyEvent.VK_* constants)
e.isShiftDown(), e.isControlDown() — modifier key state
Java — KeyListenerDemo.java
import java.awt.*;
import java.awt.event.*;

public class KeyListenerDemo extends Frame
        implements KeyListener {
    TextArea log;

    public KeyListenerDemo() {
        super("Key Events");
        setLayout(new BorderLayout());
        setSize(400, 300);

        TextField input = new TextField("Type here...");
        log = new TextArea(8, 40);
        log.setEditable(false);

        input.addKeyListener(this);

        add(input, BorderLayout.NORTH);
        add(log, BorderLayout.CENTER);
        setVisible(true);
    }

    public void keyPressed(KeyEvent e) {
        log.append("PRESSED: code=" + e.getKeyCode()
            + " char=" + e.getKeyChar() + "\n");
    }
    public void keyReleased(KeyEvent e) {
        log.append("RELEASED: " + e.getKeyChar() + "\n");
    }
    public void keyTyped(KeyEvent e) {
        log.append("TYPED: '" + e.getKeyChar() + "'\n");
    }
}

WindowAdapter

The WindowListener interface has 7 methods — you'd have to implement all of them even if you only care about window closing. WindowAdapter is an adapter class that provides empty implementations of all 7 methods, so you only override the ones you need.

WindowListener Methods

MethodWhen Called
windowOpened(WindowEvent e)First time window becomes visible
windowClosing(WindowEvent e)User clicks the close (X) button — window not yet closed
windowClosed(WindowEvent e)After dispose() completes
windowIconified(WindowEvent e)Window minimized
windowDeiconified(WindowEvent e)Window restored from minimize
windowActivated(WindowEvent e)Window gains focus
windowDeactivated(WindowEvent e)Window loses focus
Important: windowClosing is where you put System.exit(0) or dispose(). Without it, clicking the X button does nothing — the window won't close!
Java — Complete AWT App with WindowAdapter
import java.awt.*;
import java.awt.event.*;

public class CompleteAWTApp extends Frame
        implements ActionListener {
    TextField nameField, ageField;
    Choice cityChoice;
    Checkbox agreeBox;
    TextArea output;

    public CompleteAWTApp() {
        super("Complete AWT Form");
        setLayout(new BorderLayout(8, 8));
        setSize(450, 400);

        // Form panel
        Panel form = new Panel(new GridLayout(4, 2, 5, 5));
        form.add(new Label("Name:"));
        nameField = new TextField();
        form.add(nameField);
        form.add(new Label("Age:"));
        ageField = new TextField();
        form.add(ageField);
        form.add(new Label("City:"));
        cityChoice = new Choice();
        cityChoice.add("Mumbai"); cityChoice.add("Delhi");
        cityChoice.add("Bangalore"); cityChoice.add("Pune");
        form.add(cityChoice);
        agreeBox = new Checkbox("I agree to terms");
        form.add(new Label(""));
        form.add(agreeBox);
        add(form, BorderLayout.NORTH);

        output = new TextArea(5, 40);
        output.setEditable(false);
        add(output, BorderLayout.CENTER);

        Button submit = new Button("Submit");
        submit.addActionListener(this);
        add(submit, BorderLayout.SOUTH);

        // WindowAdapter — only override what we need
        addWindowListener(new WindowAdapter() {
            public void windowClosing(WindowEvent e) {
                System.exit(0);
            }
            public void windowOpened(WindowEvent e) {
                output.append("Window opened!\n");
            }
        });

        setVisible(true);
    }

    public void actionPerformed(ActionEvent e) {
        output.setText("");
        output.append("Name: " + nameField.getText() + "\n");
        output.append("Age: "  + ageField.getText()  + "\n");
        output.append("City: " + cityChoice.getSelectedItem() + "\n");
        output.append("Agreed: " + agreeBox.getState() + "\n");
    }

    public static void main(String[] args) {
        new CompleteAWTApp();
    }
}

Summary — AWT Key Points:
• AWT = heavyweight, native OS widgets
• Frame is the top-level window; use WindowAdapter to handle closing
• Delegation Event Model: Source fires event → Listener handles it
• Layout Managers: FlowLayout (default Panel), GridLayout (grids), BorderLayout (default Frame, 5 regions)
• KeyListener: keyPressed/keyReleased/keyTyped — use KeyAdapter to avoid implementing all 3
• Always call setVisible(true) last after adding all components

NP1GridLayout cell count. A Frame's content is laid out with the manager below. How many equal-sized grid cells does it define?

cells = rows × cols = 6 × 4 = 24

NP7Empty grid cells. A panel uses new GridLayout(4, 2) and the program adds 3 Labels, 3 TextFields and 1 Button (7 components total). How many grid cells remain empty?

Q1 · 2 marks A Frame's content area is configured with the layout manager below, and the program then adds 12 buttons. How many grid cells stay empty?
setLayout(new GridLayout(5, 3));
GridLayout(rows, cols) defines rows × cols equal cells, filled in order.
total cells = 5 × 3 = 15 empty = 15 − 12 = 3
3 cells remain empty (answer C).
Q2 · 2 marks In the AWT "Add Two Numbers" calculator, the user types 487 and 159 and clicks Add. The handler parses both fields as int and assigns sum = n1 + n2. What value is written to the result field?
Integer.parseInt converts the text to int, so + is numeric addition, not concatenation.
487 + 159 = 646
The result field shows 646 (answer B). Option A is the mistake of adding two Strings.
Q3 · 2 marks In a null-layout Frame, a Button is positioned with the call below (signature setBounds(x, y, width, height)). What is the y-coordinate of the button's bottom edge, in pixels?
b.setBounds(30, 60, 120, 90);
The bottom edge sits at the top y plus the height (the 3rd arg 120 is width, not height).
y_bottom = y + height = 60 + 90 = 150
The bottom edge is at y = 150 px (answer C). Option B wrongly uses the width.
Q4 · 2 marks AWT components are described as heavyweight. Which statement best explains what that means?
A heavyweight component wraps a native OS peer; the OS does the actual drawing, giving native look-and-feel but a different appearance per platform. Option C describes Swing's lightweight model. Answer: B.
Q5 · 2 marks A component is added to a BorderLayout Frame with add(comp) — i.e. with no region constraint given. In which region does it appear?
In BorderLayout, add(comp) with no constraint defaults to CENTER, which stretches to fill the space left by the four edge regions. Answer: C.
Q6 · 2 marks Under the Delegation Event Model, you want your code to run when the user clicks an AWT Button. Which listener interface do you implement, and which method handles the click?
A Button click fires an ActionEvent, handled by an ActionListener registered with addActionListener(...); the callback is actionPerformed(ActionEvent e). MouseListener fires for low-level mouse activity, not the semantic button "action". Answer: B.
