📚 History of Java

Java is one of the most widely used programming languages in the world. Its journey from a small research project to a global standard is a fascinating story.

1991
Sun Microsystems formed the "Green Team" led by James Gosling, Mike Sheridan, and Patrick Naughton. The goal was to create a language for consumer electronics (set-top boxes, TVs).
1992
The language was originally named "Oak" (after the oak tree outside Gosling's office). A demo device called Star7 was built running Oak.
1995
Oak was renamed "Java" (reportedly after Java coffee). Sun officially released Java 1.0 with the famous slogan: "Write Once, Run Anywhere" (WORA). Netscape browser added Java applet support.
1996–2004
Java grew rapidly. Java 1.1, 1.2 ("Playground"), 1.3 ("Kestrel"), 1.4 ("Merlin"), and Java 5.0 ("Tiger") introduced generics, enums, annotations, and enhanced for-loops.
2006
Sun made Java open-source under the GNU GPL license — the OpenJDK project was born.
2010
Oracle Corporation acquired Sun Microsystems for $7.4 billion, gaining ownership of Java.
2023
On the TIOBE index, Java is ranked 4th (after Python, C, and C++). Still one of the most in-demand languages for enterprise, Android, and backend development.
Key People & Facts:
  • James Gosling — Father of Java, designed the original language
  • Green Team — Small research group at Sun Microsystems (1991)
  • Sun Microsystems — Original developer, founded 1982
  • Oracle — Current owner since 2010
  • Renamed from Oak → Java in 1995

⚙️ JDK / JRE / JVM

Understanding the three components of Java's execution model is fundamental. They are nested: JDK ⊃ JRE ⊃ JVM.

JDK — Java Development Kit

The complete package for developers. Contains everything needed to write, compile, and run Java programs.

  • javac — Java compiler (.java → .class)
  • java — Java launcher (runs .class)
  • javadoc — Documentation generator
  • jar — Archive tool
  • jdb — Java debugger

JRE — Java Runtime Environment

Needed to run Java programs (not develop). Contains the JVM plus the standard class libraries.

  • JVM — Bytecode executor
  • java.lang — Core classes
  • java.util — Utility classes
  • java.io — I/O classes
  • Other standard libraries

JVM — Java Virtual Machine

The engine that executes bytecode. Platform-specific — different JVM for Windows, Linux, macOS.

  • Reads .class (bytecode) files
  • Interprets or JIT-compiles to native machine code
  • Manages memory (Garbage Collection)
  • Provides security sandbox
  • Platform-specific implementation

Compilation and Execution Flow

Source Code (.java) ——[javac]—→ Bytecode (.class) ——[JVM]—→ Machine Output
Key Insight: The bytecode (.class file) is platform-independent. The JVM is platform-specific. This is the secret behind "Write Once, Run Anywhere".

🌐 Platform Independence (WORA)

"Write Once, Run Anywhere" — Java source code is compiled once into platform-neutral bytecode. That bytecode can run on any machine that has a JVM installed, regardless of the underlying OS or CPU architecture.

Why is Bytecode Platform-Independent?

Bytecode is an intermediate representation — it's NOT specific to any CPU (x86, ARM, MIPS). The JVM on each platform translates bytecode to the platform's native machine code.

JVM is Platform-Specific

Even though bytecode is universal, you need a JVM installed on each target platform. Oracle, OpenJDK, and others provide JVMs for every major OS.

✨ Features of Java

Java was designed with a rich set of features. Click any card to reveal its description.

🥤
Simple
Java has a clean, easy-to-learn syntax. It removed complex features of C++ like pointers, operator overloading, and multiple inheritance, making it easier for beginners.
🌍
Object Oriented
Everything in Java is modeled as objects. Java supports all four OOP pillars: Abstraction, Encapsulation, Inheritance, and Polymorphism. Even primitive wrappers are classes.
📚
Interpreted
Java bytecode is interpreted by the JVM at runtime. Modern JVMs also use Just-In-Time (JIT) compilation for frequently executed code, combining interpretation and compilation.
⚡
High Performance
Through JIT compilation and efficient bytecode, Java achieves near-native performance. The JVM optimizes hot code paths automatically at runtime using HotSpot technology.
📦
Portable
Java bytecode is platform-neutral. The same .class file runs unchanged on Windows, Linux, macOS, and mobile. "Write Once, Run Anywhere" is the key portability promise.
🛡️
Robust
Java has strong type-checking at compile time, automatic garbage collection (no memory leaks), exception handling, and no direct pointer manipulation — making programs reliable.
🔒
Secure
Java runs inside the JVM sandbox, preventing unauthorized access to system resources. No explicit pointers prevent buffer overflows. Bytecode is verified before execution.
🔀
Multithreaded
Java has built-in support for multithreading via the Thread class and Runnable interface. Multiple threads can run concurrently, enabling efficient use of multicore processors.
🔁
Dynamic
Java can load classes at runtime using Class.forName(). It supports reflection, dynamic linking, and runtime polymorphism. Classes can be loaded on demand from the network or disk.
🌐
Distributed
Java's java.net package provides extensive support for TCP/IP networking, HTTP, FTP, and RMI (Remote Method Invocation) for distributed computing across networks.
🌎
Architecture-Neutral
Java's bytecode has no implementation-dependent aspects. Primitive types have fixed sizes (int is always 32 bits), unlike C/C++ where size depends on the platform architecture.

💻 First Java Program

Every Java programmer starts with Hello World. Let's understand every single keyword in this program.

First.java
public class First { public static void main(String s[]) { System.out.println("Hello Welcome to learn Java"); } }
Output:
Hello Welcome to learn Java

Keyword Breakdown

KeywordMeaning
publicAccess modifier — the class/method is accessible from anywhere (outside the package too)
classDeclares a new class — the fundamental building block in Java. Everything must be inside a class.
FirstThe class name — must match the filename (First.java). By convention, class names use PascalCase.
staticMeans the method belongs to the class itself, not an instance. JVM calls main() without creating an object.
voidReturn type — main() returns nothing. The JVM doesn't need a return value from main.
mainThe method name — the entry point of every Java application. JVM looks specifically for main(String[]).
String s[]Command-line arguments — an array of Strings passed when running: java First arg1 arg2
System.out.printlnSystem = class in java.lang, out = static PrintStream field, println = method that prints and adds a newline

How to Compile and Run

Step 1: Save as First.java
Step 2: Compile: javac First.java → creates First.class
Step 3: Run: java First → JVM executes First.class

Why is main() static?

The JVM needs to call main() before any objects exist. If main() were an instance method, the JVM would need an object of the class to call it — but you can't create objects without a running program. Making it static solves this chicken-and-egg problem.

Why String s[]?

Java accepts command-line arguments as an array of strings. Running java First Alice 25 would put "Alice" at s[0] and "25" at s[1]. You can also write it as String[] s — both are equivalent.

Class vs Object

Class = Blueprint

A class is a template or blueprint that defines the structure and behaviour. It exists in your code but takes no memory by itself. Like an architectural drawing of a house.

class Person { name, age, talk() }

Object = Instance

An object is an actual instance of a class in memory. You can have many objects from one class, each with their own data. Like an actual house built from the blueprint.

Person p1 = new Person("Bob", 32);

🏛️ Four Pillars of OOP

1. Abstraction — Hide Complexity

Show only what an object does, hide how it does it. A TV remote lets you change channels without knowing the circuit board inside.

Real-World Example: When you press the power button on a TV remote, you don't need to know about the infrared signal frequency, microcontrollers, or internal circuitry. You just see the interface — buttons.

In Java: Achieved using abstract classes and interfaces. You define what a method does (signature) without implementing how.

Note Inheritance and Polymorphism are covered in Week 2 — they build on the foundation established here.

2. Encapsulation — Protect Data

Wrap data (variables) and methods that operate on the data into a single unit (class). Make fields private and provide public getter/setter methods to access them safely.

Why Encapsulate?

  • Prevents direct modification of sensitive data
  • Validation can be added in setters
  • Internal implementation can change without breaking external code
  • Promotes modularity
Encapsulation Example
public class BankAccount { private double balance; // private — cannot access directly public void deposit(double amount) { if (amount > 0) balance += amount; // validated! } public double getBalance() { return balance; // controlled read access } }

3. Inheritance — Reuse Code

A child class inherits attributes and methods from a parent class using the extends keyword. This promotes code reuse and establishes an "is-a" relationship.

Example: A Dog class can extend Animal — it inherits eat(), sleep() from Animal and adds its own bark() method.
Covered in detail in Week 2.

4. Polymorphism — Many Forms

The ability of an object to take on many forms. One method name can behave differently based on the object type or arguments. Enables flexible, extensible code.

Example: A draw() method called on a Circle draws a circle, on a Square draws a square — same method name, different behaviour.
Covered in detail in Week 2.

🏛️ Class Structure in Java

A Java class can contain instance variables, static variables, and methods. Understanding the difference between instance and static members is crucial.

Account.java
public class Account { // Instance variables — each object gets its own copy private long accno; private float balance; // Static variable — SHARED across ALL objects public static float interestRate; // Instance method — operates on a specific object's data public void withdraw(float amount) { if (amount <= balance) balance -= amount; } public void deposit(float amount) { balance += amount; } // Static method — belongs to the class, not any object public static void printInterestRate() { System.out.println("Interest Rate: " + interestRate + "%"); } }
Conceptual output (if Account.interestRate = 3.5f; Account.printInterestRate(); is called):
Interest Rate: 3.5%

Instance Variables

  • Each object gets its own copy
  • Created when object is created (with new)
  • Accessed via object reference: a1.accno
  • Destroyed when object is garbage collected

Static Variables

  • One copy shared by ALL objects
  • Created when class is loaded into memory
  • Accessed via class name: Account.interestRate
  • Destroyed when JVM terminates

🛠️ Constructors

What is a Constructor? A constructor is a special method that is automatically called when an object is created using new. It initializes the object's state.

Rules for Constructors

  • The constructor name must be exactly the same as the class name
  • Constructors have no return type (not even void)
  • Called automatically when new ClassName() is used
  • A class can have multiple constructors (overloading)
  • If no constructor is defined, Java provides a default no-arg constructor

Three Types of Constructors

Circle.java — Default, Parameterized, and Copy Constructors
public class Circle { private float radius, area; // 1. Default Constructor — no parameters, sets default values public Circle() { radius = 10; } // 2. Parameterized Constructor — accepts arguments public Circle(float radius) { this.radius = radius; // 'this' refers to current object } // 3. Copy Constructor — creates a copy of another object public Circle(Circle other) { this.radius = other.radius; } public float findArea() { area = 3.14f * radius * radius; return area; } public static void main(String s[]) { Circle c1 = new Circle(); // default: radius=10 Circle c2 = new Circle(5.0f); // parameterized: radius=5 Circle c3 = new Circle(c2); // copy: copies c2's radius System.out.println("c1 area: " + c1.findArea()); System.out.println("c2 area: " + c2.findArea()); System.out.println("c3 area: " + c3.findArea()); } }
Output:
c1 area: 314.0 c2 area: 78.5 c3 area: 78.5

The this Keyword

What does this mean?

this refers to the current object. In the parameterized constructor Circle(float radius), there's a naming conflict: the parameter is called radius and the instance variable is also called radius. Writing this.radius = radius says: "Set the current object's radius field equal to the parameter radius."

📦 Packages

What is a Package? A package is a namespace that organizes related classes and interfaces. Think of it like folders on a filesystem — they group related files and prevent naming conflicts.

Why Use Packages?

  • Organization: Group related classes (e.g., all database classes in one package)
  • Naming Conflicts: Two classes can have the same name if in different packages (e.g., java.util.Date vs java.sql.Date)
  • Access Control: Package-private access allows classes in the same package to share internal details
  • Reusability: Package a set of classes as a library for reuse across projects

Naming Convention

Reverse domain + project + module: com.oracle.jdbc.driver
Company: com.google.android.app
Organization: org.apache.commons.lang
Your project: com.example.myapp.utils

The package and import Keywords

com/example/myapp/Hello.java
// package declaration — MUST be the first line in the file package com.example.myapp; // import statement — brings in classes from other packages import java.util.Scanner; // specific class import import java.io.*; // wildcard — imports all classes in java.io public class Hello { public static void main(String s[]) { Scanner scan = new Scanner(System.in); System.out.println("Package: com.example.myapp"); } }
Output:
Package: com.example.myapp

🔢 Engineered Numerical Practice

Fresh exam-style computational problems built from this week's concepts. Type your answer and hit Check for instant feedback, or reveal the worked steps. (Press Enter in any answer box to check.)
NP1Operator precedence & integer division. Evaluate the expression and give the int result:
int x = 7 + 4 % 3 * 5 - 8 / 3;
%, *, / bind tighter than + / -, evaluated left to right.
4 % 3 = 1 1 * 5 = 5 8 / 3 = 2 (integer division, truncates) 7 + 5 - 2 = 10
Result = 10.
NP2Circle area. The Circle class computes area as 3.14f * radius * radius. For a circle of radius = 7, what value does findArea() return?
Using π ≈ 3.14 as in the notes:
area = 3.14 * 7 * 7 = 3.14 * 49 = 153.86
Result = 153.86.
NP3Narrowing cast / overflow. A byte is 8-bit signed (range −128..127). What is printed?
byte b = (byte) 200; System.out.println(b);
200 exceeds the byte max (127), so it wraps within the 256-value range:
200 - 256 = -56
(Equivalently 200 = 11001000₂; as signed 8-bit that is −56.) Result = -56.
NP4Char / ASCII arithmetic. Characters use Unicode code points ('A' = 65). What integer is printed?
char a = 'A', z = 'Z'; System.out.println(z - a);
'Z' = 90, 'A' = 65. In arithmetic, chars promote to int:
90 - 65 = 25
Result = 25.
NP5Ternary operator. What value is assigned to diff?
int a = 8, b = 5; int diff = (a > b) ? a - b : b - a;
a > b is 8 > 5 → true, so the first branch a - b is taken:
8 - 5 = 3
Result = 3.
NP6Integer division & modulo. Evaluate the int result:
int r = 18 / 4 + 18 % 4;
18 / 4 = 4 (quotient, truncated) 18 % 4 = 2 (remainder) 4 + 2 = 6
Result = 6.
NP7Post-increment evaluation. What value is assigned to y?
int x = 4; int y = x++ + x++;
Each x++ uses the current value, then increments. Evaluated left to right:
first x++ → uses 4, x becomes 5 second x++ → uses 5, x becomes 6 y = 4 + 5 = 9
Result = 9 (and x ends at 6).
NP8Data-type range. A byte in Java is an 8-bit signed integer. How many distinct values can a single byte variable represent?
8 bits → 2⁸ bit patterns:
2^8 = 256 (range -128 .. 127, inclusive = 256 values)
Result = 256.

📝 Exam-Style MCQs — 2 Marks Each

Each question is worth 2 marks — a mix of applied numerical and conceptual items. Click an option to check it; the correct answer and explanation appear instantly.

Q1 · 2 marks Applying Java operator precedence and integer division, what value is assigned to z?
int z = 6 + 8 / 2 * 3 - 4 % 3;
*, /, % bind tighter than +/- and are evaluated left to right.
8 / 2 = 4 4 * 3 = 12 4 % 3 = 1 6 + 12 - 1 = 17
Reading strictly left-to-right (ignoring precedence) gives 2 — a common mistake. The answer is 17.
Q2 · 2 marks A byte is an 8-bit signed type (range −128..127). What is printed?
byte b = (byte) 300; System.out.println(b);
Narrowing to byte keeps only the low 8 bits, so the value wraps within the 256-value range.
300 mod 256 = 300 - 256 = 44 44 is within -128..127, so it stays 44
Java does not clamp to 127 (that is the distractor), and 300 is not representable. The answer is 44.
Q3 · 2 marks Characters use Unicode code points ('A' = 65). What integer is printed?
char c = (char) ('A' + 25); System.out.println((int) c);
In arithmetic, char promotes to int; the cast back to char then to int preserves the code point.
'A' + 25 = 65 + 25 = 90 (the character 'Z') (int) c = 90
The answer is 90.
Q4 · 2 marks A developer compiles Hello.java on a Windows machine and ships only the resulting Hello.class file to a colleague running Linux, who executes it unchanged. Which statement best explains why this works?
"Write Once, Run Anywhere" works because javac produces bytecode — an intermediate format independent of any CPU/OS. Each platform ships its own JVM, which loads that same bytecode and executes it. The bytecode itself is not native code (rules out A and D), and Java is compiled, not directly source-interpreted (rules out B). The answer is C.
Q5 · 2 marks A BankAccount class keeps its balance field private and exposes only deposit() and withdraw() methods that validate the amount before changing the balance. Which OOP pillar does this design primarily demonstrate?
Hiding the data (private balance) and forcing all access through validated public methods is the definition of encapsulation — bundling state with the methods that guard it. Inheritance is about reuse via parent/child classes; polymorphism is "one interface, many forms"; abstraction is hiding implementation behind a simpler model. The pillar shown here is Encapsulation.
Q6 · 2 marks Consider a class that declares no constructor at all, plus a second class that declares only a constructor taking parameters. Which statement is correct?
class A { int x; } // no constructor class B { B(int n) { /* ... */ } } // only a parameterized one
The compiler supplies an implicit no-arg default constructor only when a class declares no constructor. A qualifies, so new A() works. Once you declare any constructor (as B does), the default is not generated, so new B() with no arguments fails to compile. The answer is B.
