🎮 Reinforcement Learning Overview

Core Idea: An agent learns to make decisions by interacting with an environment. Instead of being given correct answers (supervised) or finding hidden structure (unsupervised), the agent receives reward signals and must figure out, through trial and error, which actions lead to the best long-term outcome.

Think of it like training a dog: you don't tell the dog exactly what to do step-by-step. You reward good behaviour (+treat) and discourage bad behaviour (no treat). Over time the dog learns which actions produce rewards.

How RL Differs from Other Paradigms

Supervised Learning

Given labeled (input, correct answer) pairs.

A teacher tells you: "this photo is a cat."

RL does NOT have labels.

Unsupervised Learning

No labels, discovers hidden patterns.

No feedback signal at all — just find structure.

RL DOES have rewards.

Reinforcement Learning

No labels, but gets scalar reward signals.

Feedback is delayed, noisy, and sparse.

Goal: maximise cumulative reward.

The Agent–Environment Loop

Real-World Applications

  • Game Playing: AlphaGo (2016) beat world champion Lee Sedol at Go — a game with more board positions than atoms in the universe
  • Robotics: Teaching robots to walk, grasp objects, and navigate without hard-coded motion planning
  • Self-Driving Cars: Learning driving policies from interactions with simulated and real environments
  • Algorithmic Trading: Agents learn optimal buy/sell strategies by interacting with market simulators
  • Recommender Systems: Optimise for long-term user engagement rather than just next-click CTR
  • Drug Discovery: RL agents design molecules by sequentially adding atoms and receiving feedback on desired properties

🔑 Key Concepts

Core Components

Agent

The learner and decision-maker. The agent perceives its environment through states, chooses actions, and aims to maximise cumulative reward. Examples: a chess program, a robot arm, a stock-trading bot.

Environment

Everything the agent interacts with. It receives actions from the agent and emits the next state and reward. The environment can be fully observable (agent sees everything) or partially observable (agent has limited view).

State (S)

A representation of the current situation. In chess: the board position. In a robot: joint angles + camera image. A good state captures all information needed to make the best decision (Markov property).

Action (A)

What the agent can do. Can be discrete (move Up/Down/Left/Right) or continuous (steering angle ∈ [−30°, +30°]). The set of all valid actions in state s is the action space.

Reward (R)

A scalar feedback signal received after each action. Can be positive (good) or negative (bad). It is the agent's only feedback about whether its actions are working. Designing good reward functions is an art.

  • Chess: +1 win, −1 loss, 0 draw
  • Grid world: −1 per step, +10 at goal
  • Robot: +distance moved forward, −energy used

Policy (π)

The agent's strategy — a mapping from states to actions.

Deterministic: a = π(s) — for every state, one fixed action.

Stochastic: π(a|s) = P(a|s) — a probability distribution over actions given a state. More flexible; better for environments with uncertainty.

Episode

A complete sequence from the starting state to a terminal state (e.g., winning/losing a game). Each episode: (s₀,a₀,r₁), (s₁,a₁,r₂), …, (s_T). Not all tasks are episodic — some are continuing (no terminal state).

Return (G) — Cumulative Discounted Reward

The total reward the agent collects from time t to end of episode, with later rewards discounted:

Gₜ = rₜ₊₁ + γ·rₜ₊₂ + γ²·rₜ₊₃ + … = Σₖ₌₀^∞ γᵏ · rₜ₊ₖ₊₁

Why Discount (γ)?

  • γ ∈ [0, 1]: controls how much future rewards matter
  • γ = 0: agent only cares about immediate reward (greedy)
  • γ = 1: agent weighs all future rewards equally (only works for episodic tasks)
  • γ = 0.9: typical value — rewards 10 steps away are worth 0.9¹⁰ ≈ 35% of immediate reward

Return Example

Rewards: r₁=−1, r₂=−1, r₃=−1, r₄=+10, γ=0.9

G₀ = −1 + 0.9·(−1) + 0.81·(−1) + 0.729·10

G₀ = −1 − 0.9 − 0.81 + 7.29 = +4.58

A positive return means reaching the goal was worthwhile despite the step penalties.

🔢 Markov Decision Process (MDP)

Markov Property: "The future is independent of the past given the present." Formally: P(Sₜ₊₁ | Sₜ, Aₜ, Sₜ₋₁, Aₜ₋₁, …) = P(Sₜ₊₁ | Sₜ, Aₜ). The current state captures everything needed to predict the future.

A Markov Decision Process (MDP) is the mathematical framework for RL. Almost all RL problems are formulated as MDPs.

MDP = (S, A, P, R, γ)

Why MDP?

The MDP framework lets us formally state RL problems. Once we have an MDP, we can apply algorithms to find the optimal policy — the policy that maximises expected return from every starting state. Methods that use P(s'|s,a) are called model-based; those that don't (like Q-learning) are model-free.

📈 Value Functions & the Bellman Equation

Value functions answer: "How good is it to be in state s?" or "How good is it to take action a in state s?" These functions are central to most RL algorithms.

State-Value Function V(s)

V^π(s) = E_π[Gₜ | Sₜ = s] = E_π[ Σₖ₌₀^∞ γᵏ · rₜ₊ₖ₊₁ | Sₜ = s ]

V^π(s) is the expected cumulative discounted reward starting from state s and following policy π thereafter. A higher V(s) means state s is a "better" place to be.

Action-Value Function Q(s, a) — the Q-function

Q^π(s, a) = E_π[Gₜ | Sₜ = s, Aₜ = a]

Q^π(s,a) is the expected return from state s, taking action a first, then following policy π. Q-functions directly tell you which action to take.

Relationship: V^π(s) = Σ_a π(a|s) · Q^π(s,a)  —  V is the average of Q weighted by the policy.

The Bellman Equation — Recursive Definition

The key insight: the value of a state equals the immediate reward plus the discounted value of the next state.

V^π(s) = Σ_a π(a|s) · Σ_{s'} P(s'|s,a) · [ R(s,a,s') + γ · V^π(s') ]
Q^π(s,a) = Σ_{s'} P(s'|s,a) · [ R(s,a,s') + γ · Σ_{a'} π(a'|s') · Q^π(s',a') ]

Optimal Value Functions

The optimal value functions represent the best possible performance:

V*(s) = max_a Σ_{s'} P(s'|s,a) · [ R(s,a,s') + γ · V*(s') ]
Q*(s,a) = Σ_{s'} P(s'|s,a) · [ R(s,a,s') + γ · max_{a'} Q*(s',a') ]
Optimal Policy: Once we have Q*(s,a), the optimal policy is trivial:
π*(s) = argmax_a Q*(s, a)

Just pick the action with the highest Q-value in each state!

Worked Example: Bellman Backup

A simple 2-state chain: s₀ → s₁ → terminal. γ=0.9.

R(s₁→terminal) = +10, R(s₀→s₁) = −1

Step 1: V*(terminal) = 0 (no future rewards)

Step 2: V*(s₁) = R(s₁,a,terminal) + γ·V*(terminal) = 10 + 0.9·0 = 10

Step 3: V*(s₀) = R(s₀,a,s₁) + γ·V*(s₁) = −1 + 0.9·10 = −1 + 9 = 8

So even though taking the first step costs −1, state s₀ is still worth 8 because of the future +10 reward.

⚡ Q-Learning Algorithm

Q-Learning is model-free: it does NOT need to know the transition probabilities P(s'|s,a). It learns Q* directly from experience by interacting with the environment.

Q-Table

For small state/action spaces, we store Q(s,a) in a table: rows = states, columns = actions. Initially all zeros. We update entries as the agent explores.

The Q-Learning Update Rule

Q(s,a) ← Q(s,a) + α · [ r + γ · max_{a'} Q(s',a') − Q(s,a) ]
TermNameRole
Q(s,a)Current estimateWhat we currently think action a in state s is worth
r + γ·max Q(s',a')TD TargetBetter estimate using actual reward + discounted future value
TD Target − Q(s,a)TD Error (δ)How wrong our current estimate is — the "surprise"
αLearning rateHow fast to update (0.1–0.5 typical). Small α = slow but stable.
γDiscount factorHow much future rewards matter

Intuition: Temporal Difference Learning

Q-Learning is a Temporal Difference (TD) method. It updates estimates based on the difference between consecutive predictions — like adjusting a weather forecast as new information arrives, rather than waiting until the end of the week to see if it rained.

Worked Update: Suppose Q(s=0, a=Right) = 0 (initial). Agent takes Right from s=0, lands in s=1, gets reward r=−1. max Q(s=1, a') = 0 (all zeros initially). α=0.5, γ=0.9.

Q(0,Right) ← 0 + 0.5·[−1 + 0.9·0 − 0] = 0 + 0.5·(−1) = −0.5

Now Q(0,Right) = −0.5. The agent has learned that going Right from s=0 is slightly bad (costs a step).

Epsilon-Greedy Exploration

The agent faces the exploration vs. exploitation dilemma: should it use what it knows (exploit) or try something new (explore)?

With probability ε — Explore

Pick a random action from the action space. This ensures the agent discovers new paths and doesn't get stuck in local optima.

Early in training: ε should be high (lots of exploration).

With probability 1−ε — Exploit

Pick the greedy action: a* = argmax_a Q(s,a). Use current knowledge to get the best known reward.

Late in training: ε should be low (mostly exploit).

ε decay: εₜ = ε_min + (ε_max − ε_min) · exp(−λ · t)

Common values: ε_max=1.0, ε_min=0.01, λ=0.001. After ~5000 steps ε ≈ 0.07 — the agent exploits 93% of the time.

Q-Learning Algorithm Pseudocode

Initialize Q(s,a) = 0 for all s, a
for episode = 1 to N_episodes:
  s ← initial state
  while s is not terminal:
    ε-greedy: with prob ε → a = random action
                with prob 1-ε → a = argmax_a Q(s,a)
    Execute action a → observe r, s'
    Q(s,a) ← Q(s,a) + α · [r + γ · max_{a'} Q(s',a') − Q(s,a)]
    s ← s'
  Decay ε
Return Q-table (optimal policy: π*(s) = argmax_a Q(s,a))

Problem Setup

A 4×4 grid with 16 cells numbered 0–15 (left-to-right, top-to-bottom). The agent starts at cell 0 (top-left).

  • Goal: Reach cell 15 (bottom-right) → reward +10
  • Holes: Cells 5 and 11 → reward −10 (episode ends)
  • Every step: reward −1 (encourages efficiency)
  • Actions: Up (↑), Down (↓), Left (←), Right (→)
  • Hitting a wall: stay in same cell, still get −1

MDP Parameters:

  • α (learning rate) = 0.5
  • γ (discount) = 0.9
  • ε (start) = 1.0, decays to 0.1
  • S = {0, 1, …, 15}
  • A = {Up, Down, Left, Right}
Grid Layout:
S: start (cell 0)   G: goal (cell 15)
H: hole (cells 5, 11)   .: safe cell

Grid Diagram

Episode 1 — Pure Random Exploration (ε = 1.0)

All Q-values = 0. Agent picks random actions.

Path: 0 →(Right)→ 1 →(Down)→ 5 [HOLE! r=−10, episode ends]

Updates:

Q(0, Right) ← 0 + 0.5·[−1 + 0.9·max Q(1,·) − 0] = 0.5·(−1+0) = −0.5

Episode 2 — Some Learning (ε = 0.8)

Q-table now has some non-zero entries. Agent avoids some bad moves.

Path: 0 →(Down)→ 4 →(Down)→ 8 →(Right)→ 9 →(Right)→ 10 →(Right)→ 11 [HOLE! r=−10]

Key update:

Q(10, Right) ← 0 + 0.5·[−10 + 0.9·0 − 0] = −5.0

Q(9, Right) ← 0 + 0.5·[−1 + 0.9·(−5) − 0] = 0.5·(−1−4.5) = −2.75

Agent learns to avoid going Right from cells 10 and 9.

Episode 3 — Near-Optimal Path (ε = 0.5)

Q-table has learned to avoid holes. Agent finds a near-optimal route.

Path: 0 →(Down)→ 4 →(Down)→ 8 →(Down)→ 12 →(Right)→ 13 →(Right)→ 14 →(Right)→ 15 [GOAL! r=+10]

Return: G = −1 − 0.9 − 0.81 − 0.729 − 0.656 − 0.590 + 0.9⁶·10 ≈ −4.69 + 5.31 ≈ +0.62

Q updates propagate back:

Q(14,Right) ← 0 + 0.5·[+10 + 0.9·0 − 0] = +5.0

Q(13,Right) ← 0 + 0.5·[−1 + 0.9·5.0 − 0] = 0.5·3.5 = +1.75

Q(12,Right) ← 0 + 0.5·[−1 + 0.9·1.75 − 0] = 0.5·0.575 = +0.29

🧠 Deep Q-Network (DQN)

The Scaling Problem: Q-tables work for small state spaces (16 cells, 4 actions = 64 values). But for Atari games, the state is a raw 84×84 pixel image — that's 84×84×3 ≈ 21,168 dimensions. The Q-table would be astronomically large and impossible to fill.

Solution: Neural Network as Q-Function Approximator

Instead of a table, use a neural network: Input = state s, Output = Q(s, a) for every action a. The network learns to generalize across similar states — just like how humans recognize similar situations without memorizing every pixel.

Q(s, a; θ) ≈ Q*(s, a)     θ = neural network weights

Key DQN Innovation 1: Experience Replay

Problem: consecutive experience samples (sₜ, aₜ, rₜ, sₜ₊₁) are highly correlated → unstable training.

Fix: Store transitions in a replay buffer of size N (e.g., 10,000). Sample random mini-batches to train on. This breaks correlations and allows reuse of experience.

1. Agent generates (s, a, r, s') → push to replay buffer

2. Sample random batch of 32 transitions

3. Compute TD targets: y = r + γ·max_{a'} Q̂(s',a'; θ⁻)

4. Gradient descent on loss: L = (y − Q(s,a; θ))²

Key DQN Innovation 2: Target Network

Problem: the TD target r + γ·max Q(s',a'; θ) uses the same network θ that we're updating → moving target → divergence.

Fix: Use a separate target network θ⁻ (frozen copy). Update θ⁻ ← θ every C steps (e.g., C=1000).

Online network θ: updated every step

Target network θ⁻: updated every C steps

TD target uses θ⁻, gradient uses θ

This stabilises training dramatically.

When to Use RL?

Use RL when:

  • You have a sequential decision problem
  • Actions have long-term consequences
  • No labeled data / supervisor available
  • Can simulate the environment cheaply

Challenges of RL:

  • Reward function design is hard
  • Requires millions of interactions
  • Very sensitive to hyperparameters
  • Sim-to-real gap in robotics

Recent Advances:

  • AlphaGo / AlphaZero (2016–2017)
  • OpenAI Five — Dota 2 (2019)
  • ChatGPT uses RLHF (2022)
  • RT-2 robot control (2023)
