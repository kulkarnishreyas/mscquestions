🔍 Unsupervised Learning Overview

Key Idea: No labels — the algorithm must discover hidden structure in raw data entirely on its own. Instead of learning a mapping from inputs to known outputs, we let the data speak for itself.

In supervised learning you hand the model a textbook with answers. Unsupervised learning gives the model only the questions and asks: "What patterns do you see?" This is far closer to how humans actually learn — most real-world data is unlabelled.

Why Unsupervised Learning?

Customer Segmentation

Group shoppers by purchase behaviour to personalise marketing — without knowing in advance how many groups exist.

Anomaly Detection

Credit card fraud, network intrusion. Anomalies are points that don't belong to any "normal" cluster.

Data Compression

Dimensionality reduction (PCA, autoencoders) compresses high-dimensional data while retaining key structure.

📍 K-Means Clustering

K-Means is the most widely used clustering algorithm. It partitions n data points into K non-overlapping clusters by minimising the within-cluster sum of squared distances to the centroid.

Algorithm Steps

1
Choose K — decide how many clusters you want.
2
Initialise K centroids — randomly pick K data points (or random positions).
3
Assignment step — assign each point to the nearest centroid: argmin_k ||x − μk||²
4
Update step — recompute each centroid as the mean of all assigned points: μk = (1/|Ck|) Σ x
5
Repeat steps 3–4 until assignments no longer change (convergence).
Objective: minimise J = Σᵢ Σ_{x∈Cᵢ} ||x − μᵢ||²

Time Complexity

O(n × K × d × iterations) where n = number of points, K = clusters, d = dimensions, iterations = convergence steps. Typically very fast in practice — usually converges in 10–50 iterations.

📊 Choosing K

The Elbow Method

Run K-Means for K = 1, 2, 3, ... and plot the inertia (within-cluster sum of squares) for each K. As K increases, inertia always decreases — but at some point the gains diminish. The "elbow" is where adding more clusters stops being worth it.

Inertia = Σᵢ₌₁ᴷ Σ_{x∈Cᵢ} ||x − μᵢ||²

The elbow at K=3 indicates that going from K=2 to K=3 yields a big reduction in inertia, while K=4, 5, 6 give diminishing returns.

Silhouette Score

The elbow method is subjective. The silhouette score gives a quantitative measure of how well each point fits its assigned cluster.

s(i) = (b(i) − a(i)) / max(a(i), b(i))
a(i) = mean distance from point i to all other points in the same cluster (cohesion — how tight the cluster is)
b(i) = mean distance from point i to all points in the nearest other cluster (separation — how far is the next cluster)

Range: [−1, +1]. Near +1 = well-clustered. Near 0 = on boundary. Near −1 = likely misassigned.

Silhouette Calculation for Our 8-Point Example (Final K=2 Clusters)

Cluster 1: {P1=(2,10), P8=(4,9)}  |  Cluster 2: {P2=(2,5), P3=(8,4), P4=(5,8), P5=(7,5), P6=(6,4), P7=(1,2)}

For P1=(2,10) in Cluster 1:

a(P1) = mean dist to P8 = sqrt((2−4)²+(10−9)²) = sqrt(4+1) = 2.24

b(P1) = mean dist to {P2,P3,P4,P5,P6,P7}:

d(P1,P2)=sqrt(0+25)=5.00 | d(P1,P3)=sqrt(36+36)=8.49
d(P1,P4)=sqrt(9+4)=3.61 | d(P1,P5)=sqrt(25+25)=7.07
d(P1,P6)=sqrt(16+36)=7.21 | d(P1,P7)=sqrt(1+64)=8.06
mean = (5.00+8.49+3.61+7.07+7.21+8.06)/6 = 39.44/6 = 6.57

s(P1) = (6.57 − 2.24) / max(2.24, 6.57) = 4.33 / 6.57 = 0.659

For P4=(5,8) in Cluster 2:

a(P4) = mean dist to {P2,P3,P5,P6,P7}:

d(P4,P2)=sqrt(9+9)=4.24 | d(P4,P3)=sqrt(9+16)=5.00
d(P4,P5)=sqrt(4+9)=3.61 | d(P4,P6)=sqrt(1+16)=4.12
d(P4,P7)=sqrt(16+36)=7.21
mean = (4.24+5.00+3.61+4.12+7.21)/5 = 24.18/5 = 4.84

b(P4) = mean dist to Cluster 1 = {P1, P8}:

d(P4,P1)=3.61 | d(P4,P8)=sqrt(1+1)=1.41
mean = (3.61+1.41)/2 = 2.51

s(P4) = (2.51 − 4.84) / max(4.84, 2.51) = −2.33 / 4.84 = −0.481

P4 has a negative silhouette score — it is closer to Cluster 1 than its own cluster. This makes sense geometrically: P4=(5,8) sits near the boundary.

Interpretation: The mean silhouette score for this K=2 solution is moderate, indicating these two clusters are somewhat separable but the boundary points (P4, P8) sit between clusters. A K=2 solution is structurally reasonable for this data but not perfect.

⭐ K-Means++ Initialisation

The Problem with Random Initialisation

Standard K-Means initialises centroids at random positions. This can lead to:

  • Poor convergence — many more iterations needed
  • Local optima — algorithm converges to a suboptimal clustering
  • Non-reproducibility — results vary wildly between runs

K-Means++ Algorithm

1
Pick first centroid uniformly at random from the data points.
2
For each remaining centroid to choose: compute D(x)² = squared distance from point x to the nearest already-chosen centroid.
3
Sample the next centroid with probability proportional to D(x)². Points far from existing centroids are more likely to be chosen.
4
Repeat until K centroids are chosen, then proceed with standard K-Means.
P(x chosen as next centroid) = D(x)² / Σₓ D(x)²
Theoretical guarantee: K-Means++ achieves an expected cost at most O(log K) times the optimal. In practice, it converges in fewer iterations and consistently finds better solutions than random initialisation.

Worked Example: K-Means++ on Our 8 Points (K=2)

🎨 K-Modes (Categorical Data)

Problem: K-Means requires numeric data and uses Euclidean distance + mean. What if features are categorical (hair colour, eye colour, country)?

K-Modes Modifications

Euclidean Distance → Hamming Distance

Count the number of feature positions where two records differ:

d(x,y) = Σᵢ [xᵢ ≠ yᵢ]

Example: (Black, Brown, Light) vs (Black, Blue, Dark) → differ at positions 2,3 → d=2

Mean → Mode

Instead of computing the arithmetic mean, the cluster centre is the mode (most frequent value) for each feature:

μₖⱼ = argmax_v Σᵢ [xᵢⱼ = v]

Each dimension's "centroid" is the most common category in that cluster.

🌳 Hierarchical Clustering

Agglomerative (bottom-up): Start with each point as its own cluster. Repeatedly merge the two closest clusters until one cluster remains. Produces a dendrogram.
Divisive DIANA (top-down): Start with all points in one cluster. Repeatedly split the most heterogeneous cluster. Less commonly used.

Unlike K-Means, hierarchical clustering does not require specifying K in advance. You build the full tree and cut at the desired height.

Linkage Methods

The key choice is how to measure the distance between two clusters (not just two points):

LinkageDistance FormulaCharacteristicBest For

Completemax{d(a,b): a∈A, b∈B}Compact, roughly equal-sized clustersWell-separated clusters
Averagemean{d(a,b): a∈A, b∈B}Compromise between single & completeGeneral use
WardIncrease in total varianceMinimises within-cluster variance; similar to K-MeansCompact, similar-sized clusters

🌀 Gaussian Mixture Models (GMM)

Key Insight: K-Means makes hard assignments — each point belongs to exactly one cluster. GMM makes soft assignments — each point has a probability of belonging to each cluster, enabling uncertainty quantification.

The Model

A GMM assumes data is generated from a mixture of K Gaussian distributions:

p(x) = Σₖ₌₁ᴷ πₖ × N(x | μₖ, Σₖ)
πₖ — mixing coefficient (prior probability of component k)
Σₖ πₖ = 1, all πₖ ≥ 0
N(x|μₖ,Σₖ) — Gaussian with mean μₖ and covariance Σₖ
N(x|μ,Σ) = (2π)^{-d/2} |Σ|^{-1/2} exp(−½(x−μ)ᵀΣ⁻¹(x−μ))

The EM Algorithm

We cannot directly solve for the parameters because we don't know which Gaussian generated each point. The Expectation-Maximisation (EM) algorithm iterates between two steps:

E-Step — Compute Responsibilities

Given current parameters (πₖ, μₖ, Σₖ), compute the "responsibility" of component k for data point xᵢ:

rᵢₖ = πₖ × N(xᵢ | μₖ, Σₖ) / Σⱼ πⱼ × N(xᵢ | μⱼ, Σⱼ)

rᵢₖ ∈ [0,1] is the posterior probability that point xᵢ came from component k. Think of it as a "soft cluster membership".

M-Step — Update Parameters

Using the responsibilities as weights, re-estimate each component's parameters:

Nₖ = Σᵢ rᵢₖ (effective number of points in component k)
μₖ = (1/Nₖ) Σᵢ rᵢₖ xᵢ (weighted mean)
Σₖ = (1/Nₖ) Σᵢ rᵢₖ (xᵢ − μₖ)(xᵢ − μₖ)ᵀ (weighted covariance)
πₖ = Nₖ / N (mixing coefficient = fraction of data)

Repeat E-step and M-step until log-likelihood converges.
