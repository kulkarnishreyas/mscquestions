Step-by-step Solutions

Mean Absolute Error (MAE):

MAE = (1/n) · Σ |yᵢ − ŷᵢ| = (1/5) × 12 = 2.4

Mean Squared Error (MSE):

MSE = (1/n) · Σ (yᵢ − ŷᵢ)² = (1/5) × 40 = 8

Root Mean Squared Error (RMSE):

RMSE = √MSE = √8 ≈ 2.828

Residual Sum of Squares (RSS):

RSS = Σ (yᵢ − ŷᵢ)² = 40

MSE penalises larger errors more heavily (squared) — sensitive to outliers. RMSE has the same units as the target.

Week 3 — Logistic Regression: Credit-Card Upgrade Prediction

Setup (from PDF)

A credit-card company wants to identify cardholders likely to upgrade from a Standard to a Premium card. Dataset: 30 cardholders with features:

  • x₁ = Total credit-card purchases in the last year
  • x₂ = Whether the customer bought an add-on card (1 = Yes, 0 = No)
  • Target y ∈ {0 = stay on Standard, 1 = upgrade to Premium}

Prediction Steps

  1. Train a logistic regression to learn weights w₀, w₁, w₂.
  2. For a new applicant, compute z = w₀ + w₁·x₁ + w₂·x₂.
  3. Apply sigmoid: hw(x) = 1 / (1 + e−z)
  4. If hw(x) ≥ 0.5 → predict "upgrade" (Class 1); else "stay" (Class 0).

Cost function (single observation):

If y=1: Cost = −log(hw(x)) If y=0: Cost = −log(1 − hw(x))

Gradient descent update:

wj := wj − α · Σi(hw(xᵢ) − yᵢ) · xᵢⱼ

  1. Compute target entropy of "Buy" across all 14 records.
  2. For each attribute, partition data by attribute value, compute child entropies, then the weighted average.
  3. Information Gain = Entropy(parent) − Entropyweighted(child).

Week 5 — Bayes' Theorem: Medical Diagnosis (Cancer Test)

Given Data

  • P(Cancer) = 0.008; P(No Cancer) = 0.992
  • True positive rate (test+ given Cancer) = 0.98 → False negative = 0.02
  • True negative rate (test− given No Cancer) = 0.97 → False positive = 0.03

The test returns positive. What's the more likely hypothesis?

Solution

Bayes' theorem: P(H | D) ∝ P(D | H) · P(H)

Numerator for Cancer:

P(Positive | Cancer) · P(Cancer) = 0.98 × 0.008 = 0.0078

Numerator for No Cancer:

P(Positive | No Cancer) · P(No Cancer) = 0.03 × 0.992 ≈ 0.0298

Conclusion: 0.0298 > 0.0078 → the more likely hypothesis is No Cancer (despite the positive test, because cancer is rare).

This illustrates why prior probability matters: a positive test for a rare disease can still favour "no disease" via base-rate effects.

Week 5 — Naive Bayes Classifier: Predicting Computer Buyers

Dataset (14 prior customers)

Same Age / Income / Student / Credit_rating attributes as the ID3 example. Target: buys_computer ∈ {Yes, No}.

Naive Bayes Solution Sketch

  1. Compute priors: P(Yes) and P(No) from class frequencies.
  2. Compute conditional probabilities for each attribute value given each class — assuming attributes are conditionally independent given the class:
    P(x | c) = P(x₁ | c) · P(x₂ | c) · … · P(xm | c)
  3. For a new tuple, compute:
    P(c | x) ∝ P(c) · ∏ P(xⱼ | c)
    for each class, and pick the class with the highest posterior.

Week 6 — K-Means++ Smart Initialisation

Dataset (1-D): [1, 3, 10, 15, 35, 50, 95, 100]

Goal: pick 3 initial centroids using K-Means++.

Steps

  1. Pick the first centroid randomly — say, 1.
  2. Compute the distance of each remaining point from the nearest selected centroid:
    Point31015355095100
    Distance from {1}291434499499
  3. Pick the next centroid proportional to squared distance — the farthest point (100) has the highest probability. Choose 100.
  4. Now compute min distance to {1, 100} for each remaining point; pick the next centroid (likely 50, in the middle).
  5. Final initial centroids: {1, 50, 100} — well-spaced. Continue standard K-Means.

K-Means++ reduces variability across runs and improves clustering quality vs. random initialisation.

Week 9 — Backpropagation in a Feedforward Network

Setup (from PDF Section 3 — Backprop numerical example)

A multilayer feedforward network with input layer, one hidden layer, and an output layer. Forward-pass with given weights and inputs, compute error, then back-propagate to update weights.

Five steps of backpropagation

  1. Forward pass: compute the activation a(l) = σ(W(l)a(l−1) + b(l)) layer by layer, ending in the output ŷ.
  2. Compute loss — e.g., squared error (½(y − ŷ)²) or cross-entropy.
  3. Output-layer error: δ(L) = (ŷ − y) ⊙ σ'(z(L))
  4. Back-propagate to hidden layers: δ(l) = (W(l+1))T·δ(l+1) ⊙ σ'(z(l))
  5. Weight update (gradient descent):
    W(l) := W(l) − η · δ(l) · (a(l−1))T

The PDF works through the XNOR-function multilayer example to show that a single perceptron cannot learn XNOR (non-linear), but a feedforward net with a hidden layer can.
