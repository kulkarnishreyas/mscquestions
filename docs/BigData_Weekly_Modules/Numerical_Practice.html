Week 2 — HDFS Block & Replication Math

Default block size 128 MB · default replication factor 3 (both stated in the Week 2 slides)

Q2.1 — Blocks, copies & total storage APPLIED

A 1 GB (1024 MB) file is written to HDFS with the default block size of 128 MB and replication factor 3.

Solution

blocks = ⌈1024 / 128⌉ = 8 copies = 8 × 3 = 24 storage = 24 × 128 MB = 3072 MB = 3 GB

Replication triples the raw storage footprint — a key capacity-planning fact.

Q2.2 — Fault tolerance APPLIED

With the default replication factor of 3, how many DataNodes holding a given block can fail simultaneously while the block is still available?

Solution

tolerated failures = RF − 1 = 3 − 1 = 2

As long as ≥1 replica survives the block is readable; RF=3 keeps data safe through 2 node losses.

Week 3 — MapReduce Combiner Effect

Local aggregation reduces the pairs shuffled across the network

Q3.1 — Pairs emitted with vs without a combiner APPLIED

Three mappers process one line each: M1 = Deer Bear River, M2 = Car Car River, M3 = Deer Car Bear. Each mapper emits (word, 1) per token.

Solution

Without combiner: 3 + 3 + 3 = 9 pairs (one per token) With combiner (aggregate locally per mapper): M1 Deer,Bear,River → 3 pairs M2 Car:2, River:1 → 2 pairs ← the two "Car"s merge M3 Deer,Car,Bear → 3 pairs total = 8 pairs

The combiner shrank the shuffle from 9 → 8 pairs; on real data with many repeats the saving is far larger.

Week 4 — MapReduce Word Count

Worked example straight from the Week 4 slides (distinct from the cat/bat/rat one)

Q4.1 — Trace the Word Count FROM PDF

Input lines: Deer Bear River / Car Car River / Deer Car Bear.

Solution

Map emits 9 pairs (3 per line). Shuffle groups by key: Bear → [1,1] Car → [1,1,1] Deer → [1,1] River → [1,1] Reduce sums: Bear=2, Car=3, Deer=2, River=2

Week 6 — HBase Shell

Scan ranges & row keys from the Week 6 shell examples

Q6.1 — Rows returned by a range scan APPLIED

The Week 6 slide shows scan 'users', {STARTROW => 'user100', STOPROW => 'user200'}. Row keys are user100, user101, …. STARTROW is inclusive and STOPROW is exclusive. How many rows does the scan return?

Solution

user100 … user199 inclusive (user200 excluded) = 200 − 100 = 100 rows

(A separate slide uses {LIMIT => 10}, which would instead cap the result at 10 rows.)

Week 7 — MongoDB Replica Sets

Election majority & failover (slides state members = 3 or 5, "majority", odd numbers)

Q7.1 — Election majority & failover APPLIED

A replica set has 5 members. A new primary needs a majority of votes.

majority = ⌊N/2⌋ + 1 failures tolerated = N − majority

Solution

majority = ⌊5/2⌋ + 1 = 2 + 1 = 3 failures = 5 − 3 = 2

This is why odd member counts (3, 5, 7) are recommended — an even count adds a node without raising the failures tolerated, and risks split-vote ties.

Week 9 — Cassandra Data Types

Numeric type bit-widths listed on the Week 9 slide

Q9.1 — Type sizes FROM PDF

The Week 9 slide lists: int = 32-bit, bigint = 64-bit, float = 32-bit, double = 64-bit.

How many bits does a float occupy?

Which type would you choose to store a 64-bit whole number (e.g. a large event ID)?

Solution

float = 32 bits. A 64-bit whole number needs bigint (64-bit integer); int is only 32-bit, and double is 64-bit but floating-point, not an exact integer.

Week 10 — Cassandra Replication & Consistency

Multi-datacentre keyspace & tunable consistency · sensor CSV (all from Week 10 slides)

Q10.1 — Multi-datacentre replication FROM PDF

The Week 10 slide creates a keyspace with NetworkTopologyStrategy, DC1 = 3 and DC2 = 2.

Solution

total replicas = 3 + 2 = 5 QUORUM = ⌊(sum of RF)/2⌋ + 1 = ⌊5/2⌋ + 1 = 3

Q10.2 — Tunable consistency (R + W > RF) APPLIED

A keyspace uses SimpleStrategy with RF = 3. Cassandra gives strong consistency when R + W > RF.

QUORUM = ⌊RF/2⌋ + 1

If both reads and writes use QUORUM (R = 2, W = 2), is strong consistency guaranteed?

Solution

QUORUM = ⌊3/2⌋ + 1 = 2 R + W = 2 + 2 = 4 > 3 = RF → strong consistency ✓

The read and write quorums are guaranteed to overlap on at least one node, so a read always sees the latest write.

Q10.3 — Sensor readings FROM PDF

The COPY-FROM slide imports two readings: 25.5 at 10:00:00 and 26.1 at 10:01:00. What is the average temperature?

Solution

avg = (25.5 + 26.1) / 2 = 51.6 / 2 = 25.8 (rate of change = (26.1 − 25.5) / 1 min = 0.6 °/min)

Week 11 — K-Means (MapReduce) Assignment Step

The Map phase assigns each point to its nearest centroid by Euclidean distance

Q11.1 — Assign a point to a cluster APPLIED

Point P = (1, 1). Centroids C1 = (2, 2) and C2 = (5, 5).

d = √((x₁−x₂)² + (y₁−y₂)²)

Which cluster does the mapper emit P to?

Solution

d(P,C1) = √((1−2)² + (1−2)²) = √2 = 1.414 d(P,C2) = √((1−5)² + (1−5)²) = √32 = 5.657 1.414 < 5.657 → emit (C1, P)

In MapReduce K-Means the mapper emits (nearest_centroid_id, point); the reducer then averages all points per centroid to get the new centroid.

Week 12 — Association Rule Mining

Support, confidence & lift — the three metrics named on the Week 12 Apriori slide

Q12.1 — Support, confidence & lift APPLIED

Five market-basket transactions:

TIDItems
T1Bread, Milk
T2Bread, Milk
T3Bread, Milk
T4Bread
T5Butter
support(X) = count(X) / N confidence(A→B) = support(A∪B) / support(A) lift(A→B) = confidence(A→B) / support(B)

Solution

N = 5 Bread in T1,T2,T3,T4 → support(Bread) = 4/5 = 0.8 Milk in T1,T2,T3 → support(Milk) = 3/5 = 0.6 {Bread,Milk} in T1,T2,T3 → support = 3/5 = 0.6 confidence(Bread→Milk) = 0.6 / 0.8 = 0.75 lift(Bread→Milk) = 0.75 / 0.6 = 1.25

Lift = 1.25 > 1 → buying bread makes milk more likely than its baseline rate (a positive association worth a rule).

Answer: C. RF − 1 = 3 − 1 = 2 failures tolerated (one replica must survive).

Answer: C. Replication triples the footprint: 1 GB × 3 = 3 GB (8 blocks × 3 × 128 MB = 3072 MB).

Answer: C. "Car" appears twice in line 2 and once in line 3 → 3.

2 MARKS Three mappers emit 9 (word, 1) pairs in total for that input. With a per-mapper combiner, how many pairs are shuffled?

Answer: B. The combiner merges mapper 2's two "Car" pairs into (Car,2): 3 + 2 + 3 = 8 pairs.

2 MARKS scan 'users', {STARTROW => 'user100', STOPROW => 'user200'} (STOPROW exclusive). Rows returned?

Answer: B. user100 … user199 inclusive (user200 excluded) = 100 rows.

2 MARKS A MongoDB replica set has 5 members. How many votes form the majority needed to elect a primary?

Answer: B. majority = ⌊5/2⌋ + 1 = 3 (so a 5-member set tolerates 2 node failures).

2 MARKS A Cassandra keyspace has RF = 3. The QUORUM consistency level equals how many replicas?

Answer: B. QUORUM = ⌊RF/2⌋ + 1 = ⌊3/2⌋ + 1 = 2.

2 MARKS A keyspace uses NetworkTopologyStrategy with DC1 = 3 and DC2 = 2. How many replicas of each row exist in total?

Answer: C. Replicas are summed across data centres: 3 + 2 = 5.

2 MARKS Given support(Bread) = 0.8, support(Milk) = 0.6 and support{Bread,Milk} = 0.6, the lift of the rule Bread → Milk is:

Answer: D. confidence = 0.6/0.8 = 0.75; lift = confidence/support(Milk) = 0.75/0.6 = 1.25 (>1 → positive association).

2 MARKS In Cassandra, choosing read and write consistency levels so that R + W > RF guarantees:

Answer: B. R + W > RF forces the read and write quorums to overlap on ≥1 node, so a read always sees the latest write — strong consistency.

2 MARKS Why are odd numbers of members (3, 5, 7) recommended for a replica set / quorum cluster?

Answer: B. An even count can split votes 50/50 (no majority) and adds a node without raising fault tolerance — odd counts always yield a decisive majority.
