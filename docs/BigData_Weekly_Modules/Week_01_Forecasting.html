1️⃣ Big Data: An Introduction

What is Big Data?

Big Data refers to vast and complex datasets that are difficult to process and analyse using traditional data management tools and techniques. These datasets are characterised by their high volume, velocity, and variety.

💡 Core idea: Big Data isn't just "lots of data" — it's data whose scale, speed, or structure exceeds what conventional relational databases and spreadsheet tools can handle.

Evolution of Big Data

The concept of Big Data has evolved over time, driven by the exponential growth of digital information and the development of new technologies for data collection, storage, and analysis — sensors, mobile devices, social media, IoT, cloud computing.

Types of Data

🗂️ Structured

Highly organised, fixed schema, easily searchable — typically stored in relational databases (e.g., transactions, spreadsheets).

📄 Semi-structured

Some organisation but no rigid schema — e.g., JSON, XML, log files.

🖼️ Unstructured

No predefined format — text documents, images, audio, video, social-media posts. The majority of new data.

Sources of Big Data

📱 Social Media

Tweets, posts, likes, shares — high-velocity, unstructured user-generated content

🌐 Sensor Networks / IoT

Continuous telemetry from devices, smart meters, vehicles, factories

💳 Business Transactions

Banking, retail, e-commerce records — structured, high volume

📜 Server / Application Logs

Machine-generated traces, clickstreams, audit trails

2️⃣ The Vs of Big Data — Characteristics

The characteristics of Big Data are most commonly described by the 3 Vs — Volume, Velocity, and Variety — with two more (Veracity and Value) often added.

📦 Volume

The sheer scale of data — Terabytes → Petabytes → Exabytes generated and stored every day.

⚡ Velocity

The speed at which data is generated and must be processed — real-time and near-real-time streams.

🎭 Variety

The different forms of data — structured, semi-structured, and unstructured combined.

✓ Veracity

The accuracy and trustworthiness of data — noise, uncertainty, completeness.

💎 Value

The insights and benefits that can be derived — data without value is just storage cost.

5️⃣ Phases of the Analytics Cycle

The analytics cycle transforms raw data into actionable insights. It's iterative — results from one phase often send you back to an earlier one.

The 6 Classical Phases

① Discovery
→
② Data Preparation
→
③ Model Planning
④ Model Building
→
⑤ Communicate Results
→
⑥ Operationalize
  1. Discovery — understand the business problem, identify relevant data sources, explore initial data, define goals.
  2. Data Preparation — clean, transform, handle missing values, correct errors, convert formats.
  3. Model Planning — pick the analysis type (descriptive / predictive / prescriptive) and choose algorithms or tools.
  4. Model Building — apply the chosen models, train, experiment with parameters, optimise performance.
  5. Communicate Results — visualise, report, and present findings to stakeholders.
  6. Operationalize — deploy the model into production systems and business processes for real decisions.
💡 Iterative loop: Insights from "Communicate Results" frequently lead to refining the model or collecting new data — analytics is rarely a one-shot process.

8️⃣ Big Data Technologies: Hadoop (Overview)

Hadoop is an open-source framework for storing and processing very large datasets across clusters of commodity hardware. It is purpose-built to handle Big Data's volume, velocity, variety, and veracity. (Deep dive in Week 2.)

Key Features of Hadoop

🌐 Distributed Storage

Data spread across many nodes — scalable & fault-tolerant

⚙️ Parallel Processing

Compute runs in parallel across nodes — faster execution

📈 Scalability

Add more nodes → handle more data

🛡️ Fault Tolerance

Block-level replication ensures durability if nodes fail

🎭 Flexibility

Handles structured, semi-structured, and unstructured data

Brief History

Inspired by Google's MapReduce and Google File System (GFS) papers. Doug Cutting and Mike Cafarella started Hadoop in 2005 as part of the Nutch project. Became a top-level Apache project in 2008.

HDFS — The Storage Component

The Hadoop Distributed File System stores large files across many nodes using a master-slave architecture: a NameNode holds metadata; DataNodes store the actual data blocks. Default block size 128 MB, replication factor 3.

Hadoop Ecosystem

⚙️ YARN

Resource manager — schedules jobs across the cluster

🔀 MapReduce

Original parallel processing programming model

🐝 Hive

SQL-like warehousing over HDFS data

🐷 Pig

High-level data-flow language (Pig Latin)

📊 HBase

NoSQL column store on top of HDFS

Advantages of Hadoop

  • Cost-effective — runs on commodity hardware
  • Scalable — easily handles growing data volumes
  • Flexible — varied data formats
  • Fault-tolerant — replication ensures durability and availability

Applications of Hadoop

🏛️ Data Warehousing

Large-scale warehouses for historical data

🔄 ETL Processing

Extract / Transform / Load on big data

🤖 Machine Learning

Train/deploy ML on massive datasets

📜 Log Processing

Server/app/network logs at scale

📱 Social Media Analytics

User behaviour & sentiment

📝 Summary & Exam Tips

🎯 Key Takeaways

  • Big Data = vast, complex datasets that exceed traditional tools — characterised by high Volume, Velocity, Variety (+ Veracity, Value).
  • Three data types: Structured (RDBMS, tables), Semi-structured (JSON, XML, logs), Unstructured (text, images, video).
  • Challenges: Storage, Processing, Analysis, Visualisation, Variety/Veracity, Security & Privacy.
  • Applications: Retail, Healthcare, Finance, Manufacturing, Telecom, Government, Education.
  • 6-phase analytics cycle: Discovery → Data Preparation → Model Planning → Model Building → Communicate Results → Operationalize. The cycle is iterative.
  • Roles: Data Scientist, Data Engineer, Data Analyst, Business Analyst, Project Manager (+ DBA, Domain Expert).
  • Analytics methods spectrum: Descriptive (what) → Diagnostic (why) → Predictive (what next) → Prescriptive (what to do) → Cognitive (human-like reasoning).
  • Hadoop = open-source framework for distributed storage (HDFS) + parallel processing on commodity hardware. Inspired by Google's GFS + MapReduce papers (2003–2004).
  • HDFS: NameNode (metadata) + DataNodes (data blocks). Default block 128 MB, replication factor 3.
  • Hadoop ecosystem: YARN, MapReduce, Hive, Pig, HBase, Spark, Flume, Sqoop, ZooKeeper.
  • Hadoop applications: Data warehousing, ETL, ML, log processing, social-media analytics.
📌 Common Exam Pitfalls:
  • Confusing the 3 data types — JSON/XML are semi-structured, not unstructured
  • Listing only 3 Vs when 5 are commonly referenced (Veracity, Value often added)
  • Saying NameNode stores actual data — it stores only metadata
  • Forgetting that the analytics cycle is iterative, not linear
  • Confusing Cognitive (human-like over unstructured) with Predictive (statistical/ML over historical)
