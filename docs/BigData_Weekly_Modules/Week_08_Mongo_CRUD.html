1️⃣ JSON Refresher

JSON (JavaScript Object Notation) = lightweight, text-based data interchange format.

Structure

📦 Objects

Unordered key/value pairs: {"k": "v"}

📋 Arrays

Ordered values: [1, 2, 3]

JSON Data Types

TypeExample
String"hello"
Number123, 45.67
Booleantrue, false
Array[1, 2, 3]
Object{"name": "Alice"}
nullnull

Full Example

{ "name": "Alice", "age": 30, "active": true, "skills": ["Python", "SQL", "MongoDB"], "address": { "city": "Vellore", "country": "India" }, "manager": null }
💡 Why JSON for MongoDB? MongoDB documents look like JSON. Drivers in any language can naturally serialize/deserialize between language objects and MongoDB.

2️⃣ Import / Export

Tools

ToolPurpose

Importing JSON

# Basic JSON import mongoimport --db mydb --collection users --file users.json # JSON array file (multiple docs in one array) mongoimport --db mydb --collection users --file users.json --jsonArray # Drop existing collection first mongoimport --db mydb --collection users --file users.json --drop # Upsert (update if exists, insert if new) mongoimport --db mydb --collection users --file users.json --mode upsert

Importing CSV

mongoimport --db mydb --collection products \ --type csv --file products.csv --headerline # Specify columns manually mongoimport --db mydb --collection products \ --type csv --fields name,price,quantity --file products.csv

Exporting

# Export whole collection as JSON mongoexport --db mydb --collection users --out users.json # Pretty-print mongoexport --db mydb --collection users --out users.json --pretty # Export with query (filter) mongoexport --db mydb --collection users \ --query '{"status":"active"}' --out active_users.json # Export specific fields as CSV mongoexport --db mydb --collection users \ --type=csv --fields name,age,email --out users.csv

Backup & Restore

# Backup whole database mongodump --db mydb --out /backup/ # Restore from backup mongorestore --db mydb /backup/mydb

3️⃣ CRUD Operations

CRUD = Create, Read, Update, Delete — the 4 basic data operations.

CREATE — Insert Documents

// Insert single document db.users.insertOne({ name: "Bob", age: 25, email: "bob@example.com" }) // Insert multiple documents db.users.insertMany([ { name: "Carol", age: 30 }, { name: "Dan", age: 22 }, { name: "Eve", age: 28 } ]) // Note: _id is auto-generated if not provided

READ — Find Documents

// Find all db.users.find() // Find with filter db.users.find({ age: { $gt: 25 } }) // Find one db.users.findOne({ name: "Bob" }) // Combined filter db.users.find({ status: "active", age: { $lt: 30 } }) // Projection (which fields) db.users.find({}, { name: 1, age: 1, _id: 0 }) // Sort + limit db.users.find().sort({ age: -1 }).limit(5) // Count db.users.countDocuments({ age: { $gte: 18 } })

Query Operators

OperatorMeaning
$eq, $neequal, not equal
$gt, $gte, $lt, $ltegreater/less than
$in, $ninin / not in array
$and, $or, $not, $norlogical
$existsfield exists
$regexpattern matching

UPDATE — Modify Documents

// Update one (first match) db.users.updateOne( { name: "Bob" }, // filter { $set: { age: 26 } } // update ) // Update many db.users.updateMany( { status: "pending" }, { $set: { status: "active" } } ) // Increment a field db.users.updateOne( { name: "Bob" }, { $inc: { points: 10 } } ) // Replace entire document db.users.replaceOne( { name: "Bob" }, { name: "Bob", age: 27, email: "bob_new@example.com" } ) // Upsert (insert if not found) db.users.updateOne( { name: "Frank" }, { $set: { age: 35 } }, { upsert: true } )

Update Operators

OperatorPurpose
$setSet field value
$unsetRemove field
$incIncrement number
$pushAppend to array
$pullRemove from array
$addToSetAdd to array if not exists
$renameRename a field

DELETE — Remove Documents

// Delete one db.users.deleteOne({ name: "Bob" }) // Delete many db.users.deleteMany({ status: "inactive" }) // Delete all (dangerous!) db.users.deleteMany({}) // Drop collection (faster than deleteMany({}) for large collections) db.users.drop()

4️⃣ Database & Collection Management

Database Operations

// List all databases show dbs // (or show databases) // Switch to / create database use mydb // (DB is actually created upon first data insertion) // Show current database db // Drop current database (irreversible!) db.dropDatabase()

Collection Operations

// List collections show collections // Create collection explicitly db.createCollection("users") // Create with options (capped, max size) db.createCollection("logs", { capped: true, size: 100000, max: 1000 }) // Drop collection db.users.drop() // Rename collection db.users.renameCollection("active_users")

5️⃣ Finding Documents — Detailed

Basic find()

// All docs (returns cursor) db.users.find() // Pretty-print db.users.find().pretty() // First matching doc db.users.findOne({ name: "Alice" })

Filter (Query Document)

// Exact match db.users.find({ status: "active" }) // Multiple conditions (implicit AND) db.users.find({ status: "active", age: { $gt: 18 } }) // Explicit AND db.users.find({ $and: [{ status: "active" }, { age: { $gt: 18 } }] }) // OR db.users.find({ $or: [{ status: "active" }, { vip: true }] }) // Array contains db.users.find({ tags: "admin" }) // Nested field db.users.find({ "address.city": "Vellore" })

Projection (Select Fields)

// Include only name and email db.users.find({}, { name: 1, email: 1 }) // Exclude _id db.users.find({}, { name: 1, email: 1, _id: 0 }) // Exclude specific field db.users.find({}, { password: 0 })

Sort, Limit, Skip

// Sort ascending db.users.find().sort({ age: 1 }) // Sort descending db.users.find().sort({ age: -1 }) // Limit + skip (pagination) db.users.find().sort({ age: -1 }).skip(20).limit(10) // → Skip first 20, return next 10

Common Query Operators

// Comparison db.users.find({ age: { $gt: 18, $lte: 65 } }) // In a list db.users.find({ role: { $in: ["admin", "moderator"] } }) // Not in db.users.find({ status: { $nin: ["deleted", "banned"] } }) // Field exists db.users.find({ email: { $exists: true } }) // Regex db.users.find({ name: { $regex: "^Al", $options: "i" } }) // Array size db.users.find({ skills: { $size: 3 } }) // Array element match db.users.find({ scores: { $elemMatch: { $gt: 80, $lt: 100 } } })

6️⃣ Aggregation Pipeline

The Aggregation Pipeline is MongoDB's powerful framework for data processing. Documents flow through a sequence of stages, each transforming them.

Pipeline Concept

$match
→
$group
→
$sort
→
$project
→
$limit

Common Stages

StagePurpose
$matchFilter documents (like find filter)
$groupGroup by key + apply accumulators
$sortOrder results
$projectReshape (select, rename, compute fields)
$limit / $skipPagination
$unwindDeconstruct array → one doc per element
$lookupLeft outer join with another collection
$countCount remaining documents

Accumulator Operators (in $group)

OperatorPurpose
$sumSum of values (use $sum: 1 to count)
$avgAverage
$min / $maxMin/Max value
$first / $lastFirst/last value in group
$pushArray of all values
$addToSetArray of unique values

Worked Examples

Example 1: Total quantity & avg price per item type

db.orders.aggregate([ { $match: { status: "completed" } }, // Stage 1: filter { $group: { // Stage 2: group _id: "$itemType", totalQty: { $sum: "$quantity" }, avgPrice: { $avg: "$price" } }} ])

Example 2: Top 5 customers by total spend

db.orders.aggregate([ { $group: { _id: "$customer_id", totalSpent: { $sum: "$amount" }, orderCount: { $sum: 1 } }}, { $sort: { totalSpent: -1 } }, { $limit: 5 } ])

Example 3: Sales per month with projection

db.orders.aggregate([ { $group: { _id: { year: { $year: "$date" }, month: { $month: "$date" } }, revenue: { $sum: "$amount" } }}, { $project: { _id: 0, period: { $concat: [ { $toString: "$_id.year" }, "-", { $toString: "$_id.month" } ]}, revenue: 1 }}, { $sort: { period: 1 } } ])

Example 4: $unwind + $lookup (join)

db.orders.aggregate([ // Look up customer info { $lookup: { from: "customers", localField: "customer_id", foreignField: "_id", as: "customer" }}, { $unwind: "$customer" }, // flatten the array { $project: { orderId: "$_id", customerName: "$customer.name", amount: 1 }} ])

7️⃣ MapReduce in MongoDB

MongoDB also supports MapReduce as an alternative aggregation method (older, more flexible but slower than aggregation pipeline).

Two Phases

① Map

Processes each input document. Emits one or more (key, value) pairs. Written in JavaScript.

② Reduce

Takes the key + list of values. Combines them. Produces final output. Also JavaScript.

Example

db.orders.mapReduce( // Map function function() { emit(this.customer_id, this.amount); }, // Reduce function function(key, values) { return Array.sum(values); }, { out: "total_per_customer" // output collection } )
💡 When to use which?
  • Aggregation Pipeline → faster, simpler, recommended for most cases
  • MapReduce → only for complex custom aggregations not expressible via pipeline

8️⃣ Indexes

Indexes are special data structures that store a small portion of data in an easy-to-traverse form. They dramatically speed up queries.

Why Indexes Matter

Without an index, MongoDB must scan EVERY document in the collection (collection scan). With an index, it jumps directly to matching documents.

Automatic Index

MongoDB automatically creates a unique index on _id for every collection.

Index Types

TypeUse Case
Single FieldIndex on one field
CompoundIndex on multiple fields (order matters!)
MultikeyIndex on array elements
TextFull-text search on strings
Geospatial (2dsphere, 2d)Location queries (proximity, geometry)
HashedFor sharding via hash partitioning
TTLAuto-delete documents after expiration

Creating Indexes

// Single field (ascending) db.users.createIndex({ email: 1 }) // Single field (descending) db.users.createIndex({ created_at: -1 }) // Compound index db.users.createIndex({ city: 1, age: -1 }) // Unique index (no duplicates allowed) db.users.createIndex({ email: 1 }, { unique: true }) // Text index db.products.createIndex({ name: "text", description: "text" }) // Geospatial db.places.createIndex({ location: "2dsphere" }) // TTL — auto-delete after 30 days db.sessions.createIndex({ created_at: 1 }, { expireAfterSeconds: 2592000 })

Inspecting Indexes

// List indexes db.users.getIndexes() // Drop a specific index db.users.dropIndex("email_1") // Drop all indexes (except _id) db.users.dropIndexes() // Explain query (see if index is used) db.users.find({ email: "alice@example.com" }).explain("executionStats")

Trade-offs

✅ Pros

  • Massively faster reads / queries
  • Sort operations don't require in-memory sort
  • Unique indexes enforce data integrity

❌ Cons

  • Writes slower (must update index too)
  • Consumes additional disk space
  • Index for every query is overkill

Index Strategy

  • Index fields used in find filters, sort, $match
  • For compound indexes, put equality fields first, then sort, then range
  • Use .explain() to verify indexes are being used
  • Don't over-index — every write incurs cost

📝 Summary & Exam Tips

🎯 Key Takeaways

  • JSON: text format with String, Number, Bool, Array, Object, null types
  • CRUD: insertOne/Many, find/findOne, updateOne/Many, deleteOne/Many
  • Query operators: $gt, $lt, $in, $and, $or, $exists, $regex
  • Update operators: $set, $inc, $push, $pull, $addToSet, $unset
  • Aggregation pipeline: $match → $group → $sort → $project → $limit (any combination)
  • Accumulators: $sum, $avg, $min, $max, $push, $addToSet
  • MapReduce: Map (emit) + Reduce (combine) — JS functions, slower than pipeline
  • Indexes: single, compound, multikey, text, geospatial, hashed, TTL
  • Trade-off: indexes accelerate reads but slow writes & cost storage
📌 Common Exam Pitfalls:
  • Using {age > 25} instead of {age: {$gt: 25}} (MongoDB syntax!)
  • Forgetting updateOne requires update operators ($set, etc.) — not raw object
  • Confusing $set (modify field) vs $inc (add to number)
  • Thinking aggregation is just for SQL-like SELECT — it can transform, join, even output to collections
  • Over-indexing: every index costs writes & storage
