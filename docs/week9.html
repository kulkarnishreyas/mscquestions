JavaServer Pages (JSP)

Dynamic web pages with Java — scripting elements, directives, implicit objects, Expression Language, and complete MVC applications

Introduction to JSP

JavaServer Pages (JSP) is a server-side technology for creating dynamic web content. A JSP file is essentially an HTML page with embedded Java code. The server translates the JSP into a Servlet, compiles it, and executes it — returning pure HTML to the browser.

Key Insight: JSP and Servlets do the same thing — generate dynamic web pages. JSP is just easier to write when you have lots of HTML with a little Java. A Servlet is better when you have lots of Java with a little HTML.

JSP

HTML-centric. Java embedded via special tags. Good for View layer (presentation).

Servlet

Java-centric. HTML written via out.println(). Good for Controller layer (logic).

MVC Pattern

Use Servlet as Controller + JSP as View + JavaBean as Model — the industry standard.

JSP — hello.jsp (your first JSP)
<!DOCTYPE html>
<html>
<body>
  <h2>Hello from JSP!</h2>

  <%-- JSP Comment (not sent to browser) --%>

  <%-- Scriptlet: Java code executed on server --%>
  <%
    String name = "World";
    int year = 2024;
  %>

  <p>Hello, <%= name %>! It is <%= year %>.</p>

</body>
</html>
Browser receives: <!DOCTYPE html> <html> <body> <h2>Hello from JSP!</h2> <p>Hello, World! It is 2024.</p> </body> </html>

JSP Lifecycle — 7 Steps

When a browser first requests a JSP, the server goes through 7 phases. After the first request, only step 6 repeats for each subsequent request.

1
Translation — The JSP container (Tomcat) converts your .jsp file into a .java Servlet source file. This happens only once (or when the file changes).
2
Compilation — The generated .java file is compiled to a .class bytecode file using javac.
3
Loading — The compiled Servlet class is loaded into the JVM's classloader.
4
Instantiation — The container creates an instance (object) of the generated Servlet class.
5
Initialization — jspInit() is called once. Override this for one-time setup (DB connections, config loading).
6
Servicing — _jspService() is called for every HTTP request. This is where your page logic runs. You cannot override this method.
7
Destroy — jspDestroy() is called when the container removes the JSP (server shutdown, redeployment). Override for cleanup.
First request is slow, subsequent requests are fast — Steps 1-5 only happen once. For every request after that, only step 6 runs, making JSP nearly as fast as a pre-compiled Servlet.

Scripting Elements

Scripting elements let you embed Java code directly inside a JSP page. There are three types:

Scriptlet

<% code %>

Executes Java code on each request. Variables declared here are local to the _jspService() method.

Expression

<%= expr %>

Evaluates a Java expression and outputs its String value. No semicolon! Equivalent to out.print(expr).

Declaration

<%! var/method %>

Declares class-level variables and methods. Shared across all requests (thread-safety concern!).

JSP — scriptingDemo.jsp
<%@ page import="java.util.Date" %>
<!DOCTYPE html><html><body>

<%-- DECLARATION: class-level variable and method --%>
<%!
  int visitCount = 0;  // shared across requests!

  String greet(String name) {
    return "Welcome, " + name + "!";
  }
%>

<%-- SCRIPTLET: code runs on each request --%>
<%
  visitCount++;  // increments each visit
  String user = request.getParameter("name");
  if (user == null) user = "Guest";
%>

<%-- EXPRESSION: outputs a value inline --%>
<h2><%= greet(user) %></h2>
<p>Page visited <%= visitCount %> times.</p>
<p>Server time: <%= new Date() %></p>

<%-- Conditional HTML using scriptlet --%>
<% if (visitCount > 5) { %>
  <p style="color:red">You visit too much!</p>
<% } %>

</body></html>
With ?name=Alice, visitCount=3: <h2>Welcome, Alice!</h2> <p>Page visited 3 times.</p> <p>Server time: Thu May 21 10:30:00 IST 2026</p>
Scriptlet vs Expression comparison:
<% out.print("Hello " + name); %> — scriptlet form
<%= "Hello " + name %> — expression form (same output, cleaner)
Always prefer expressions or EL over scriptlets for output.

JSP Directives

Directives give instructions to the JSP container at translation time. They affect how the JSP is converted to a Servlet. Syntax: <%@ directive attribute="value" %>

page Directive

<%@ page ... %>

Controls page-level settings: imports, content type, session, error handling, buffering.

include Directive

<%@ include ... %>

Statically includes another file's content at translation time. The included file becomes part of the same Servlet.

taglib Directive

<%@ taglib ... %>

Declares a custom tag library (like JSTL) so you can use its tags in the JSP.

page Directive Attributes

JSP Action Tags

Action tags are XML-style tags that perform specific server-side actions. Unlike directives (which work at translation time), action tags execute at request time.

Action TagPurposeExample
<jsp:include>Dynamically includes another resource at runtime<jsp:include page="header.jsp" />
<jsp:forward>Forwards the request to another resource (browser URL doesn't change)<jsp:forward page="result.jsp" />
<jsp:useBean>Instantiates or retrieves a JavaBean<jsp:useBean id="u" class="User" scope="session" />
<jsp:setProperty>Sets a property on a JavaBean<jsp:setProperty name="u" property="name" value="Alice" />
<jsp:getProperty>Gets and prints a bean property<jsp:getProperty name="u" property="name" />
<jsp:param>Passes parameters with include/forward<jsp:param name="msg" value="Hello" />
JSP — useBean/setProperty/getProperty
<!-- User.java must be a JavaBean: no-arg constructor, getters/setters -->
<%@ page contentType="text/html" %>

<%-- Create or retrieve User bean from session scope --%>
<jsp:useBean id="user" class="com.demo.User"
             scope="session" />

<%-- Automatically map all request params to bean properties --%>
<jsp:setProperty name="user" property="*" />

<html><body>
<h2>Hello, <jsp:getProperty name="user"
                         property="name" />!</h2>
<p>Email: <jsp:getProperty name="user"
                       property="email" /></p>
</body></html>
Request: POST /profile.jsp?name=Alice&email=alice@example.com <h2>Hello, Alice!</h2> <p>Email: alice@example.com</p>
include vs forward:
<%@ include file="h.jsp" %> — static, happens at translation time, becomes one Servlet
<jsp:include page="h.jsp" /> — dynamic, happens at request time, separate Servlet call

Expression Language (EL)

EL provides a simpler, cleaner way to access data stored in beans, request params, session, and application scope — without writing Java scriptlets. Syntax: ${expression}

Without EL (scriptlet)

<%
  User u = (User) session.getAttribute("user");
%>
<p>Hello, <%= u.getName() %></p>

With EL (clean)

<p>Hello, ${sessionScope.user.name}</p>

<!-- or simply: -->
<p>Hello, ${user.name}</p>

A full working example using Servlet (Controller) + JSP (View) + JavaBean (Model). This is the standard enterprise pattern.

// Model — JavaBean
public class User {
    private String username, password;

    // No-arg constructor required for JavaBean
    public User() {}

    public User(String u, String p) {
        username = u; password = p;
    }

    public String getUsername() { return username; }
    public void setUsername(String u) { username = u; }
    public String getPassword() { return password; }
    public void setPassword(String p) { password = p; }

    public boolean isValid() {
        return "admin".equals(username)
            && "password123".equals(password);
    }
}

import java.io.*;
import javax.servlet.*;
import javax.servlet.http.*;
import javax.servlet.annotation.*;

@WebServlet("/LoginServlet")
public class LoginServlet extends HttpServlet {

    protected void doPost(HttpServletRequest req,
                           HttpServletResponse res)
            throws ServletException, IOException {

        String username = req.getParameter("username");
        String password = req.getParameter("password");

        User user = new User(username, password);

        if (user.isValid()) {
            // Store user in session and forward to welcome
            HttpSession session = req.getSession();
            session.setAttribute("user", user);

            RequestDispatcher rd =
                req.getRequestDispatcher("welcome.jsp");
            rd.forward(req, res);
        } else {
            // Forward to error page with message
            req.setAttribute("error",
                "Invalid username or password");
            RequestDispatcher rd =
                req.getRequestDispatcher("error.jsp");
            rd.forward(req, res);
        }
    }
}

JSTL — JSP Standard Tag Library

JSTL provides a set of standard tags for common tasks — loops, conditionals, formatting, SQL — reducing or eliminating the need for scriptlets.

Add JSTL: <%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>
TagPurposeExample
<c:out>Output a value (with HTML escaping)<c:out value="${user.name}"/>
<c:set>Set a variable in a scope<c:set var="x" value="10" scope="session"/>
<c:if>Conditional rendering<c:if test="${age > 18}">...</c:if>
<c:choose>Switch/if-else block<c:choose><c:when test="..."><c:otherwise>
<c:forEach>Loop over a collection or range<c:forEach items="${list}" var="item">
<c:forTokens>Loop over a delimited string<c:forTokens items="a,b,c" delims=",">
<c:url>URL with session tracking<c:url value="/servlet" />
<c:redirect>HTTP redirect<c:redirect url="login.jsp"/>

Error Handling in JSP

JSP provides a built-in mechanism for catching unhandled exceptions and redirecting to a dedicated error page.

Step 1: Declare errorPage

On any JSP that might throw exceptions, add the errorPage attribute to the page directive:

<%@ page
  errorPage="error.jsp"
%>
<%
  int result = 10 / 0; // ArithmeticException
%>

Step 2: Create Error Page

Set isErrorPage="true" to get access to the exception implicit object:

<%@ page
  isErrorPage="true"
  contentType="text/html"
%>
<h2>Error Occurred</h2>
<p><%= exception.getMessage() %></p>
web.xml approach (preferred): Define error pages globally in web.xml: 
<error-page>
  <exception-type>java.lang.ArithmeticException</exception-type>
  <location>/error.jsp</location>
</error-page>
<error-page>
  <error-code>404</error-code>
  <location>/notfound.jsp</location>
</error-page>

JSP Key Takeaways:
• JSP = HTML + Java. Compiled to Servlet on first request. 7-step lifecycle.
• 3 scripting elements: scriptlets <% %>, expressions <%= %>, declarations <%! %>
• 3 directives: page (settings), include (static), taglib (custom tags)
• 9 implicit objects — request, response, out, session, application, config, pageContext, page, exception
• EL ${expr} is cleaner than scriptlets for outputting data
• MVC: Servlet = Controller, JSP = View, JavaBean = Model
• JSTL replaces scriptlets with clean tags: c:forEach, c:if, c:choose

NP1Expression Language arithmetic. A JSP page contains the following EL expression. What integer does it render?
${ (7 + 5) * 4 }
EL respects normal arithmetic precedence; the parentheses force the addition first.
7 + 5 = 12 12 * 4 = 48
Rendered value = 48.
NP2Scriptlet loop output count. How many Hello lines does this scriptlet write to the response?
<% for (int i = 1; i <= 12; i++) { out.println("Hello " + i + "<br>"); } %>
The loop runs for i = 1, 2, …, 12 inclusive — one out.println per iteration.
Iterations: 12 - 1 + 1 = 12
Output lines = 12.
NP3Form + result.jsp sum. A form posts num1=46 and num2=58. result.jsp contains the expression below. What number is printed?
<%= Integer.parseInt(request.getParameter("num1")) + Integer.parseInt(request.getParameter("num2")) %>
Each parameter is parsed to an int, then added.
46 + 58 = 104
Printed value = 104.
NP4JSP declaration method. A JSP declares a method and then calls it in an expression. What does the page display?
<%! int squared(int n) { return n * n; } %> <%= "Square is: " + squared(9) %>
The declaration defines squared(n) = n*n; the expression calls it with 9.
squared(9) = 9 * 9 = 81
Result = 81.
NP5jsp:useBean addition. A bean class Add has int add(int x, int y){ return x + y; }. The JSP runs:
<jsp:useBean id="beanObj" class="com.vit.example.Add"/> <% int answer = beanObj.add(125, 238); out.print("Total = " + answer); %>
What integer is printed as the total?
The bean's add method returns the sum of its two arguments.
125 + 238 = 363
Printed total = 363.
NP6EL nested arithmetic. Evaluate the rendered integer:
${ (3 + 4) * (10 - 6) }
Evaluate each parenthesised group, then multiply.
3 + 4 = 7 10 - 6 = 4 7 * 4 = 28
Rendered value = 28.
NP7EL modulo operator. JSP EL supports the mod operator. What integer does this render?
${ 15 mod 4 }
mod is the remainder operator (same as % in Java).
15 = 4 * 3 + 3 → 15 mod 4 = 3
Rendered value = 3.
NP8Implicit objects count. How many JSP implicit objects (predefined variables available in every JSP without declaration) are there?
The 9 implicit objects are: request, response, out, session, application, config, pageContext, page, exception. Total = 9.

📝 Exam-Style MCQs — 2 Marks Each

Each question is worth 2 marks — a mix of applied numerical and conceptual items. Click an option to check it; the correct answer and explanation appear instantly.

Q1 · 2 marks A JSP page contains the Expression Language statement below. What does the browser display?
<p>Result: ${ (10 + 5) * 2 - 4 }</p>
EL evaluates arithmetic with normal operator precedence (parentheses → multiplication → subtraction).
(10 + 5) = 15 → 15 × 2 = 30 → 30 − 4 = 26
Since EL is enabled by default in modern JSP, the literal is not shown — the value is. The answer is Result: 26.
Q2 · 2 marks How many times is "Hello" printed by the scriptlet loop below?
<% for (int i = 1; i <= 20; i++) { if (i % 3 == 0) { %> <p>Hello</p> <% } } %>
"Hello" is emitted only when i % 3 == 0 for i from 1 to 20.
Multiples of 3 in [1,20]: 3, 6, 9, 12, 15, 18 → count = 6
The HTML between the scriptlet braces runs inside the loop body, so it prints 6 times.
Q3 · 2 marks Given ${17 div 5} and ${17 mod 5} in a JSP page (EL), what two values appear?
In EL the div operator performs floating-point division, while mod gives the remainder.
17 div 5 = 17 / 5 = 3.4 · 17 mod 5 = 17 − (3 × 5) = 2
Unlike Java integer division, EL's div yields 3.4, and mod yields 2.
Q4 · 2 marks Which JSP scripting element is translated into an instance variable / method of the generated servlet class (placed outside the _jspService() method)?
The three scripting elements translate differently: a declaration <%! %> defines fields/methods at class level, outside _jspService(). A scriptlet's code and an expression (auto out.print) both go inside _jspService(). A directive is not a scripting element at all. The answer is the Declaration — <%! ... %>.
Q5 · 2 marks A developer wants to import a Java package and set the content type for the whole JSP page. Which directive should be used?
There are three directives: page (page-level settings such as import, contentType, errorPage), include (static include at translation time), and taglib (registers a tag library). Importing packages and setting content type are jobs for the page directive.
Q6 · 2 marks In the Model-2 (MVC) architecture for JSP/Servlet applications, which component acts as the Controller?
In Model-2 MVC: the Servlet is the Controller (receives requests, invokes logic, then forwards), the JavaBean/POJO is the Model (data + business logic), and the JSP is the View (presentation only). Putting logic in the JSP would be the older Model-1 anti-pattern. The answer is the Servlet.
