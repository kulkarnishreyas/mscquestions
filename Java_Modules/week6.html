1. Introduction to Swing

Swing is Java's primary GUI toolkit, introduced in Java 1.2 as part of the Java Foundation Classes (JFC). It superseded the older Abstract Window Toolkit (AWT) by addressing its fundamental limitations.

Key insight: AWT components are heavyweight — they delegate to the native OS for rendering. Swing components are lightweight — they paint themselves entirely in Java, giving you consistent behaviour across every platform.

Advantages of Swing over AWT

Platform-Independent Look & Feel

Swing ships with multiple pluggable L&Fs: Metal (default), Nimbus, Motif, and OS-native (Windows, GTK, Aqua). You switch them at runtime with a single call.

Lightweight Components

Every Swing component renders itself via Java2D. No OS handles, no peer objects. This means no Z-order clipping bugs between components and perfect cross-platform fidelity.

Richer Component Set

JTable, JTree, JTabbedPane, JEditorPane, JSpinner, JColorChooser — none of these existed in AWT.

Pluggable Look and Feel (PLAF)

The UI delegate pattern allows the appearance of every component to be swapped independently of its model and behaviour.

MVC Architecture

Each component separates Model (data), View (rendering), and Controller (input). E.g. JTable uses TableModel, TableCellRenderer, and TableCellEditor.

Accessibility & Internationalisation

Built-in accessibility API, full Unicode support, bidirectional text, and locale-aware components out of the box.

Thread rule: All Swing GUI updates must happen on the Event Dispatch Thread (EDT). Use SwingUtilities.invokeLater(Runnable) to schedule work on the EDT from any other thread.

2. Swing Component Hierarchy

Swing sits on top of AWT's Component/Container hierarchy. Understanding this tree tells you which methods are available on every object.

Rule of thumb: If a class name starts with J, it is a Swing component (extends JComponent). Classes without the J prefix (like Frame, Button) belong to AWT.

3. Basic Swing Program

Every Swing application follows a standard pattern: create a JFrame, configure it, add components, and make it visible — all from within SwingUtilities.invokeLater().

Java BasicSwingExample.java 
import javax.swing.*;

public class BasicSwingExample {
    public static void main(String[] args) {
        // All Swing code MUST run on the Event Dispatch Thread
        SwingUtilities.invokeLater(() -> {
            // 1. Create the top-level window
            JFrame frame = new JFrame("Swing Example");

            // 2. Close the app when the window is closed
            frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);

            // 3. Create a button
            JButton button = new JButton("Click Me!");

            // 4. Attach an action listener (lambda)
            button.addActionListener(e ->
                JOptionPane.showMessageDialog(frame, "Button was clicked!")
            );

            // 5. Add button to the frame's content pane
            frame.getContentPane().add(button);

            // 6. Size and display
            frame.setSize(300, 150);
            frame.setLocationRelativeTo(null); // centre on screen
            frame.setVisible(true);
        });
    }
}

4. Login Screen with GridLayout

GridLayout arranges components in a grid of equal-sized cells. GridLayout(rows, cols) fills left-to-right, top-to-bottom.

Java LoginScreen.java
import javax.swing.*;
import java.awt.*;

public class LoginScreen {
    public static void main(String[] args) {
        SwingUtilities.invokeLater(() -> {
            JFrame frame = new JFrame("User Login");
            frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
            frame.setSize(340, 160);
            frame.setLocationRelativeTo(null);

            // 3 rows, 2 columns → 6 cells
            JPanel panel = new JPanel(new GridLayout(3, 2, 8, 8));
            panel.setBorder(BorderFactory.createEmptyBorder(16, 16, 16, 16));

            JLabel     lblUser  = new JLabel("Username:");
            JTextField txtUser  = new JTextField(20);

            JLabel         lblPass  = new JLabel("Password:");
            JPasswordField txtPass  = new JPasswordField(20);

            JLabel  lblEmpty  = new JLabel("");       // spacer
            JButton btnLogin  = new JButton("Login");

            btnLogin.addActionListener(e -> {
                String user = txtUser.getText();
                // getPassword() returns char[] for security; never use getText()
                char[] pass = txtPass.getPassword();
                JOptionPane.showMessageDialog(frame,
                    "Welcome, " + user + "!\nPassword length: " + pass.length,
                    "Login Successful",
                    JOptionPane.INFORMATION_MESSAGE);
                // Zero out the char array — security best practice
                java.util.Arrays.fill(pass, '\0');
            });

            panel.add(lblUser);  panel.add(txtUser);
            panel.add(lblPass);  panel.add(txtPass);
            panel.add(lblEmpty); panel.add(btnLogin);

            frame.add(panel);
            frame.setVisible(true);
        });
    }
}
Security note: JPasswordField.getPassword() returns char[] (not String) because strings are immutable and remain in memory until GC. After using the password, overwrite the array with Arrays.fill(pass, '\0').

5. JOptionPane Dialogs

JOptionPane provides pre-built modal dialog boxes. Four static factory methods cover the most common use-cases.

MethodDialog typeReturns
showMessageDialogInformation, warning, error, plainvoid
showConfirmDialogYes / No / Cancel buttonsint (0=YES, 1=NO, 2=CANCEL)
showInputDialogText-entry fieldString (null if cancelled)
showOptionDialogFully custom buttonsint (index of chosen button)
Java JOptionPaneExample.java
import javax.swing.*;

public class JOptionPaneExample {
    public static void main(String[] args) {
        // 1. Message dialog — purely informational
        JOptionPane.showMessageDialog(null,
            "Hello, World!",
            "Greeting",
            JOptionPane.INFORMATION_MESSAGE);

        // 2. Input dialog — ask the user for text
        String name = JOptionPane.showInputDialog(null,
            "Enter your name:",
            "Name Input",
            JOptionPane.QUESTION_MESSAGE);

        if (name == null || name.isBlank()) return; // user cancelled

        // 3. Confirm dialog — ask Yes/No/Cancel
        int choice = JOptionPane.showConfirmDialog(null,
            "Greet " + name + "?",
            "Confirm",
            JOptionPane.YES_NO_CANCEL_OPTION);

        if (choice == JOptionPane.YES_OPTION) {
            // 4. Option dialog — custom button labels
            String[] buttons = { "Wave", "High-Five", "Skip" };
            int greeting = JOptionPane.showOptionDialog(null,
                "How to greet " + name + "?",
                "Greet",
                JOptionPane.DEFAULT_OPTION,
                JOptionPane.QUESTION_MESSAGE,
                null, buttons, buttons[0]);

            String msg = switch (greeting) {
                case 0 -> "*waves at " + name + "*";
                case 1 -> "*high-fives " + name + "*";
                default -> "Skipped greeting.";
            };
            JOptionPane.showMessageDialog(null, msg);
        } else if (choice == JOptionPane.NO_OPTION) {
            JOptionPane.showMessageDialog(null,
                "Goodbye, " + name + "!",
                "Farewell",
                JOptionPane.WARNING_MESSAGE);
        }
    }
}

JFileChooser displays a standard open/save dialog. It works purely in Java and looks consistent across platforms.

FileChooserDemo — open a file and display its content

Java FileChooserDemo.java
import javax.swing.*;
import java.io.*;
import java.nio.file.*;

public class FileChooserDemo {
    public static void main(String[] args) {
        SwingUtilities.invokeLater(() -> {
            JFrame frame = new JFrame("File Viewer");
            frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
            frame.setSize(600, 400);

            JTextArea textArea = new JTextArea();
            textArea.setEditable(false);
            JScrollPane scroll = new JScrollPane(textArea);

            JButton openBtn = new JButton("Open File...");
            openBtn.addActionListener(e -> {
                JFileChooser chooser = new JFileChooser();
                int result = chooser.showOpenDialog(frame);
                if (result == JFileChooser.APPROVE_OPTION) {
                    File file = chooser.getSelectedFile();
                    try {
                        String content = Files.readString(file.toPath());
                        textArea.setText(content);
                        frame.setTitle("Viewing: " + file.getName());
                    } catch (IOException ex) {
                        JOptionPane.showMessageDialog(frame,
                            "Error reading file: " + ex.getMessage(),
                            "Error", JOptionPane.ERROR_MESSAGE);
                    }
                }
            });

            frame.add(openBtn, BorderLayout.NORTH);
            frame.add(scroll,   BorderLayout.CENTER);
            frame.setVisible(true);
        });
    }
}

TextEditor — type text, save via JFileChooser

Java TextEditor.java
import javax.swing.*;
import java.io.*;
import java.nio.file.*;

public class TextEditor {
    public static void main(String[] args) {
        SwingUtilities.invokeLater(() -> {
            JFrame    frame    = new JFrame("Simple Text Editor");
            JTextArea editor   = new JTextArea();
            JButton   saveBtn  = new JButton("Save As...");

            saveBtn.addActionListener(e -> {
                JFileChooser chooser = new JFileChooser();
                if (chooser.showSaveDialog(frame) == JFileChooser.APPROVE_OPTION) {
                    try {
                        Files.writeString(
                            chooser.getSelectedFile().toPath(),
                            editor.getText());
                        JOptionPane.showMessageDialog(frame, "File saved!");
                    } catch (IOException ex) {
                        JOptionPane.showMessageDialog(frame,
                            "Save failed: " + ex.getMessage(),
                            "Error", JOptionPane.ERROR_MESSAGE);
                    }
                }
            });

            frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
            frame.setSize(500, 400);
            frame.add(new JScrollPane(editor), BorderLayout.CENTER);
            frame.add(saveBtn, BorderLayout.SOUTH);
            frame.setVisible(true);
        });
    }
}

7. Overview of JavaFX

JavaFX was introduced with Java 8 as the designated successor to Swing. From Java 11 onwards it was extracted into a separate open-source module (openjfx.io) and must be added as a dependency.

Scene Graph

All UI elements are nodes in a hierarchical tree. Parents lay out children; transformations and effects compose naturally up the tree.

FXML

UI layout is declared in XML files, cleanly separating design from logic. A FXMLLoader inflates the file into live objects at runtime.

CSS Styling

JavaFX nodes accept CSS stylesheets. You can theme an entire application by swapping a single .css file.

2D & 3D Graphics

Built-in Canvas for 2D drawing, plus a full 3D scene graph with lighting, cameras, and mesh shapes.

Animation API

Timeline, Transition subclasses, and AnimationTimer enable smooth, hardware-accelerated animations.

Properties & Bindings

JavaFX properties (e.g. StringProperty) support bidirectional bindings — change one value and all bound properties update automatically.

9. JavaFX Application Lifecycle

Every JavaFX app extends javafx.application.Application and follows a well-defined lifecycle managed by the JavaFX runtime.

MethodThreadNotes
launch()Main threadStatic; bootstraps the JavaFX Application Thread. Call once.
init()JavaFX-launcher threadOptional. Do not create Stages here.
start(Stage)JavaFX Application ThreadRequired override. All GUI setup goes here.
stop()JavaFX Application ThreadCalled when last window closes. Release file handles, DB connections, etc.

10. EventHandler in JavaFX

EventHandler<T extends Event> is a functional interface with a single method handle(T event). Three ways to implement it:

Java EventHandlerStyles.java
import javafx.event.*;
import javafx.scene.control.Button;

Button btn = new Button("Click");

// ── 1. Anonymous inner class ─────────────────────────────────
btn.setOnAction(new EventHandler<ActionEvent>() {
    @Override
    public void handle(ActionEvent event) {
        System.out.println("Clicked! (anonymous class)");
    }
});

// ── 2. Lambda expression (preferred) ─────────────────────────
btn.setOnAction(event -> System.out.println("Clicked! (lambda)"));

// ── 3. Method reference ───────────────────────────────────────
btn.setOnAction(this::handleClick);

private void handleClick(ActionEvent event) {
    System.out.println("Clicked! (method reference)");
}
Anonymous class — verbose but useful when you need to store state or call multiple methods. Clear intent when reading unfamiliar code.
Lambda — concise, preferred in modern Java. Use when the handler body is simple (1–3 lines). The compiler infers the functional interface.
Common event types: ActionEvent (button/menu), MouseEvent (click/drag), KeyEvent (key press), WindowEvent (close/hide), DragEvent, ScrollEvent.

11. SimpleJavaFXApp

Java SimpleJavaFXApp.java
import javafx.application.Application;
import javafx.scene.*;
import javafx.scene.control.*;
import javafx.scene.layout.VBox;
import javafx.stage.Stage;

public class SimpleJavaFXApp extends Application {

    @Override
    public void start(Stage primaryStage) {  // Step 1: entry point

        // Step 2: Create a Label (leaf node)
        Label label = new Label("Hello, JavaFX!");

        // Step 3: Create a Button (leaf node)
        Button button = new Button("Click Me");

        // Step 4: Attach an action handler
        button.setOnAction(e -> label.setText("Button Clicked!"));

        // Step 5: Create layout container
        VBox layout = new VBox(10);            // 10px spacing

        // Step 6: Add nodes to the layout
        layout.getChildren().addAll(label, button);

        // Step 7: Create Scene with root node and dimensions
        Scene scene = new Scene(layout, 300, 250);

        // Step 8: Set the scene on the Stage
        primaryStage.setScene(scene);

        // Step 9: Show the window
        primaryStage.show();
    }

    public static void main(String[] args) {
        launch(args);   // triggers init() → start() lifecycle
    }
}

Step-by-Step Breakdown

1
start(Stage primaryStage) — JavaFX runtime calls this after init(). The primaryStage is the main OS window.
2
Create Label — A read-only text node. Initial text: "Hello, JavaFX!".
3
Create Button — A clickable node with a text label: "Click Me".
4
Register setOnAction — Lambda that mutates the Label's text property when the button fires an ActionEvent.
5
Create VBox(10) — A layout pane that stacks children vertically with 10 px gaps.
6
getChildren().addAll() — Adds nodes to the VBox's observable list. Order determines visual order.
7
Create Scene — Wraps the root node (VBox) and specifies viewport dimensions (300 × 250 px).
8
primaryStage.setScene(scene) — Attaches the scene to the Stage. One Scene visible per Stage at a time.
9
primaryStage.show() — Makes the OS window visible. The JavaFX pulse loop begins rendering.

12. LoginScreen (JavaFX GridPane)

Java LoginScreenFX.java
import javafx.application.Application;
import javafx.geometry.*;
import javafx.scene.*;
import javafx.scene.control.*;
import javafx.scene.layout.GridPane;
import javafx.stage.Stage;

public class LoginScreenFX extends Application {

    @Override
    public void start(Stage stage) {
        GridPane grid = new GridPane();

        // Centre the grid in the scene
        grid.setAlignment(Pos.CENTER);

        // Gaps between rows and columns
        grid.setHgap(12);
        grid.setVgap(10);

        // Padding around the grid edges (top, right, bottom, left)
        grid.setPadding(new Insets(25, 25, 25, 25));

        // Row 0 — title spanning 2 columns
        Label title = new Label("Welcome Back");
        title.setStyle("-fx-font-size: 18px; -fx-font-weight: bold;");
        GridPane.setColumnSpan(title, 2);
        grid.add(title, 0, 0); // col=0, row=0

        // Row 1 — username
        grid.add(new Label("Username:"), 0, 1);
        TextField userField = new TextField();
        userField.setPromptText("Enter username");
        grid.add(userField, 1, 1);

        // Row 2 — password
        grid.add(new Label("Password:"), 0, 2);
        PasswordField passField = new PasswordField();
        passField.setPromptText("Enter password");
        grid.add(passField, 1, 2);

        // Row 3 — login button + feedback label
        Button  loginBtn = new Button("Sign In");
        Label   feedback = new Label();
        feedback.setStyle("-fx-text-fill: #00d4aa;");

        loginBtn.setOnAction(e -> {
            String user = userField.getText();
            String pass = passField.getText();
            if (user.isBlank() || pass.isBlank()) {
                feedback.setText("Please fill in all fields.");
                feedback.setStyle("-fx-text-fill: #ff6b6b;");
            } else {
                feedback.setText("Hello, " + user + "!");
                feedback.setStyle("-fx-text-fill: #00d4aa;");
            }
        });

        grid.add(loginBtn, 1, 3);
        grid.add(feedback, 1, 4);

        stage.setScene(new Scene(grid, 360, 240));
        stage.setTitle("Login");
        stage.show();
    }

    public static void main(String[] args) { launch(args); }
}

13. AWT vs Swing vs JavaFX — Comparison

Feature AWT Swing JavaFX
Introduced Java 1.0 (1996) Java 1.2 (1998) Java 8 (2014); standalone from Java 11
Component weight Heavyweight (OS native peers) Lightweight (Java-rendered) Lightweight + hardware-accelerated (Prism)
Look & Feel Always OS native Pluggable (Metal, Nimbus, OS native) CSS + modena theme; fully customisable
UI declaration Java code only Java code only Java code or FXML (XML)
Styling Minimal (setBackground, fonts) UIManager / synth Full CSS3 subset
Data binding None None (manual listeners) Built-in Properties + bidirectional bindings
Animation Manual (Swing Timer) Swing Timer / Animator Timeline, Transitions, AnimationTimer
3D graphics No No Yes (Box, Sphere, custom meshes, cameras)
Threading model EDT (single-threaded) EDT (single-threaded) JavaFX Application Thread
Rich text No JEditorPane / JTextPane HTMLEditor, WebView (full WebKit)
Table / Tree No JTable, JTree TableView, TreeView (virtual scrolling)
Charts No No (third-party needed) Yes — BarChart, LineChart, PieChart, etc.
MVC separation Minimal Model+UI delegate Strong (FXML + Controller + Properties)
Modern status Legacy — avoid for new work Maintained; widely used Active development; preferred for new apps
When to use Swing: Maintaining existing enterprise applications, desktop tools that need minimal dependencies, or when targeting Java environments without JavaFX on the classpath.
When to use JavaFX: New GUI applications, data-rich dashboards, apps that need charts, 3D, web views, or tight CSS-based theming. Add the openjfx Maven/Gradle dependency and you are good to go.

🔢 Engineered Numerical Practice

Fresh exam-style computational problems built from this week's concepts. Type your answer and hit Check for instant feedback, or reveal the worked steps. (Press Enter in any answer box to check.)
NP1JOptionPane return value. A Yes/No/Cancel confirm dialog is shown and the user clicks Cancel. What int does showConfirmDialog(...) return? (Recall YES_OPTION=0, NO_OPTION=1, CANCEL_OPTION=2.)
int r = JOptionPane.showConfirmDialog(null, "Save changes?", "Confirm", JOptionPane.YES_NO_CANCEL_OPTION);
The button constants map to fixed ints: YES=0, NO=1, CANCEL=2.
user clicked Cancel → CANCEL_OPTION = 2
The method returns 2.
NP2GridPane cell → linear index. A JavaFX GridPane with 3 columns is filled row-by-row (row-major). A control is added at grid.add(node, 2, 3) — i.e. column 2, row 3 (both 0-based). What is its 0-based linear position counting left-to-right, top-to-bottom?
In row-major order: index = row × columns + col. Note add(node, col, row) takes column first.
index = 3 × 3 + 2 = 9 + 2 = 11
Linear index = 11.
NP3Scene area. A JavaFX scene is built with new Scene(grid, 300, 275) (width, height in px). What is the scene's area in square pixels?
Scene(root, width, height) → area = width × height.
area = 300 × 275 = 82500
The scene area is 82500 px².
NP4JavaFX lifecycle calls. A JavaFX Application is launched once, runs, then the window is closed normally. Counting init(), start() and stop(), how many lifecycle-method invocations occur in total over that single run?
Each of these runs exactly once per application run.
init() = 1 start() = 1 stop() = 1 total = 3
Total = 3 calls.
NP5Scene-graph node count. The code builds a layout and adds two controls to it. Counting the root container itself plus all the nodes it contains, how many nodes are in this scene graph?
Label label = new Label("Hello, JavaFX!"); Button button = new Button("Click Me"); VBox layout = new VBox(10); layout.getChildren().addAll(label, button);
root VBox = 1 children = label + button = 2 total = 1 + 2 = 3
The scene graph has 3 nodes.
NP6Container component count. A Swing login frame using GridLayout adds exactly these to the frame: a username JLabel, a username JTextField, a password JLabel, a JPasswordField, and a Login JButton. How many components are added?
2 labels + 1 text field + 1 password field + 1 button = 5
5 components are added.
NP7YES_OPTION index. In a Yes/No confirm dialog the user clicks Yes. What int value does JOptionPane.YES_OPTION equal (the value returned)?
The Swing constants are indexed from zero: YES_OPTION = 0, NO = 1, CANCEL = 2.
YES_OPTION = 0
Answer: 0.
NP8Top-level container — concept. In the JavaFX primary-elements hierarchy (Stage → Scene → Nodes), which element is the top-level container that represents the primary application window?
The Stage is the top-level JavaFX container (the window). Each Stage holds one Scene, and the Scene holds the tree of Nodes (controls, layout panes like VBox). Answer: A — Stage.

Q1 · 2 marks A JavaFX confirmation alert offers Yes / No / Cancel. The user presses Cancel. Using the classic Swing constant convention YES=0, NO=1, CANCEL=2, what integer index corresponds to the chosen option?
The option constants are zero-indexed in declaration order.
YES = 0, NO = 1, CANCEL = 2
Cancel maps to 2 (answer B).
Q2 · 2 marks Nodes are placed in a JavaFX GridPane that has 3 columns. A button is added at column 2, row 1 with gp.add(btn, 2, 1) (signature add(node, colIndex, rowIndex)). If cells are numbered left-to-right, top-to-bottom starting at 0, what is this cell's linear index?
For a row-major grid, index = row × columns + col.
index = 1 × 3 + 2 = 5
The cell's linear index is 5 (answer C). Note the arguments are (col, row), a common trap.
Q3 · 2 marks A JavaFX scene is created with new Scene(root, 600, 400) (width, height in px). What is the scene's area in square pixels?
Area = width × height.
600 × 400 = 240000
The scene covers 240000 px² (answer C). Option A is the mistake of adding 600 + 400.
Q4 · 2 marks What is the core architectural difference between an AWT Button and a Swing JButton?
AWT components delegate rendering to native OS peers (heavyweight). Swing components are lightweight — they paint themselves with Java2D, giving identical appearance on every platform. Answer: A.
Q5 · 2 marks Swing is said to support a pluggable look-and-feel. What does that allow a developer to do?
Because Swing widgets paint themselves, the look-and-feel is decoupled from behaviour. A single UIManager.setLookAndFeel(...) call swaps the whole theme (Metal, Nimbus, Motif, OS-native) without changing the component code. Answer: D.
Q6 · 2 marks In the MVC pattern as applied to a GUI form, which component is responsible for holding the application's data and business state (and is unaware of how it is displayed)?
In MVC the Model stores data and business logic; the View renders it; the Controller handles input and updates the Model. The Model is intentionally decoupled from any display concern. Answer: C.
