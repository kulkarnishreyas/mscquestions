⭐ Professor's Golden Rules

Answer everything — never leave a blank. "Even if you don't know the syntax, write at least the outline / a few relevant lines. Generally marks are given for attempts."
The uploaded slides are enough. No need to study from outside sources — but you must understand the code, not just memorise it (e.g. why RemoteException, why UnicastRemoteObject, what each line does).
Every module carries a program + MCQs. The paper is 3 sets of questions covering all 7 modules — expect 1-mark MCQs, 2-mark MCQs, and program-based questions from each.
For paired programs, one side is enough. In networking (TCP/UDP), write either the server or the client program — not both.

Module 2 — Exception Handling & Multithreading Week 3 Week 4

🔴 Exception Handling

  1. If you divide 5 / 0, what type of exception is raised? MCQ ⭐ArithmeticException (divide by zero).
  2. What is the difference between the catch block and the finally block? theory ⭐finally always executes (last); catch runs only if the matching error is generated in try.
  3. Name the exception-handling keywords. MCQtry, catch, finally, throw, throws.
  4. What is the purpose of the throw keyword? theoryTo manually throw an exception/error.
  5. Write a user-defined exception program (e.g. if age < 18 throw "invalid citizen"; or validate subject marks / product quantity). program ⭐

🧵 Multithreading

  1. Write a multithreaded program using either the Thread class or the Runnable interface (either is allowed). program ⭐
  2. What is the purpose of Thread.sleep(1000)? MCQPauses the thread for the given milliseconds.
  3. What is the purpose of the wait() method? What do start() and run() do? MCQ
  4. Name the thread lifecycle states / methods. theory
  5. What is synchronization? What is the synchronized keyword used for? theory ⭐Locks a shared resource so only one thread accesses it at a time, then unlocks it for the next request.

Module 7 — Networking & RMI Week 11 Week 12

"The easiest chapter." Networking = program-heavy; RMI = one program + theory.
TCP / UDP program RMI program MCQ + theory

🌐 Networking — Theory & MCQ

Professor: "From networking you expect the program; from the program you also get theory MCQs. Only the 4 programs (TCP client/server, UDP client/server) matter."
  1. Which class is used for the client vs the server in TCP? MCQ ⭐Client = Socket; Server = ServerSocket. (Package: java.net.*)
  2. Which classes are used for UDP? MCQ ⭐DatagramSocket (connection) + DatagramPacket (message) — used in BOTH client & server.
  3. Difference between TCP and UDP (connection-oriented vs connectionless, reliable vs unreliable). theory ⭐TCP = connection-oriented, sends acknowledgement, reliable. UDP = connectionless, no acknowledgement, unreliable but faster.
  4. What is the equivalent IP address of localhost? MCQ ⭐127.0.0.1
  5. Given a URL, what do getProtocol(), getHost(), getPort(), getFile(), getPath(), getQuery() return? MCQRequires the URL class from java.net.*.
  6. What does InetAddress.getByName(...) return? MCQThe IP address details of the given host name.
  7. Difference between IPv4 and IPv6? MCQIPv4 = 32-bit; IPv6 = 128-bit (far more addresses).

💻 Networking — Programs

  1. Write a TCP program (client OR server — one side is enough). program ⭐
  2. Write a UDP program (client OR server). May include string handling — e.g. convert received text to uppercase, count words, or sentence length. program ⭐

🛰️ RMI — Remote Method Invocation

Professor: "Only one program. Understand the code — why RemoteException, why UnicastRemoteObject, purpose of rebind. Expect code-based MCQs (find the answer from the code)."
  1. What is RMI? What are stub and skeleton? theory ⭐Stub (client-side) packages the request; skeleton (server-side) passes it to the remote method and returns the result.
  2. A remote interface extends which interface? Which exception does it throw? MCQ ⭐Extends Remote; throws RemoteException. Package: java.rmi.*
  3. The implementation class extends which superclass? MCQ ⭐UnicastRemoteObject.
  4. What is the purpose of the Naming.rebind() / bind() method? And Naming.lookup()? MCQbind/rebind = register a remote object under a name; lookup = get a reference to a remote object.
  5. Name the Naming class methods. MCQbind, rebind, unbind, lookup, list.
  6. What is the rmic tool / rmiregistry used for? theoryrmic generates the stub/skeleton; the registry stores remote services on a port.
  7. Write an RMI program (e.g. remote add(a,b)) — follow the 6 steps: interface → implementation → server (rebind) → client (lookup). program ⭐
