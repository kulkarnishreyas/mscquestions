1️⃣ Ensemble Methods

Combine multiple models for better, more robust predictions. Key principles: Diversity, Independence, Aggregation.

Bagging vs Boosting

FeatureBaggingBoosting
TrainingParallelSequential
FocusVariance reductionBias reduction
Overfit RiskLowHigher if untuned
ExamplesRandom ForestAdaBoost, GBM, XGBoost, LightGBM
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier rf = RandomForestClassifier(n_estimators=100).fit(X_train, y_train) gbm = GradientBoostingClassifier().fit(X_train, y_train)

Applications in Finance

  • Credit Risk — Random Forest assesses default likelihood from many attributes
  • Stock Forecasting — GBM uncovers complex patterns
  • Fraud Detection — AdaBoost focuses on misclassified transactions (imbalanced data)

2️⃣ Support Vector Machines

SVM finds the hyperplane that best separates classes with maximum margin. Support vectors = points closest to the boundary.

f(x) = w · x + b    (sign > 0 → +1, < 0 → −1)

Kernel Functions

KernelUse
LinearLinearly separable data
PolynomialCurved boundaries
RBF (Gaussian)Most non-linear cases — flexible
from sklearn.svm import SVC svm = SVC(kernel='rbf', C=1.0, gamma='scale') svm.fit(X_train, y_train) y_pred = svm.predict(X_test)

Tuning Parameters

  • C: regularisation. Low C = wider margin (more tolerance for errors)
  • Gamma (γ): kernel reach. High γ = tight fit per point

Strengths & Limits

✅ Strengths❌ Limits
High-dimensional dataSlow on large datasets
Robust to overfitting (well-tuned)"Black box" — less interpretable
Effective with clear marginsSensitive to imbalanced data

Finance Applications

  • Credit scoring (default / no-default)
  • Stock direction prediction
  • Fraud detection

4️⃣ Deep Learning

Multi-layer neural networks for very complex tasks.

Key Architectures

🖼️ CNN

Spatial data — scanned cheques, chart patterns, document OCR

🔁 RNN

Sequential — credit payment history, transaction sequences

⏰ LSTM

Unsupervised — anomaly/fraud detection, dimensionality reduction

Challenges

  • Resource-intensive — GPUs/TPUs required

  • Bias risk — biased data → biased model

Trend Prediction Pipeline

  3. Apply sentiment scoring
  4. Aggregate over time, plot trend

6️⃣ Reinforcement Learning

Agent learns by interacting with environment, receiving rewards/penalties, maximising cumulative reward over time. Perfect for finance where markets change.

Q-Learning Concepts

  • State — current market condition
  • Action — Buy / Hold / Sell
  • Reward — profit/loss
  • Q-table — expected reward for state-action pairs
  • ε-greedy — balance exploration vs exploitation
Q(s,a) ← Q(s,a) + α [r + γ · max Q(s', a') − Q(s,a)]

RL Applications in Finance

📈 Algorithmic Trading

Learn buy/sell rules from market data

💼 Portfolio Mgmt

Dynamic rebalancing based on volatility

🛡️ Fraud Detection

PayPal — continuously learn new fraud patterns

📊 Credit Risk

Citigroup — adaptive default prediction

Real-World Use Cases

  • JPMorgan Chase — RL-based dynamic trading strategies
  • BlackRock — automated asset allocation
  • PayPal — real-time fraud detection improving over time
  • Citigroup — credit risk prediction
  • Two Sigma — high-frequency trading
⚠️ Limitations: needs lots of data, computationally heavy, can overfit historical data, less interpretable.

📝 Summary & Exam Tips

🎯 Key Takeaways

  • Bagging (Random Forest) reduces variance; Boosting (XGBoost) reduces bias
  • SVM finds max-margin hyperplane; RBF kernel for non-linear data
  • Neural Networks excel at non-linear, high-dim data but are black-boxes
  • Deep Learning architectures: CNN (spatial), RNN (sequential), LSTM (long memory), Autoencoder (anomaly)
  • NLP in finance: sentiment, NER, chatbots, topic modelling
  • RL: agent-environment interaction with rewards; powers trading, portfolio management, fraud detection
  • Real-world: JPMorgan, BlackRock, PayPal, Citigroup, Two Sigma
📌 Pitfalls:
  • Choosing deep learning when simple models suffice — adds complexity without benefit
  • Black-box models may fail regulatory scrutiny — pair with SHAP/LIME
  • SVMs on huge datasets are slow — consider RF/GBM instead
  • RL agents can overfit historical data — validate forward
